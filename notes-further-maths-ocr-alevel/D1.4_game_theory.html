D1.4: Game Theory

1. Zero-Sum Two-Player Games

A zero-sum two-player game has two players (R and C) whose gains sum to zero at every outcome. A pay-off matrix \(\mathbf{A}\) shows the pay-offs to player R: if R plays row \(i\) and C plays column \(j\), R wins \(a_{ij}\) (and C wins \(-a_{ij}\)).

Each player chooses a strategy — we'll first look for a deterministic (pure) solution, then mixed strategies.

2. Play-Safe (Maximin / Minimax)

Row Player's Analysis

For each row, find the minimum (worst case for R), then choose the row with the largest such minimum. This is the maximin value.

Column Player's Analysis

For each column, find the maximum (worst case for C), then choose the column with the smallest such maximum. This is the minimax value.

Saddle Point and Stable Solution

A saddle point occurs when maximin = minimax. The game has a stable solution with a pure strategy for each player, and the value of the game equals this common number.

3. Dominance

Row \(i\) dominates row \(k\) (for R) if every entry of row \(i\) is \(\ge\) the corresponding entry of row \(k\). R will never play a dominated row — remove it.

Similarly for columns (from C's point of view — but C wants small values, so column \(j\) dominates column \(k\) if every entry of \(j\) is \(\le\) the corresponding entry of \(k\)).

Iteratively remove dominated rows and columns to simplify.

4. Mixed Strategies for a \(2\times 2\) Game

Suppose the reduced matrix is

\(\begin{pmatrix} a & b \\ c & d \end{pmatrix}\)

R plays row 1 with probability \(p\), row 2 with probability \(1 - p\). For R's expected pay-off to be independent of C's choice:

\(pa + (1-p)c = pb + (1-p)d,\)

giving \(p = \dfrac{d - c}{(a - b) + (d - c)}\) and value \(V = pa + (1 - p)c\).

Similarly for C's probability \(q\) of playing column 1.

5. Larger Games via Linear Programming

For \(m \times n\) games with no pure-strategy solution and no reduction to \(2\times 2\), we use LP.

LP Formulation (Row Player)

R chooses \(p_1, \ldots, p_m \ge 0\) with \(\sum p_i = 1\). R wants to maximise the minimum expected gain \(V\):

\(\text{Maximise } V \text{ subject to } \sum_i p_i a_{ij} \ge V \text{ for each } j, \quad \sum p_i = 1.\)

After a shift to ensure positivity, this becomes a standard LP — solve with the Simplex method.

Trick: Positive Payoffs

If any pay-off is \(\le 0\), add a constant \(k\) to all entries so they are all positive. This shifts the value of the game by \(k\) but does not change the optimal strategies.

6. Interpretation and Context

Typical questions ask you to:

  • Identify a saddle point (if any) and state the value.
  • Reduce via dominance and check.
  • Find mixed strategies if no saddle point.
  • Formulate the LP for a larger game (but not necessarily solve it fully in exam time).

7. Exam Style Strategy

  • Always start with play-safe strategies — if maximin = minimax, you're done.
  • Try dominance before anything else — it often shrinks the problem to \(2 \times 2\).
  • For \(2 \times 2\) mixed, solve for \(p\) by equating the two possible expected pay-offs.
  • Express the value in exact form (fractions, not decimals) when possible.
