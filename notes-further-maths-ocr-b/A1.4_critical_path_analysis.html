OCR B A-Level Further Mathematics (MEI, H645) — Modelling with Algorithms Section A1.4

A1.4: Critical Path Analysis

1. Project Networks

A project network models activities (tasks) and their dependencies. In activity-on-arc (AOA) style, arrows represent activities, nodes represent events (start/end points).

2. Forward and Backward Passes

For each event \(i\):

  • Early time \(E_i\): earliest possible start time — computed via forward pass (max over predecessors).
  • Late time \(L_i\): latest allowable time without delaying project — computed via backward pass (min over successors).

3. Critical Path

The critical path is a path from start to end where every activity has zero float. Its length is the minimum project duration.

Total Float of Activity \((i\to j)\) of duration \(d\)

\(\text{TF}=L_j-E_i-d\)

Zero TF ⇒ on critical path.

Free Float

\(\text{FF}=E_j-E_i-d\)

Amount activity can be delayed without affecting any successor start.

4. Dummy Activities

Zero-duration arrows used to enforce dependencies in AOA representation when activities share a start or end event.

5. Gantt Charts

Horizontal bars along time axis. Critical activities at their earliest start; non-critical show float as a shaded extension.

6. Resource Scheduling

Real projects have resource limits. Level the schedule by shifting non-critical activities within float windows to even out resource usage.

7. Crashing

Shortening activity durations at extra cost. Crash activities on the critical path starting with the cheapest cost-per-day saved.

8. PERT Extension

Each activity has optimistic (\(o\)), most likely (\(m\)), pessimistic (\(p\)) estimates. Expected duration:

\(t_e=\dfrac{o+4m+p}{6}\)

Variance of activity = \(((p-o)/6)^2\). Project variance = sum along critical path.

9. Exam Strategy

  • Draw network neatly; label events.
  • Perform forward pass from start (E), then backward pass from end (L).
  • Identify critical path by zero-float activities.
  • Compute floats for all activities and state critical path duration.
  • For scheduling/crashing, focus on critical path first.
