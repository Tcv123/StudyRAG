4.7.1 Internal Hardware Components of a Computer

The Processor (CPU)

The Central Processing Unit (CPU) is the primary component responsible for executing instructions. It contains the Arithmetic Logic Unit (ALU), the Control Unit (CU), and a set of registers. Modern CPUs also contain cache memory to speed up data access.

Processor: An electronic circuit that executes instructions from a program by performing arithmetic, logic, control, and input/output operations.

Main Memory

Main memory (primary storage) holds data and instructions that the CPU needs to access quickly. It consists of:

  • RAM (Random Access Memory) — volatile; stores currently running programs and data. Contents are lost when power is removed. Can be read from and written to.
  • ROM (Read Only Memory) — non-volatile; stores firmware such as the BIOS/UEFI that boots the computer. Contents are permanent and cannot be modified during normal operation.

The System Bus

The system bus is a set of parallel wires that connects the CPU to main memory and other components. It consists of three distinct buses:

BusPurposeDirection
Address BusCarries memory addresses from the CPU to memory/I O devices to identify the location being accessedUnidirectional (CPU → memory)
Data BusCarries actual data between the CPU, memory, and I/O devicesBidirectional
Control BusCarries control signals (read, write, clock, interrupt, bus request/grant) to coordinate operationsBidirectional
The width of the address bus determines how many unique memory locations can be addressed (2n locations for an n-bit bus). The width of the data bus determines how much data can be transferred in a single operation.

Control signals carried on the control bus include:

  • Memory Read / Memory Write — signals to read from or write to memory
  • Bus Request / Bus Grant — used by devices requesting control of the bus
  • Clock — the timing signal that synchronises operations
  • Interrupt — signals from I/O devices requesting CPU attention
  • Reset — resets the CPU to its initial state

4.7.2 Von Neumann Architecture

Stored Program Concept: The idea that instructions and data are stored together in the same main memory, and instructions are fetched and executed sequentially (unless a branch instruction is encountered).

The Von Neumann architecture (proposed by John von Neumann in 1945) is based on the stored program concept. Key features:

  • A single memory space holds both instructions and data
  • A single bus system connects the CPU to memory
  • Instructions are fetched and executed sequentially
  • The CPU contains an ALU, CU, and registers

Key Registers

RegisterAbbreviationPurpose

Memory Address RegisterMARHolds the address of the memory location currently being read from or written to. Connected to the address bus.
Memory Data RegisterMDR (or MBR)Holds the data that has just been read from memory or is about to be written to memory. Connected to the data bus.
Current Instruction RegisterCIRHolds the instruction currently being decoded and executed. The instruction is split into opcode and operand.
AccumulatorACCA general-purpose register that stores intermediate results of arithmetic and logic operations performed by the ALU.

In exam questions, always be precise about what each register holds. The PC holds the address of the next instruction, not the current one. The MAR holds an address; the MDR holds data or an instruction that was fetched from that address.

4.7.3 Harvard Architecture

The Harvard architecture uses physically separate memory and buses for instructions and data. This allows the CPU to fetch an instruction and read/write data simultaneously.

FeatureVon NeumannHarvard
MemorySingle shared memory for instructions and dataSeparate memory for instructions and data
Bus systemSingle bus (potential bottleneck)Separate buses for instruction and data
Simultaneous accessNo — fetch or data access, not bothYes — can fetch and access data at the same time
SpeedSlower (Von Neumann bottleneck)Faster for specific tasks
FlexibilityMore flexible (programs can treat data as code)Less flexible (fixed memory allocation)
Typical useGeneral-purpose computersEmbedded systems, DSPs, microcontrollers
Von Neumann Bottleneck: The limitation that arises because both data and instructions share the same bus, meaning the CPU can be idle while waiting for data to be transferred. Harvard architecture avoids this by having separate buses.

Modified Harvard architecture is used in most modern processors: separate L1 caches for instructions and data (Harvard), but a unified main memory (Von Neumann). This provides the speed benefits of Harvard while retaining the flexibility of Von Neumann.

4.7.4 The Fetch-Decode-Execute Cycle

The CPU continuously repeats three stages to process each instruction:

1. Fetch

  1. The contents of the PC are copied to the MAR via the address bus.
  2. The PC is incremented (PC ← PC + 1) to point to the next instruction.
  3. The address in the MAR is sent along the address bus to main memory.
  4. The control unit sends a memory read signal along the control bus.
  5. The data at the addressed memory location is transferred along the data bus into the MDR.
  6. The contents of the MDR are copied to the CIR.

2. Decode

  1. The control unit decodes the instruction held in the CIR.
  2. The instruction is split into the opcode (the operation to perform) and the operand (the data or address to operate on).
  3. The CU determines what signals need to be sent to execute the instruction.
  4. If the operand is a memory address, additional memory accesses may be needed (depending on the addressing mode).

3. Execute

  1. The decoded instruction is carried out. This could involve:
    • An ALU operation (arithmetic or logic calculation)
    • A data transfer between registers, memory, or I/O devices
    • A branch/jump — loading a new address into the PC
    • A comparison that sets flags in the status register
    • A halt instruction that stops the CPU
  2. Results may be stored in the accumulator or written back to memory.
  3. The status register is updated to reflect the result (e.g., zero flag, carry flag).
When describing the FDE cycle, always reference the specific registers and buses involved. Saying "the instruction is fetched from memory" is not detailed enough — you must describe the role of PC, MAR, MDR, CIR, and the address/data/control buses.

Interrupts

After the execute phase, the CPU checks for interrupts. An interrupt is a signal from hardware or software requesting the CPU's attention (e.g., I/O completion, timer, error). If an interrupt is detected and interrupts are enabled:

  1. The CPU saves its current state (register contents, PC value) onto the stack.
  2. The PC is loaded with the address of the Interrupt Service Routine (ISR).
  3. The ISR is executed.
  4. On completion, the CPU restores its previous state from the stack and resumes the interrupted program.

4.7.5 Processor Instruction Set

Machine Code: The lowest-level programming language, consisting of binary instructions that the CPU can directly execute. Each instruction consists of an opcode and (usually) one or more operands.

Instruction Format

A machine code instruction is divided into two parts:

  • Opcode — specifies the operation to perform (e.g., ADD, LOAD, STORE, BRANCH, HALT)
  • Operand — specifies the data to use or the address of the data. Some instructions have no operand (e.g., HALT).
| Opcode | Operand |
|  ADD   |  #42    |   ← Add the value 42 to the accumulator
|  LDR   |  200    |   ← Load the value at memory address 200

Types of Operation

  • Data transfer: LOAD, STORE, MOVE
  • Arithmetic: ADD, SUB, MUL, DIV
  • Logical: AND, OR, NOT, XOR
  • Shift: LSL (logical shift left), LSR (logical shift right), ASR (arithmetic shift right)
  • Branch: BRA (unconditional), BEQ (branch if equal/zero), BNE (branch if not equal), BGT (branch if greater than)
  • Comparison: CMP (compare two values, set flags)
  • I/O: IN, OUT
  • HALT: Stop execution

Addressing Modes

ModeDescriptionExample
ImmediateThe operand is the actual data value to be used.ADD #5 — add the value 5
DirectThe operand is the memory address where the data is stored.LDR 200 — load the value stored at address 200
IndirectThe operand is the address of a memory location that contains the address of the actual data.LDR @200 — go to address 200, read the value there, and use that value as the address to load from
IndexedThe operand is added to the contents of an index register to form the effective address.LDR 200, X — load from address (200 + value in index register X)
Indirect addressing is like a pointer — the memory location holds a reference to another memory location. Indexed addressing is useful for iterating through arrays: the base address stays constant while the index register is incremented.

4.7.6 Factors Affecting Processor Performance

FactorHow It Affects Performance
Clock SpeedMeasured in GHz. Higher clock speed means more FDE cycles per second. However, increasing clock speed generates more heat and consumes more power.
Number of CoresEach core can execute its own FDE cycle independently. More cores allow parallel processing of tasks. However, software must be designed for parallelism to benefit, and not all tasks can be parallelised.
Cache Size and LevelsCache is fast SRAM between the CPU and main memory. L1 is fastest and smallest; L2 is larger but slower; L3 is shared between cores. Larger cache reduces memory access time by storing frequently used data closer to the CPU.
Word LengthThe number of bits the CPU can process in a single operation (e.g., 32-bit, 64-bit). A larger word length means more data can be processed per cycle and more memory can be addressed.
Bus WidthA wider data bus can transfer more data per clock cycle. A wider address bus can address more memory locations.
PipeliningMultiple instructions are overlapped by executing different stages of different instructions simultaneously. While instruction 1 is being executed, instruction 2 is decoded, and instruction 3 is fetched.
When asked about factors affecting performance, always explain how and why each factor improves performance, and mention any limitations. For example, more cores do not help if software is single-threaded.

4.7.7 Pipelining

Pipelining is a technique where multiple instructions are overlapped during execution. Each stage of the FDE cycle handles a different instruction simultaneously:

Clock CycleFetch UnitDecode UnitExecute Unit
1Instruction 1——
2Instruction 2Instruction 1—
3Instruction 3Instruction 2Instruction 1
4Instruction 4Instruction 3Instruction 2

Benefits: Higher throughput — after the pipeline is full, one instruction completes per cycle instead of every three cycles.

Limitations:

  • Pipeline hazards: Branch instructions can cause the pipeline to be flushed (all partially completed instructions are discarded) because the next instruction is unknown until the branch is resolved.
  • Data hazards: An instruction may depend on the result of a previous instruction that has not yet completed.
  • Pipeline stalls: The pipeline may need to wait (insert "bubbles") when hazards occur.

Branch prediction is used by modern processors to guess the outcome of a branch and speculatively fetch instructions. If the prediction is wrong, the pipeline is flushed.

4.7.9 Multicore and Parallel Processing

Multicore Processors

A multicore processor has two or more independent processing units (cores) on a single chip. Each core has its own ALU, CU, and L1 cache, and can execute its own FDE cycle independently.

  • Dual-core: 2 cores
  • Quad-core: 4 cores
  • Octa-core: 8 cores

Advantages:

  • Multiple tasks can be processed simultaneously (true parallelism)
  • Better performance for multi-threaded applications
  • Lower power consumption and heat generation compared to increasing clock speed

Limitations:

  • Software must be written to use multiple threads to benefit from multiple cores
  • Not all tasks can be parallelised (some are inherently sequential)
  • Amdahl's Law — the speedup from parallelism is limited by the proportion of the task that must be done sequentially
  • Overhead from coordinating cores (inter-core communication, synchronisation)

Parallel Processing

Parallel processing involves dividing a task into smaller sub-tasks that can be executed simultaneously across multiple cores or processors. Types include:

  • Data parallelism: The same operation is applied to different pieces of data simultaneously (e.g., applying a filter to every pixel in an image).
  • Task parallelism: Different operations are performed simultaneously on the same or different data.

4.7.10 Co-processors (GPUs)

Co-processor: A supplementary processor that is designed to handle specific types of computation, offloading work from the main CPU.

Graphics Processing Unit (GPU)

A GPU is a co-processor originally designed for rendering graphics but now widely used for general-purpose parallel computation (GPGPU).

FeatureCPUGPU
CoresFew powerful cores (4–16 typical)Thousands of simpler cores
Optimised forSequential, complex tasksMassively parallel, repetitive tasks
Control logicComplex (branch prediction, out-of-order execution)Simpler per-core
Use casesGeneral-purpose computing, OS tasksGraphics rendering, machine learning, cryptography, scientific simulations

Applications of GPU computing:

  • Training and running neural networks / machine learning models
  • Cryptocurrency mining
  • Scientific simulations (weather, molecular dynamics)
  • Image and video processing
  • Real-time ray tracing in gaming
