4.9.1 Communication Methods

Serial vs Parallel Transmission

Skew: In parallel transmission, bits arrive at different times due to wire length/impedance differences, corrupting data.
Crosstalk: EM interference between adjacent wires where one wire's signal induces current in another.

Synchronous vs Asynchronous Transmission

FeatureSynchronousAsynchronous
TimingContinuous stream, shared clock signalOne byte at a time, framed with start bit (0) and stop bit(s) (1)

EfficiencyHigher (minimal framing overhead)Lower (~20% overhead for start/stop bits)
Use casesNetwork links, video streaming, VoIPKeyboard, RS-232, UART, Bluetooth

Bit Rate, Baud Rate, Bandwidth, Latency

Bit rate: Bits transmitted per second (bps) — actual data throughput.
Baud rate: Signal changes (symbols) per second. Each symbol can encode multiple bits.
Bandwidth: Range of frequencies available. Higher bandwidth = higher data rate.
Latency: Time delay between sending and receiving. Affected by distance, hops, congestion. 
Bit rate = Baud rate x bits per symbol

Example: baud rate = 1000, 4 bits per symbol (16 levels):
  Bit rate = 1000 x 4 = 4000 bps

Bit rate = baud rate ONLY when 1 bit per symbol.
Bit rate and baud rate are only equal when each signal encodes one bit. Modern systems (QAM) encode many bits per symbol, so bit rate exceeds baud rate. Know the formula.

4.9.4 Protocols and the TCP/IP Stack

Protocol: A set of rules governing communication between devices. Protocols define message format, timing, error handling, and addressing so that different devices from different manufacturers can communicate.

The TCP/IP Protocol Stack

A layered model for network communication. Each layer provides services to the layer above and uses services from the layer below. Layering enables modularity — layers can be updated independently.

2Internet (Network)Handles logical addressing (IP addresses) and routing packets across networks.IP, ICMP, ARPPacket

Benefits of Layering

  • Modularity — each layer has a defined interface; internal changes do not affect other layers
  • Interoperability — devices from different manufacturers can communicate via standard protocols
  • Easier development — developers focus on one layer without understanding all others
  • Easier troubleshooting — problems can be isolated to a specific layer

Packet Structure

Each packet contains a header, payload (data), and trailer:

  • Header: source IP, destination IP, packet number (sequence), TTL (Time to Live), protocol, total packet count
  • Payload: the actual data being transmitted (a portion of the original message)
  • Trailer: error-checking data (checksum/CRC) to detect corruption during transmission

At the destination, packets are reassembled in the correct order using sequence numbers. Missing or corrupt packets are requested again (TCP retransmission).

4.9.6 The Internet

IP Addressing

An IP address uniquely identifies a device on a network. IPv4: 32-bit address (4 octets, e.g., 192.168.1.1) — approximately 4.3 billion addresses (now exhausted). IPv6: 128-bit address (8 groups of 4 hex digits, e.g., 2001:0db8:85a3::8a2e:0370:7334) — virtually unlimited addresses.

DNS (Domain Name System)

DNS translates human-readable domain names to IP addresses. It is a hierarchical, distributed system:

  1. Browser checks its local cache
  2. OS checks its DNS cache
  3. Query sent to recursive DNS resolver (usually ISP's)
  4. Resolver queries root DNS server → TLD server (.com, .uk) → authoritative server for the domain
  5. IP address returned and cached at each level

Routers

A router forwards packets between different networks using IP addresses. It maintains a routing table mapping destination networks to next-hop addresses. Routers operate at the Internet (Network) layer of TCP/IP.

Firewalls

A firewall monitors and controls network traffic based on security rules. It acts as a barrier between trusted internal networks and untrusted external networks (the internet).
  • Packet filtering — examines packet headers (source/destination IP, port, protocol) against rules
  • Stateful inspection — tracks the state of active connections; only allows packets belonging to established sessions
  • Application-level gateway (proxy) — inspects the actual data content (deep packet inspection)

4.9.7 Encryption

Encryption: Converting plaintext into ciphertext using an algorithm and a key, so that only authorised parties (with the correct key) can read the data.
FeatureSymmetric EncryptionAsymmetric Encryption
KeysSame key for encryption and decryptionPair of keys: public key (encrypt) and private key (decrypt)
SpeedFast (simpler maths)Slow (complex maths, large keys)
Key distributionProblem: how to securely share the key?No problem: public key is openly shared
ExamplesAES, DES, 3DESRSA, Diffie-Hellman, ECC
UseBulk data encryption, file encryption, VPNsKey exchange, digital signatures, HTTPS handshake

In practice, both are used together: Asymmetric encryption securely exchanges a symmetric session key (during the TLS handshake), then symmetric encryption handles the fast bulk data transfer.

Digital Signatures and Certificates

  • Digital signature: The sender hashes the message and encrypts the hash with their private key. The receiver decrypts with the sender's public key and compares hashes. This verifies authenticity (it came from the claimed sender) and integrity (the message was not altered).
  • Digital certificate: Issued by a Certificate Authority (CA). Contains the server's public key, the domain name, the CA's digital signature, and an expiry date. Browsers use certificates to verify that a website is who it claims to be (essential for HTTPS).
Know the difference between symmetric (one key, fast, key distribution problem) and asymmetric (key pair, slow, no distribution problem). TLS/HTTPS uses both: asymmetric for key exchange, symmetric for data. Digital signatures use the sender's private key (not public). Digital certificates are issued by Certificate Authorities to verify identity.

Know the difference between a hub (broadcasts to all), a switch (sends only to correct port), and a router (routes between networks). Hubs = collisions. Switches = no collisions. Routers = different networks. This is a very common exam topic.

4.9.10 Network Topologies — Extended (Ring and Mesh)

Ring Topology

Each device is connected to exactly two other devices, forming a closed loop. Data travels in one direction (or both in dual-ring) around the ring until it reaches its destination.

AdvantagesDisadvantages
No collisions — token passing controls access (token ring)One broken cable or device can break the entire ring
Equal access for all devices; no central point of failureDifficult to add/remove devices without disrupting network
Predictable performance under heavy loadTroubleshooting is complex
Cheap to install initiallyLargely obsolete; replaced by switched star networks

Mesh Topology

Each device connects directly to multiple (or all) other devices. In a full mesh, every node connects to every other node. A partial mesh has redundant links but not complete connectivity.

AdvantagesDisadvantages
Highly resilient — data can reroute if a link failsVery expensive: n(n-1)/2 links for n nodes (4 nodes = 6 links; 10 nodes = 45 links)
No single point of failureComplex to set up and manage
High bandwidth: multiple simultaneous pathsLarge amounts of cabling (or wireless channels)
Used in the internet backbone and WANsNot practical for typical LANs

Subnet Masks

A subnet mask divides an IP address into a network portion and a host portion. Devices with the same network portion are on the same subnet and can communicate directly without a router. 
IP Address:   192.168.1.45    = 11000000.10101000.00000001.00101101
Subnet Mask:  255.255.255.0   = 11111111.11111111.11111111.00000000

Network:      192.168.1.0      (first 24 bits = /24 notation)
Host range:   192.168.1.1  to  192.168.1.254
Broadcast:    192.168.1.255

/24 means 24 bits for network, 8 bits for hosts = 256 addresses (254 usable)

NAT (Network Address Translation)

NAT: Allows many devices on a private network (using private IP addresses: 192.168.x.x, 10.x.x.x, 172.16-31.x.x) to share a single public IP address when communicating with the internet. The router maintains a translation table mapping private IP:port to the public IP:port.
  • Conserves IPv4 addresses — one household uses one public IP for all devices
  • Security benefit — private addresses are hidden from the internet; direct inbound connections are blocked by default
  • Disadvantage — breaks true end-to-end connectivity; complicates hosting servers or P2P applications from behind NAT

Private vs Public IP Addresses

RangeClassUse
10.0.0.0 — 10.255.255.255ALarge private networks (enterprises)
172.16.0.0 — 172.31.255.255BMedium private networks
192.168.0.0 — 192.168.255.255CHome/small office networks
127.0.0.1—Loopback (localhost) — tests local stack
Private IP addresses (192.168.x.x, 10.x.x.x) are not routable on the internet — NAT converts them. IPv6 makes NAT unnecessary. Subnet masks identify which part of the IP is the network and which is the host — know how to determine the network address by applying a bitwise AND with the subnet mask.

Bluetooth

Bluetooth: A short-range wireless technology (IEEE 802.15.1) for Personal Area Networks (PANs). Uses 2.4 GHz ISM band with frequency hopping to reduce interference. Range typically 1-100 metres depending on class.
  • Bluetooth Classic — continuous data streams (audio, file transfer): headphones, speakers
  • Bluetooth Low Energy (BLE) — short bursts, very low power: fitness trackers, smart home devices, beacons
  • Pairing: devices exchange keys during pairing process to encrypt subsequent communication
  • Piconet: one master device controlling up to 7 active slave devices
  • Security concerns: Bluejacking (unsolicited messages), Bluesnarfing (unauthorized data access), Bluebugging (full device control)

VPN (Virtual Private Network): Establishes an encrypted, authenticated tunnel over the internet between a client and a VPN server. The client's real IP is hidden; all traffic appears to originate from the VPN server. Uses protocols such as OpenVPN, WireGuard, IPSec/L2TP, or PPTP.

Common Wireless Attacks

  • Evil twin attack — attacker sets up a rogue WAP with the same SSID as a legitimate network; victims connect unknowingly, traffic is intercepted
  • Packet sniffing — capturing unencrypted wireless packets; countered by WPA2/3 and HTTPS
  • Deauthentication attack — flooding a device with fake deauth frames forces it to disconnect and reconnect, capturing the WPA handshake for offline cracking
  • Wardriving — driving around to find unsecured or poorly secured wireless networks
Always recommend WPA2 or WPA3 for wireless security — never WEP. Know that MAC filtering and SSID hiding are weak measures easily bypassed (MAC spoofing, passive scanning). For complete security, combine strong encryption (WPA3), complex passwords, firewall, and VPN on public networks. Know what a VPN does: encrypts all traffic and hides the user's IP address.

4.9.14 Routing

Routing Tables

A router maintains a routing table — a database of destination networks and how to reach them. When a packet arrives, the router looks up the destination IP in its table and forwards the packet to the next hop.

Example routing table:
Destination Network | Subnet Mask     | Next Hop       | Interface
--------------------|-----------------|----------------|----------
192.168.1.0         | 255.255.255.0   | Direct         | eth0
10.0.0.0            | 255.0.0.0       | 192.168.1.1    | eth0
0.0.0.0             | 0.0.0.0         | 203.0.113.1    | eth1   <- default gateway

The 0.0.0.0/0 entry is the default route (gateway of last resort)
used when no specific match is found.

Routing Algorithms

  • Distance vector (e.g., RIP) — each router tells its neighbours its distance to all known networks. Simple but slow to converge; susceptible to routing loops.
  • Link state (e.g., OSPF) — each router maps the entire network topology. Faster convergence; more efficient; used in most modern networks.
  • BGP (Border Gateway Protocol) — the protocol used between autonomous systems on the internet backbone; policy-based routing.

How Packet Switching Works End-to-End

  1. Source device breaks message into packets; each packet gets a header with source/destination IP, sequence number, TTL
  2. Packets forwarded by routers — each router makes an independent forwarding decision based on its routing table
  3. Packets may travel different paths and arrive out of order
  4. TTL (Time to Live) field decremented at each hop — packet discarded when TTL reaches 0 (prevents infinite loops)
  5. Destination reassembles packets in correct order using sequence numbers (TCP handles retransmission of lost packets)
The routing table question is common: understand that routers use the longest prefix match (most specific route wins). Know what TTL does — prevents packets looping forever. Understand that the default gateway is the router a device sends packets to when the destination is on a different network.

4.9.15 How a Web Request Works — End-to-End

This ties together all the networking concepts in a realistic scenario: what happens when you type https://www.bbc.co.uk into a browser?

  1. DNS resolution: Browser checks local cache → OS cache → queries ISP recursive resolver → resolver queries root DNS → TLD DNS (.uk) → authoritative DNS for bbc.co.uk. Returns IP address (e.g., 151.101.0.81). Cached for TTL period.
  2. TCP connection: Browser opens a TCP connection to the server IP on port 443 (HTTPS). Three-way handshake: SYN → SYN-ACK → ACK.
  3. TLS handshake: Browser and server negotiate encryption. Server sends digital certificate (CA-verified, containing public key). Browser verifies certificate against trusted CAs. Asymmetric encryption used to exchange a symmetric session key. All subsequent data encrypted symmetrically (e.g., AES-256).
  4. HTTP request: Browser sends GET /index.html HTTP/2 request over the encrypted channel.
  5. Server response: Web server sends back HTTP 200 OK with the HTML content.
  6. Rendering: Browser parses HTML, discovers CSS/JS/image references, makes further HTTP requests for each resource.
  7. TCP teardown: When done, connection closed gracefully: FIN → ACK → FIN → ACK.
This end-to-end scenario is often used in exam questions to test understanding of multiple topics together (DNS, TCP, TLS, HTTP). Practice explaining each step and naming the relevant protocol and layer. Be precise: "DNS resolves the domain name to an IP address" not just "finds the address".
