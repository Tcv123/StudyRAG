4.9.1 Communication Methods

Serial vs Parallel Transmission

FeatureSerialParallel
DescriptionBits are sent one at a time along a single wire/channelMultiple bits are sent simultaneously along multiple wires/channels
Speed (short distance)SlowerFaster (multiple bits per cycle)
Speed (long distance)Faster (no skew)Slower (skew issues)
CostCheaper (fewer wires)More expensive (more wires)
SkewNo skew (single channel)Bits can arrive at different times (skew), causing errors at longer distances
CrosstalkMinimalElectromagnetic interference between adjacent wires (crosstalk)

Skew: In parallel transmission, the phenomenon where bits sent simultaneously arrive at the destination at slightly different times due to differences in wire lengths or electrical properties, potentially causing data corruption.
Crosstalk: Electromagnetic interference between adjacent parallel wires that can corrupt signals.

Synchronous vs Asynchronous Transmission

FeatureSynchronousAsynchronous
TimingData is sent in a continuous stream synchronised by a shared clock signalData is sent one character at a time without a shared clock; uses start and stop bits
ClockBoth sender and receiver share a clock signalNo shared clock — each character is framed with start/stop bits
EfficiencyMore efficient for large data transfers (less overhead)Less efficient (2+ extra bits per character for start/stop)
SpeedGenerally fasterGenerally slower
Use casesVideo streaming, VoIP, high-speed network linksKeyboard input, RS-232 serial ports, modems

Bit Rate and Baud Rate

Bit rate: The number of bits transmitted per second (bps).
Baud rate: The number of signal changes (symbols) per second.
Bandwidth: The range of frequencies available for data transmission; a higher bandwidth allows a higher data rate.
Latency: The time delay between sending data and it arriving at the destination.

If each signal change encodes more than one bit (e.g., using multiple voltage levels or phase shifts):

Bit rate = Baud rate x Number of bits per signal change

For example, if the baud rate is 1000 and each signal encodes 4 bits: bit rate = 1000 x 4 = 4000 bps.

4.9.2 Network Topologies

Topology: The arrangement of devices (nodes) and connections (links) in a network. Physical topology describes the physical layout; logical topology describes how data flows.
TopologyDescriptionAdvantagesDisadvantages
Star All devices connect to a central switch or hub
  • If one cable fails, only that device is affected
  • Easy to add new devices
  • Easy to troubleshoot
  • High performance (dedicated connection)
  • Central switch is a single point of failure
  • Requires more cabling than bus
  • Central switch can be expensive
Bus All devices connect to a single backbone cable with terminators at each end
  • Cheap and easy to set up
  • Requires less cabling than star
  • Good for small, temporary networks
  • If the backbone fails, the entire network goes down
  • Performance degrades with more devices (collisions)
  • Difficult to troubleshoot
  • Limited cable length and number of devices
  • Security risk (all data visible to all devices)
Mesh Full mesh: every device connects to every other device. Partial mesh: some devices have redundant connections.
  • Highly redundant — no single point of failure
  • Data can take multiple routes
  • Fault tolerant and reliable
  • Very expensive (lots of cabling and ports)
  • Complex to set up and manage
  • Full mesh is impractical for large networks
Hybrid A combination of two or more topologies (e.g., star-bus, star-mesh)
  • Flexible and scalable
  • Can be tailored to specific needs
  • Complex to design and manage
  • Can be expensive

4.9.5 Network Hardware

DeviceFunction
NIC (Network Interface Card/Controller)Hardware component (built-in or add-on) that enables a device to connect to a network. Each NIC has a unique MAC address (48-bit, hardware-level identifier). Can be wired (Ethernet) or wireless (Wi-Fi).
RouterForwards data packets between different networks (e.g., between a LAN and the internet). Uses routing tables to determine the best path for packets. Operates at Layer 3 (Network) of the OSI model using IP addresses.
SwitchConnects devices within a LAN. Learns the MAC addresses of connected devices and forwards frames only to the intended recipient (not to all ports). Operates at Layer 2 (Data Link). More efficient than a hub.
HubA simple device that broadcasts incoming data to all connected ports. No intelligence — does not learn MAC addresses. Causes unnecessary traffic and collisions. Largely obsolete, replaced by switches.
WAP (Wireless Access Point)Allows wireless devices to connect to a wired network. Broadcasts an SSID (network name). Uses Wi-Fi standards (802.11ac, 802.11ax).
GatewayConnects networks that use different protocols and translates between them. Acts as an entry/exit point for a network. A home router often functions as a gateway.
FirewallMonitors and controls incoming and outgoing network traffic based on predefined security rules. Can be hardware-based (standalone device) or software-based (running on a server/host). Inspects packet headers and can perform deep packet inspection.
Know the difference between a switch and a hub. A switch is intelligent (sends data only to the destination device using MAC addresses), while a hub blindly broadcasts to all ports. Also remember that a router operates at the network layer using IP addresses, while a switch operates at the data link layer using MAC addresses.

4.9.6 The Internet

The Internet: A global network of interconnected networks that communicate using the TCP/IP protocol suite. It is a physical infrastructure of cables, routers, and servers.
The World Wide Web (WWW): A collection of web pages and resources accessed via the internet using HTTP/HTTPS. The web is a service that runs on the internet — they are not the same thing.

Internet Structure

  • The internet is a hierarchical network of networks connected by high-capacity backbone links (fibre optic cables, including undersea cables)
  • Internet Exchange Points (IXPs) are physical locations where different networks peer and exchange traffic
  • Internet Service Providers (ISPs) provide end users with access to the internet. ISPs connect to larger ISPs and backbone providers in a tiered hierarchy (Tier 1, Tier 2, Tier 3).
  • DNS servers translate domain names to IP addresses
  • Web servers host websites and respond to HTTP requests

4.9.7 TCP/IP Protocol Stack

Protocol: A set of rules that govern how data is formatted, transmitted, and received across a network. Protocols ensure devices from different manufacturers can communicate.

The TCP/IP model has four layers. Data passes down through the layers on the sender's side (encapsulation) and up through the layers on the receiver's side (de-encapsulation).

LayerNameFunctionProtocols
4 (Top) Application Provides network services directly to user applications. Encodes data in the format required by the application protocol. HTTP, HTTPS, FTP, SMTP, IMAP, POP3, SSH, DNS, DHCP
3 Transport Provides end-to-end communication between applications. Segments data, provides error checking, flow control, and (with TCP) reliable delivery. Uses port numbers to identify applications. TCP, UDP
2 Internet (Network) Handles logical addressing (IP addresses) and routing. Packets are routed across networks towards the destination. IP (IPv4, IPv6), ARP, ICMP
1 (Bottom) Link (Network Interface) Handles physical transmission of data frames over the network medium (Ethernet, Wi-Fi). Uses MAC addresses for local delivery. Ethernet, Wi-Fi (802.11), PPP

TCP vs UDP

FeatureTCP (Transmission Control Protocol)UDP (User Datagram Protocol)
ConnectionConnection-oriented (three-way handshake: SYN, SYN-ACK, ACK)Connectionless (no handshake)
ReliabilityReliable — guarantees delivery, ordering, and error checking. Retransmits lost packets.Unreliable — no guarantee of delivery, ordering, or retransmission
SpeedSlower (overhead from handshake, acknowledgements)Faster (minimal overhead)
Flow controlYes (windowing mechanism)No
Use casesWeb browsing (HTTP), email (SMTP), file transfer (FTP)Live streaming, VoIP, online gaming, DNS queries

Public vs Private IP Addresses

  • Public IP addresses are globally unique and routable on the internet. Assigned by ISPs.
  • Private IP addresses are used within local networks and are not routable on the internet. Multiple LANs can use the same private ranges. Standard ranges:
    • 10.0.0.0 – 10.255.255.255 (Class A)
    • 172.16.0.0 – 172.31.255.255 (Class B)
    • 192.168.0.0 – 192.168.255.255 (Class C)

Static vs Dynamic IP Addresses

FeatureStaticDynamic
AssignmentManually configured; does not changeAutomatically assigned by DHCP; may change over time
Use caseServers, printers, network devices that need a consistent addressClient devices (laptops, phones, desktops)
ManagementMore admin effortLess admin effort

NAT (Network Address Translation)

NAT: A technique used by routers to translate private IP addresses on a local network to a single public IP address (and vice versa) when communicating with the internet. This allows multiple devices to share one public IP and helps conserve the limited IPv4 address space.

The router maintains a NAT table that maps internal private IP:port combinations to the external public IP:port, so responses can be routed back to the correct device.

Subnet Masking

A subnet mask divides an IP address into a network part and a host part. It determines which portion of the address identifies the network and which identifies the specific device.

IP Address:   192.168.1.100
Subnet Mask:  255.255.255.0

Network Part: 192.168.1     (where 255 appears in the mask)
Host Part:    .100           (where 0 appears in the mask)

All devices with IPs 192.168.1.x are on the same subnet.

CIDR notation: 192.168.1.0/24 means the first 24 bits are the network part (equivalent to subnet mask 255.255.255.0).

Subnetting allows a large network to be divided into smaller, more manageable sub-networks, improving security, performance, and organisation.

4.9.11 Firewalls, Proxy Servers, and Encryption

Firewalls

A firewall filters network traffic based on a set of security rules. It can:

  • Packet filtering: Inspect packet headers (source/destination IP, port numbers, protocol) and allow or deny based on rules
  • Stateful inspection: Track the state of active connections and make decisions based on the context of traffic
  • Application-layer filtering: Inspect the content of packets at the application level (deep packet inspection)
  • Block specific ports, IP addresses, or protocols
  • Prevent unauthorised inbound and outbound connections

Proxy Servers

Proxy Server: An intermediary server that sits between client devices and the internet. Requests from clients go to the proxy, which forwards them to the destination server on the client's behalf.

Functions:

  • Anonymity: Hides the client's IP address from the destination server
  • Caching: Stores frequently accessed web content locally, reducing bandwidth and improving speed
  • Content filtering: Blocks access to specific websites or content categories (used by schools, businesses)
  • Logging: Records all web activity for monitoring and auditing
  • Security: Can filter malicious content before it reaches the internal network

Encryption — SSL/TLS

SSL (Secure Sockets Layer) / TLS (Transport Layer Security): Cryptographic protocols that provide secure, encrypted communication over a network. TLS is the successor to SSL and is used by HTTPS.

How TLS works (simplified handshake):

  1. Client sends a "Client Hello" message with supported TLS versions and cipher suites
  2. Server responds with "Server Hello", its digital certificate (containing its public key), and chosen cipher suite
  3. Client verifies the server's certificate against a trusted Certificate Authority (CA)
  4. Client generates a session key (symmetric key), encrypts it with the server's public key, and sends it
  5. Server decrypts the session key with its private key
  6. Both parties now share the symmetric session key and use it to encrypt all subsequent communication

This uses asymmetric encryption (public/private keys) for the key exchange, then symmetric encryption (shared session key) for the actual data transfer — symmetric is much faster for bulk data.

4.9.12 Socket Programming Concepts

Socket: An endpoint for communication between two devices over a network. A socket is identified by an IP address combined with a port number (e.g., 192.168.1.5:8080).

A socket allows applications on different devices to send and receive data. The combination of IP address and port number uniquely identifies a specific application on a specific device.

Key Concepts

  • Port number: A 16-bit number (0–65535) that identifies a specific application or service. Well-known ports: 80 (HTTP), 443 (HTTPS), 25 (SMTP), 22 (SSH).
  • Server socket: Listens on a specific port, waiting for incoming connections.
  • Client socket: Initiates a connection to a server's IP address and port.
  • Connection process (TCP): The server binds to a port and listens. The client connects. Data is exchanged. The connection is closed.
# Python socket example (server)
import socket

server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)  # TCP socket
server.bind(('0.0.0.0', 8080))    # Bind to all interfaces, port 8080
server.listen(5)                    # Queue up to 5 connections

client_socket, address = server.accept()   # Wait for connection
data = client_socket.recv(1024)            # Receive up to 1024 bytes
client_socket.send(b'Hello from server')   # Send response
client_socket.close()                      # Close connection
# Python socket example (client)
import socket

client = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
client.connect(('192.168.1.5', 8080))      # Connect to server
client.send(b'Hello from client')           # Send data
response = client.recv(1024)                # Receive response
client.close()                              # Close connection

4.9.14 Packet Structure

When data is transmitted across a network, it is broken into packets. Each packet contains:

ComponentContentsPurpose
Header
  • Source IP address
  • Destination IP address
  • Packet sequence number
  • Protocol identifier
  • TTL (Time to Live)
  • Source and destination port numbers
  • Header checksum
  • Packet length
  • Flags (e.g., fragmentation)
 Provides all the information needed to route the packet to its destination and reassemble the original data correctly.
Payload (Data) A portion of the actual data being transmitted The useful content of the packet. The size is limited by the Maximum Transmission Unit (MTU) — typically 1500 bytes for Ethernet.
Trailer
  • Error detection code (CRC — Cyclic Redundancy Check)
  • End-of-packet marker
 Allows the receiver to detect transmission errors. If the CRC does not match the expected value, the packet is discarded and retransmission is requested (by TCP).

Key Fields Explained

TTL (Time to Live): A counter that is decremented by 1 each time the packet passes through a router. When TTL reaches 0, the packet is discarded. This prevents packets from circling the network indefinitely if there is a routing loop. Typical initial TTL values are 64 or 128.

Checksum: A calculated value based on the packet's contents. The sender computes the checksum and includes it in the header. The receiver recalculates the checksum and compares it to the received value. If they do not match, the packet has been corrupted during transmission.

Sequence number: Packets may arrive out of order because they can take different routes. The sequence number allows the receiving device to reassemble packets in the correct order.

When describing how data is transmitted, explain the full process: data is split into packets, each packet has a header (with source/destination IP, sequence number, TTL), payload, and trailer (with checksum). Packets may take different routes. At the destination, packets are reassembled in sequence order, checksums are verified, and missing or corrupted packets are re-requested (using TCP).
