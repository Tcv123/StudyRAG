4.9.1 Communication Methods

Serial vs Parallel Transmission

FeatureSerialParallel
How it worksBits sent one at a time along a single channelMultiple bits sent simultaneously along multiple wires
Speed (short)SlowerFaster (multiple bits per cycle)
Speed (long)Faster (no skew)Slower (skew limits speed)
CostCheaper (fewer wires)More expensive

ExamplesUSB, SATA, EthernetInternal CPU-to-memory bus
Skew: In parallel transmission, bits arrive at different times due to wire length/impedance differences, corrupting data.
Crosstalk: EM interference between adjacent wires where one wire's signal induces current in another.

Synchronous vs Asynchronous Transmission

FeatureSynchronousAsynchronous
TimingContinuous stream, shared clock signalOne byte at a time, framed with start bit (0) and stop bit(s) (1)
ClockShared clock (embedded or separate)No shared clock; start bit triggers synchronisation
EfficiencyHigher (minimal framing overhead)Lower (~20% overhead for start/stop bits)
Use casesNetwork links, video streaming, VoIPKeyboard, RS-232, UART, Bluetooth

Bit Rate, Baud Rate, Bandwidth, Latency

Bit rate: Bits transmitted per second (bps) — actual data throughput.
Baud rate: Signal changes (symbols) per second. Each symbol can encode multiple bits.
Bandwidth: Range of frequencies available. Higher bandwidth = higher data rate.
Latency: Time delay between sending and receiving. Affected by distance, hops, congestion. 
Bit rate = Baud rate x bits per symbol

Example: baud rate = 1000, 4 bits per symbol (16 levels):
  Bit rate = 1000 x 4 = 4000 bps

Bit rate = baud rate ONLY when 1 bit per symbol.
Bit rate and baud rate are only equal when each signal encodes one bit. Modern systems (QAM) encode many bits per symbol, so bit rate exceeds baud rate. Know the formula.

4.9.4 Protocols and the TCP/IP Stack

Protocol: A set of rules governing communication between devices. Protocols define message format, timing, error handling, and addressing so that different devices from different manufacturers can communicate.

The TCP/IP Protocol Stack

A layered model for network communication. Each layer provides services to the layer above and uses services from the layer below. Layering enables modularity — layers can be updated independently.

Benefits of Layering

  • Modularity — each layer has a defined interface; internal changes do not affect other layers
  • Interoperability — devices from different manufacturers can communicate via standard protocols
  • Easier development — developers focus on one layer without understanding all others
  • Easier troubleshooting — problems can be isolated to a specific layer

Packet Structure

Each packet contains a header, payload (data), and trailer:

  • Header: source IP, destination IP, packet number (sequence), TTL (Time to Live), protocol, total packet count
  • Payload: the actual data being transmitted (a portion of the original message)
  • Trailer: error-checking data (checksum/CRC) to detect corruption during transmission

At the destination, packets are reassembled in the correct order using sequence numbers. Missing or corrupt packets are requested again (TCP retransmission).

4.9.6 The Internet

IP Addressing

An IP address uniquely identifies a device on a network. IPv4: 32-bit address (4 octets, e.g., 192.168.1.1) — approximately 4.3 billion addresses (now exhausted). IPv6: 128-bit address (8 groups of 4 hex digits, e.g., 2001:0db8:85a3::8a2e:0370:7334) — virtually unlimited addresses.

DNS (Domain Name System)

DNS translates human-readable domain names to IP addresses. It is a hierarchical, distributed system:

  1. Browser checks its local cache
  2. OS checks its DNS cache
  3. Query sent to recursive DNS resolver (usually ISP's)
  4. Resolver queries root DNS server → TLD server (.com, .uk) → authoritative server for the domain
  5. IP address returned and cached at each level

Routers

A router forwards packets between different networks using IP addresses. It maintains a routing table mapping destination networks to next-hop addresses. Routers operate at the Internet (Network) layer of TCP/IP.

Firewalls

A firewall monitors and controls network traffic based on security rules. It acts as a barrier between trusted internal networks and untrusted external networks (the internet).
  • Packet filtering — examines packet headers (source/destination IP, port, protocol) against rules
  • Stateful inspection — tracks the state of active connections; only allows packets belonging to established sessions
  • Application-level gateway (proxy) — inspects the actual data content (deep packet inspection)

4.9.7 Encryption

Encryption: Converting plaintext into ciphertext using an algorithm and a key, so that only authorised parties (with the correct key) can read the data.
FeatureSymmetric EncryptionAsymmetric Encryption
KeysSame key for encryption and decryptionPair of keys: public key (encrypt) and private key (decrypt)
SpeedFast (simpler maths)Slow (complex maths, large keys)
Key distributionProblem: how to securely share the key?No problem: public key is openly shared
ExamplesAES, DES, 3DESRSA, Diffie-Hellman, ECC
UseBulk data encryption, file encryption, VPNsKey exchange, digital signatures, HTTPS handshake

In practice, both are used together: Asymmetric encryption securely exchanges a symmetric session key (during the TLS handshake), then symmetric encryption handles the fast bulk data transfer.

Digital Signatures and Certificates

  • Digital signature: The sender hashes the message and encrypts the hash with their private key. The receiver decrypts with the sender's public key and compares hashes. This verifies authenticity (it came from the claimed sender) and integrity (the message was not altered).
  • Digital certificate: Issued by a Certificate Authority (CA). Contains the server's public key, the domain name, the CA's digital signature, and an expiry date. Browsers use certificates to verify that a website is who it claims to be (essential for HTTPS).
Know the difference between symmetric (one key, fast, key distribution problem) and asymmetric (key pair, slow, no distribution problem). TLS/HTTPS uses both: asymmetric for key exchange, symmetric for data. Digital signatures use the sender's private key (not public). Digital certificates are issued by Certificate Authorities to verify identity.

4.9.8 Network Security Threats

ThreatDescriptionPrevention
MalwareMalicious software: viruses, worms, trojans, ransomware, spyware, adwareAnti-malware, regular updates, user education, sandboxing
PhishingFraudulent communications (email, SMS) impersonating trusted entities to steal credentialsUser training, email filtering, 2FA, checking URLs
SQL injectionInserting malicious SQL into input fields to access/modify databasesParameterised queries, input validation, least privilege
DDoSFlooding a server with traffic from many sources to make it unavailableRate limiting, traffic filtering, CDNs, ISP-level mitigation
Man-in-the-middleAttacker intercepts communication between two partiesHTTPS/TLS, certificate pinning, VPNs
Brute forceSystematically trying all possible passwordsAccount lockout, CAPTCHA, strong passwords, 2FA
Social engineeringManipulating people into revealing informationStaff training, security policies, verification procedures

Prevention Measures

  • Firewalls — filter traffic based on rules
  • Encryption — protect data in transit and at rest
  • Authentication — passwords, biometrics, multi-factor (something you know + have + are)
  • Access control — principle of least privilege; role-based access
  • Regular updates/patches — fix known vulnerabilities
  • Penetration testing — authorised simulated attacks to find weaknesses
  • User education — training staff to recognise phishing and social engineering
For security questions, always pair threats with specific countermeasures. Know SQL injection well: '; DROP TABLE users; -- and explain why parameterised queries prevent it. For encryption, know how TLS secures HTTPS using both symmetric and asymmetric methods.
