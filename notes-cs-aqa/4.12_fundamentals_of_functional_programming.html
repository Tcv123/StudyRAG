4.12.1 The Functional Programming Paradigm

Functional programming is a paradigm where computation is treated as the evaluation of mathematical functions. Programs are constructed by applying and composing functions, rather than executing statements that change state.

Key principles that distinguish functional from imperative programming:

  • Functions are first-class objects — can be assigned to variables, passed as arguments, returned from functions
  • Programs avoid side effects — functions do not modify external state
  • Data is immutable — once created, cannot be changed. "Modification" creates new values.
  • Functions are pure — same inputs always produce same output, nothing else affected
  • Recursion replaces iteration (loops require mutable counters)
  • Computation through function application and composition rather than assignment and loops

Functional vs Imperative

AspectFunctionalImperative (Procedural / OOP)
StateNo mutable state; data immutableMutable variables; state changes throughout
Control flowRecursion and function compositionLoops, conditional branches, sequences
Side effectsAvoided; functions are pureCommon (modifying globals, I/O)
FunctionsFirst-class objects; higher-order functions centralFunctions exist but not always first-class
Thinking style"What to compute" (declarative)"How to compute it" (imperative)
ExamplesHaskell, Lisp, Erlang, ML, ClojureC, Java, Python (multi-paradigm)
-- Imperative (Python) - summing a list:
total = 0              # mutable variable
for x in [1,2,3,4]:   # loop
    total = total + x  # state mutation
print(total)           # 10

-- Functional (Haskell) - summing a list:
mySum []     = 0                    -- base case
mySum (x:xs) = x + mySum xs         -- recursion, no state mutation
-- mySum [1,2,3,4] = 1 + 2 + 3 + 4 + 0 = 10

Advantages of functional programming:

  • Easier to reason about — no hidden state changes; function behaviour depends only on inputs
  • Easier to test — pure functions are deterministic; test by checking input-output pairs
  • Parallelism — immutable data requires no locks or synchronisation, making concurrent execution safe
  • Fewer bugs — no unintended side effects, no shared mutable state
  • Referential transparency — expressions can be replaced with their values without changing program behaviour

4.12.2 First-Class Objects

Functions are first-class objects (first-class citizens) when they have the same status as any other data value. Specifically, a function can be:
  • Assigned to a variable or stored in a data structure
  • Passed as an argument to another function
  • Returned as the result of another function
  • Created dynamically at runtime (anonymous/lambda functions)
-- Haskell examples:

-- Assign a function to a name
double = \x -> x * 2           -- lambda assigned to "double"

-- Pass a function as an argument
applyTwice f x = f (f x)
applyTwice double 3            -- double(double(3)) = double(6) = 12

-- Return a function from a function
makeAdder n = \x -> x + n      -- returns a NEW function
add5 = makeAdder 5             -- add5 is now a function that adds 5
add5 3                         -- 8
# Python equivalents:

# Assign function to variable
double = lambda x: x * 2

# Pass function as argument
def apply_twice(f, x):
    return f(f(x))
apply_twice(double, 3)         # 12

# Return function from function
def make_adder(n):
    return lambda x: x + n
add5 = make_adder(5)
add5(3)                        # 8

# Store functions in a data structure
operations = [lambda x: x + 1, lambda x: x * 2, lambda x: x ** 2]
for op in operations:
    print(op(5))               # 6, 10, 25

4.12.3 Higher-Order Functions

A higher-order function is a function that either: (1) takes one or more functions as arguments, or (2) returns a function as its result, or (3) both. Higher-order functions enable powerful abstractions over common patterns.

Map

map applies a function to every element of a list, returning a new list of results. The original list is unchanged (immutability). 
-- Haskell:
map (*2) [1, 2, 3, 4, 5]        -- [2, 4, 6, 8, 10]
map (^2) [1, 2, 3, 4]           -- [1, 4, 9, 16]
map toUpper "hello"              -- "HELLO"

-- Python:
list(map(lambda x: x * 2, [1, 2, 3, 4, 5]))   # [2, 4, 6, 8, 10]
list(map(str.upper, ["hello", "world"]))         # ["HELLO", "WORLD"]

-- The pattern: map f [a, b, c] = [f(a), f(b), f(c)]

Filter

filter takes a predicate function (returns True/False) and a list, returning a new list containing only the elements for which the predicate returns True. 
-- Haskell:
filter even [1, 2, 3, 4, 5, 6]       -- [2, 4, 6]
filter (>3) [1, 2, 3, 4, 5]          -- [4, 5]
filter (/= ' ') "h e l l o"          -- "hello"

-- Python:
list(filter(lambda x: x % 2 == 0, [1,2,3,4,5,6]))  # [2, 4, 6]
list(filter(lambda x: x > 3, [1,2,3,4,5]))          # [4, 5]

-- The pattern: filter p [a, b, c] keeps elements where p(element) is True

Fold (Reduce)

fold (called reduce in Python) combines all elements of a list into a single value by repeatedly applying a binary function. It takes: a combining function, an initial accumulator value, and a list. 
-- Haskell:
-- foldl f acc [x1, x2, x3] = f (f (f acc x1) x2) x3

foldl (+) 0 [1, 2, 3, 4, 5]        -- 15   (0+1+2+3+4+5)
foldl (*) 1 [1, 2, 3, 4, 5]        -- 120  (1*1*2*3*4*5 = 5!)
foldl max 0 [3, 1, 4, 1, 5]        -- 5    (maximum)

-- Trace of foldl (+) 0 [1,2,3]:
-- foldl (+) 0 [1,2,3]
-- = foldl (+) (0+1) [2,3]
-- = foldl (+) 1 [2,3]
-- = foldl (+) (1+2) [3]
-- = foldl (+) 3 [3]
-- = foldl (+) (3+3) []
-- = 6

-- Python:
from functools import reduce
reduce(lambda acc, x: acc + x, [1,2,3,4,5], 0)   # 15
reduce(lambda acc, x: acc * x, [1,2,3,4,5], 1)   # 120

# Implementing common operations with fold:
# Sum:     fold (+) 0 list
# Product: fold (*) 1 list
# Length:  fold (\acc _ -> acc + 1) 0 list
# Reverse: fold (\acc x -> x : acc) [] list

Combining Higher-Order Functions

-- Sum of squares of even numbers from 1 to 10:
-- Haskell:
foldl (+) 0 (map (^2) (filter even [1..10]))
-- filter even [1..10] = [2, 4, 6, 8, 10]
-- map (^2) [2,4,6,8,10] = [4, 16, 36, 64, 100]
-- foldl (+) 0 [4,16,36,64,100] = 220

-- Python:
from functools import reduce
nums = range(1, 11)
evens = filter(lambda x: x % 2 == 0, nums)
squares = map(lambda x: x ** 2, evens)
total = reduce(lambda acc, x: acc + x, squares, 0)  # 220
Know map, filter, and fold/reduce precisely. Map transforms every element. Filter selects elements matching a condition. Fold/reduce combines all elements into one value. Be able to trace through each step showing intermediate results. Remember: map and filter return lists; fold returns a single value.

4.12.4 Function Application

Function application is the process of applying a function to its argument(s) to produce a result. In functional programming, this is the fundamental operation — all computation is expressed as function application.

In Haskell, function application is written by juxtaposition (placing the function next to its argument, separated by a space):

-- Haskell function application (no parentheses needed):
double 5           -- applies double to 5, result: 10
add 3 4            -- applies add to 3 and 4, result: 7
map double [1,2,3] -- applies map to double and [1,2,3]

-- Function application is left-associative:
f x y = (f x) y   -- f is first applied to x, then the result to y

-- Function application has the highest precedence:
double 3 + 1 = (double 3) + 1 = 6 + 1 = 7   (NOT double 4)

4.12.5 Function Composition

Function composition combines two functions to create a new function. The output of one becomes the input of the other. Written as f . g (read "f after g"), meaning (f . g)(x) = f(g(x)). 
-- Haskell:
double x = x * 2
addOne x = x + 1

-- Compose: first double, then addOne
doubleAndAdd = addOne . double
doubleAndAdd 5    -- addOne(double(5)) = addOne(10) = 11

-- Compose: first addOne, then double
addAndDouble = double . addOne
addAndDouble 5    -- double(addOne(5)) = double(6) = 12

-- Longer composition chains:
process = negate . (*2) . (+1)
process 3         -- negate((*2)((+1) 3)) = negate((*2) 4) = negate 8 = -8

-- Python:
def compose(f, g):
    return lambda x: f(g(x))

double_and_add = compose(lambda x: x + 1, lambda x: x * 2)
double_and_add(5)    # 11

Key properties:

  • Composition is associative: (f . g) . h = f . (g . h)
  • The identity function (id x = x) is the identity element: f . id = id . f = f
  • Composition reads right to left: f . g . h applies h first, then g, then f

4.12.6 Partial Application and Currying

Partial Application

Partial application creates a new function by fixing (supplying) some but not all of the arguments to an existing function. The result is a function that takes the remaining arguments. 
-- Haskell:
add x y = x + y          -- takes two arguments

add5 = add 5             -- partially applied: fixes x=5
add5 3                   -- 8   (5 + 3)
add5 10                  -- 15  (5 + 10)

multiply x y = x * y
double = multiply 2      -- partially applied: fixes x=2
triple = multiply 3      -- partially applied: fixes x=3

double 7                 -- 14
triple 7                 -- 21

-- Practical use with map:
map (add 10) [1, 2, 3]     -- [11, 12, 13]
map (multiply 3) [1, 2, 3] -- [3, 6, 9]
filter (> 5) [1, 3, 5, 7]  -- [7]
# Python:
from functools import partial

def add(x, y):
    return x + y

add5 = partial(add, 5)    # fix x=5
add5(3)                    # 8

# Or using closures:
def make_adder(x):
    def adder(y):
        return x + y
    return adder

add5 = make_adder(5)
add5(3)                    # 8

Currying

Currying transforms a function that takes multiple arguments into a chain of functions, each taking a single argument. A function f(x, y, z) becomes f(x)(y)(z). In Haskell, all functions are automatically curried. 
-- In Haskell, add x y is actually (add x) y:
-- add :: Int -> Int -> Int
-- This means: add takes an Int, returns a FUNCTION (Int -> Int)
-- add 5 returns a function that adds 5 to its argument

-- The type signature reveals the currying:
add :: Int -> (Int -> Int)    -- takes Int, returns function
-- which is the same as:
add :: Int -> Int -> Int      -- Haskell convention (right-associative)

-- This is WHY partial application works in Haskell:
-- add 5 is not an error; it returns the "add 5 to" function
# Python - manual currying:
def add_curried(x):
    def inner(y):
        return x + y
    return inner

add_curried(3)(4)              # 7
add5 = add_curried(5)          # partial application via currying
add5(10)                       # 15

# Three-argument curried function:
def volume(l):
    def width(w):
        def height(h):
            return l * w * h
        return height
    return width

volume(2)(3)(4)                # 24
area_with_length_2 = volume(2) # partial application

Currying vs Partial Application:

  • Currying is a transformation of a function's structure (multi-arg to chain of single-arg)
  • Partial application is the result of supplying fewer arguments than needed
  • Currying enables partial application. In Haskell, because all functions are curried, partial application is automatic.
Partial application fixes some arguments to create a new, simpler function. Currying transforms a multi-argument function into nested single-argument functions. In Haskell, all functions are curried automatically, which is why add 5 works (it returns a function). Be able to show the type signature: add :: Int -> Int -> Int means add :: Int -> (Int -> Int).

4.12.7 Lists in Functional Programming

In functional programming, lists are the primary data structure. A list is either empty ([]) or consists of a head (first element) and a tail (the rest of the list). Lists are immutable — operations return new lists.

List Operations

-- Haskell list operations:
head [1, 2, 3]          -- 1       (first element)
tail [1, 2, 3]          -- [2, 3]  (everything except first)
length [1, 2, 3]        -- 3
null []                 -- True    (is the list empty?)
null [1]                -- False

-- Cons operator (:) prepends an element:
1 : [2, 3]              -- [1, 2, 3]
1 : 2 : 3 : []          -- [1, 2, 3]   (list construction)

-- Concatenation (++):
[1, 2] ++ [3, 4]        -- [1, 2, 3, 4]

-- List comprehension (like set-builder notation):
[x * 2 | x <- [1..5]]              -- [2, 4, 6, 8, 10]
[x | x <- [1..20], x `mod` 3 == 0] -- [3, 6, 9, 12, 15, 18]
[(x, y) | x <- [1..3], y <- [1..3], x /= y]  -- all pairs where x/=y

Recursive List Processing

-- Haskell: length using recursion
myLength []     = 0                      -- base case: empty list
myLength (x:xs) = 1 + myLength xs        -- recursive case

-- Trace: myLength [4, 7, 2]
-- = 1 + myLength [7, 2]
-- = 1 + 1 + myLength [2]
-- = 1 + 1 + 1 + myLength []
-- = 1 + 1 + 1 + 0 = 3

-- Haskell: sum using recursion
mySum []     = 0
mySum (x:xs) = x + mySum xs

-- Haskell: map using recursion
myMap f []     = []
myMap f (x:xs) = f x : myMap f xs

-- Haskell: filter using recursion
myFilter p []     = []
myFilter p (x:xs)
  | p x       = x : myFilter p xs       -- keep x
  | otherwise  = myFilter p xs           -- discard x
# Python equivalents:
def my_length(lst):
    if lst == []:
        return 0
    return 1 + my_length(lst[1:])

def my_sum(lst):
    if lst == []:
        return 0
    return lst[0] + my_sum(lst[1:])

def my_map(f, lst):
    if lst == []:
        return []
    return [f(lst[0])] + my_map(f, lst[1:])

def my_filter(p, lst):
    if lst == []:
        return []
    if p(lst[0]):
        return [lst[0]] + my_filter(p, lst[1:])
    return my_filter(p, lst[1:])
For AQA, know how to write recursive list functions in both Haskell and Python. The pattern is always: base case for empty list [], recursive case splitting into head (first element) and tail (rest). Understand that (x:xs) in Haskell destructures a list into head x and tail xs. Be able to trace through recursive calls step by step, showing how the list shrinks toward the base case.

4.12.8 Closures

A closure is a function together with its referencing environment — the variables from the enclosing scope that the function captures and retains access to, even after the enclosing function has returned. Closures enable partial application and data encapsulation. 
# Python closure example:
def make_counter(start):
    count = start                    # captured by the closure
    def counter():
        nonlocal count
        count += 1
        return count
    return counter                   # returns function + captured 'count'

c = make_counter(0)
print(c())    # 1
print(c())    # 2
print(c())    # 3
# 'count' persists between calls even though make_counter has returned

# Another example: creating multiplier functions
def make_multiplier(factor):
    return lambda x: x * factor      # 'factor' captured by closure

double = make_multiplier(2)
triple = make_multiplier(3)
print(double(5))   # 10
print(triple(5))   # 15
-- Haskell closure:
makeAdder n = \x -> x + n    -- n is captured from enclosing scope
add5 = makeAdder 5            -- closure captures n=5
add5 3                        -- 8

Key properties of closures:

  • The inner function captures variables from the enclosing scope
  • These variables persist as long as the closure exists, even after the enclosing function returns
  • Each call to the enclosing function creates a separate closure with its own captured variables
  • Closures are the mechanism behind partial application and currying
A closure is a function bundled with its environment (captured variables). The key exam point: the captured variables persist after the enclosing function returns. Be able to trace through a closure example showing which variables are captured and how they retain their values across multiple calls to the closure.
