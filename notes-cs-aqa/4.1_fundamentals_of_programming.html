4.1.1 Programming Paradigms

Programming paradigm: A fundamental style or approach to programming that defines how problems are structured, how data flows, and how computations are expressed.

Procedural Programming

A paradigm where a program is structured as a sequence of instructions (procedures/subroutines) that execute one after another. The program follows a top-down design and makes use of:

  • Sequence — instructions executed in order
  • Selection — branching with if/else/switch
  • Iteration — loops (for, while, repeat)
  • Subroutines — reusable blocks of code (functions and procedures)

Examples: C, Pascal, early BASIC.

Object-Oriented Programming (OOP)

Organises code around objects that combine data (attributes) and behaviour (methods). The key principles are encapsulation, inheritance, polymorphism, and abstraction (covered in detail below).

Examples: Java, C++, Python, C#.

Functional Programming

Based on mathematical functions. Key features:

  • First-class functions — functions can be passed as arguments, returned from other functions, and assigned to variables.
  • Higher-order functions — functions that take other functions as parameters or return functions (e.g., map, filter, reduce).
  • Immutability — once a value is assigned, it cannot be changed. New values are created instead of modifying existing ones.
  • No side effects — a function's output depends only on its inputs; it does not modify external state.
  • Recursion is favoured over iteration.
  • Function composition — combining simple functions to build more complex ones.
  • Partial application / currying — creating new functions by fixing some parameters of an existing function.

Examples: Haskell, Erlang, Lisp. Python and JavaScript support functional features.

Declarative Programming

The programmer specifies what the result should be, not how to achieve it. The language runtime determines the execution strategy.

  • SQL — you describe the data you want: SELECT name FROM students WHERE grade > 80
  • HTML/CSS — you describe the structure and appearance of content
  • Prolog — logic programming, stating facts and rules
You may be asked to compare paradigms. Focus on how each approaches problem-solving: procedural = step-by-step instructions, OOP = modelling real-world entities, functional = mathematical transformations, declarative = describing desired outcomes.

4.1.2 Object-Oriented Programming

Classes and Objects

Class: A blueprint or template that defines the attributes (data) and methods (behaviour) that objects of that type will have.
Object: An instance of a class — a specific realisation with its own attribute values. 
class Dog:
    def __init__(self, name, breed):
        self.__name = name       # private attribute
        self.__breed = breed     # private attribute

    def bark(self):
        return f"{self.__name} says Woof!"

    def get_name(self):          # getter method
        return self.__name

rex = Dog("Rex", "Labrador")     # rex is an object (instance)
print(rex.bark())                # Rex says Woof!

Encapsulation

Encapsulation is the bundling of data and methods within a class, and restricting direct access to some of the object's components. Attributes are typically made private and accessed through public getter/setter methods. This:

  • Protects data from accidental modification
  • Hides internal implementation details
  • Allows validation in setter methods

Inheritance

Inheritance: A mechanism where a new class (subclass / child class) derives attributes and methods from an existing class (superclass / parent class), promoting code reuse. 
class Animal:
    def __init__(self, name):
        self._name = name

    def speak(self):
        return "..."

class Cat(Animal):                # Cat inherits from Animal
    def speak(self):              # overriding the parent method
        return f"{self._name} says Meow!"

class Dog(Animal):
    def speak(self):
        return f"{self._name} says Woof!"

Advantages of inheritance: reduces code duplication, models real-world relationships (IS-A), makes code easier to extend and maintain.

Polymorphism

Polymorphism: The ability for objects of different classes to respond to the same method call in different ways. "Many forms."

In the example above, both Cat and Dog have a speak() method. Calling speak() on any Animal reference produces the correct output for the actual object type. This is overriding polymorphism.

Overloading polymorphism is when multiple methods share the same name but have different parameter lists (supported in Java/C++, not natively in Python).

Abstraction

Abstraction means hiding unnecessary complexity and showing only the essential features of an object. An abstract class cannot be instantiated directly; it serves as a template for subclasses that must implement the abstract methods.

from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass         # subclasses MUST implement this

class Circle(Shape):
    def __init__(self, radius):
        self.__radius = radius

    def area(self):
        return 3.14159 * self.__radius ** 2

Pointers and References

A pointer stores the memory address of another variable. A reference is an alias for another variable. In languages like C/C++, pointers are used for dynamic memory allocation, linked data structures, and passing data efficiently.

Subroutines: Functions and Procedures

Procedure: A named block of code that performs a task but does not return a value.
Function: A named block of code that performs a task and returns a value.

Advantages of subroutines:

  • Code reuse — write once, call many times
  • Modularity — break complex problems into manageable parts
  • Easier testing and debugging of individual components
  • Improved readability and maintainability
  • Supports team development — different programmers work on different subroutines
FUNCTION calculateArea(length, width)
    RETURN length * width
ENDFUNCTION

PROCEDURE greetUser(name)
    OUTPUT "Hello, " + name
ENDPROCEDURE

4.1.5 Recursion

Recursion: A programming technique where a subroutine calls itself. Every recursive solution must have a base case (stopping condition) and a recursive case (where the function calls itself with a simpler input).

How Recursion Works — The Call Stack

Each recursive call is pushed onto the call stack. When the base case is reached, the stack unwinds as each call returns its result to the caller. If there is no base case or the recursion is too deep, a stack overflow error occurs.

Example: Factorial

FUNCTION factorial(n)
    IF n = 0 THEN          # base case
        RETURN 1
    ELSE                    # recursive case
        RETURN n * factorial(n - 1)
    ENDIF
ENDFUNCTION

# Call stack for factorial(4):
# factorial(4) waits for 4 * factorial(3)
#   factorial(3) waits for 3 * factorial(2)
#     factorial(2) waits for 2 * factorial(1)
#       factorial(1) waits for 1 * factorial(0)
#         factorial(0) returns 1        ← base case
#       factorial(1) returns 1 * 1 = 1
#     factorial(2) returns 2 * 1 = 2
#   factorial(3) returns 3 * 2 = 6
# factorial(4) returns 4 * 6 = 24

Example: Fibonacci

FUNCTION fibonacci(n)
    IF n <= 1 THEN         # base case
        RETURN n
    ELSE
        RETURN fibonacci(n - 1) + fibonacci(n - 2)
    ENDIF
ENDFUNCTION

Recursion vs Iteration

FeatureRecursionIteration
Code clarityOften more elegant / readable for inherently recursive problems (trees, fractals)Usually simpler for straightforward looping

Trace tables for recursion are common in exams. Show each call on the stack, the parameter values, and the return values as the stack unwinds. Always identify the base case and recursive case clearly.

4.1.6 Exception Handling

Exception: An unexpected event or error that disrupts normal program flow at runtime (e.g., division by zero, file not found, invalid input).

Exception handling allows a program to deal with errors gracefully instead of crashing.

try:
    number = int(input("Enter a number: "))
    result = 100 / number
    print(f"Result: {result}")
except ValueError:
    print("Invalid input - not a number")
except ZeroDivisionError:
    print("Cannot divide by zero")
finally:
    print("This always executes")
BlockPurpose
tryContains code that might raise an exception
except / catchHandles specific exception types when they occur
finallyCode that always executes, regardless of whether an exception occurred (used for cleanup, e.g., closing files)

Why use exception handling?

  • Prevents the program from crashing unexpectedly
  • Provides meaningful error messages to the user
  • Separates error-handling logic from normal logic
  • Ensures resources (files, connections) are properly released

4.1.7 Parameter Passing

By Value

A copy of the argument's value is passed to the subroutine. Any changes made inside the subroutine do not affect the original variable.

PROCEDURE doubleIt(x)     # x is a copy
    x ← x * 2
    OUTPUT x               # outputs 10
ENDPROCEDURE

num ← 5
doubleIt(num)
OUTPUT num                 # still outputs 5 (unchanged)

By Reference

The memory address of the argument is passed. The subroutine works directly on the original variable, so changes do affect the original.

PROCEDURE doubleIt(BYREF x)
    x ← x * 2
ENDPROCEDURE

num ← 5
doubleIt(num)
OUTPUT num                 # outputs 10 (changed)

4.1.8 Scope

Scope: The region of a program in which a variable is accessible.

Local Variables

  • Declared inside a subroutine
  • Only accessible within that subroutine
  • Created when the subroutine is called, destroyed when it ends
  • Cannot be accessed by other subroutines unless passed as parameters
  • Using local variables is considered good practice

Global Variables

  • Declared outside all subroutines (at the top level)
  • Accessible from anywhere in the program
  • Persist for the entire duration of the program
  • Generally discouraged because they make debugging harder, can be changed unexpectedly, and create dependencies between subroutines
globalCounter = 0          # global variable

def increment():
    global globalCounter   # must declare to modify global in Python
    localTemp = 1          # local variable
    globalCounter += localTemp

increment()
print(globalCounter)       # 1
# print(localTemp)         # ERROR - localTemp is not accessible here
In exam questions, identify which variables are local and global. Explain why local variables are preferred: they reduce side effects, improve modularity, and make subroutines self-contained and easier to test.

    for i in range(len(s) - 1, -1, -1):

    return result

4.1.10 File Handling

Programs need to read from and write to files for persistent storage — data that survives after the program ends.

Text File Operations

# Writing to a file
file = open("data.txt", "w")      # "w" = write (overwrites)
file.write("Line 1\n")
file.write("Line 2\n")
file.close()

# Appending to a file
file = open("data.txt", "a")      # "a" = append
file.write("Line 3\n")
file.close()

# Reading from a file
file = open("data.txt", "r")      # "r" = read
contents = file.read()            # read entire file
file.close()

# Reading line by line
file = open("data.txt", "r")
for line in file:
    print(line.strip())           # strip() removes \n
file.close()

Using with Statements

Best practice is to use with to automatically close the file, even if an exception occurs:

with open("data.txt", "r") as file:
    for line in file:
        print(line.strip())
# file is automatically closed here
ModeDescription
"r"Read — file must exist
"w"Write — creates file or overwrites existing
"a"Append — adds to end of file
"r+"Read and write

CSV Files

import csv

# Writing CSV
with open("students.csv", "w", newline="") as file:
    writer = csv.writer(file)
    writer.writerow(["Name", "Age", "Grade"])
    writer.writerow(["Alice", 17, "A"])
    writer.writerow(["Bob", 18, "B"])

# Reading CSV
with open("students.csv", "r") as file:
    reader = csv.reader(file)
    for row in reader:
        print(row)          # ['Name', 'Age', 'Grade'] etc.

4.1.11 Abstract Data Types

Abstract Data Type (ADT): A logical description of how data is viewed and the operations that can be performed on it, without specifying how it is implemented. It defines what operations are available, not how they work internally.

ADTs separate the interface (what operations exist) from the implementation (how the data is stored and manipulated). This is a form of abstraction and information hiding.

Common ADTs

ADTKey OperationsPossible Implementations
Stackpush, pop, peek, isEmptyArray, linked list
Queueenqueue, dequeue, peek, isEmptyArray (circular), linked list
Listinsert, delete, search, access by indexArray, linked list
Dictionary / Mapadd, remove, lookup by keyHash table, BST
Setadd, remove, union, intersection, differenceHash table, BST
Treeinsert, delete, traverseLinked nodes, array
GraphaddVertex, addEdge, traverseAdjacency matrix, adjacency list
Priority Queueinsert with priority, remove highest priorityHeap, sorted array

Benefits of ADTs:

  • Encapsulation — internal workings are hidden from the user
  • Flexibility — the implementation can be changed without affecting code that uses the ADT
  • Modularity — complex systems can be built from well-defined components
  • Testability — each ADT can be tested independently
You may be asked to distinguish between an ADT and a data structure. An ADT is a theoretical concept (e.g., "a stack supports push and pop"). A data structure is the concrete implementation (e.g., "a stack implemented using an array with a top pointer"). The same ADT can have multiple implementations.
