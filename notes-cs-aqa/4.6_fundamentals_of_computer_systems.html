4.6.1 Hardware and Software

Hardware is the physical components of a computer system (CPU, RAM, hard drive, keyboard, monitor). Software is the set of instructions (programs) that tell the hardware what to do.

The relationship between hardware and software:

  • Hardware is useless without software to control it.
  • Software cannot run without hardware to execute it on.
  • Software provides the interface between the user and the hardware.
  • The same hardware can run different software to perform completely different tasks — this is the key insight of the stored-program concept.

Types of Application Software

  • General-purpose software — can be used for many different tasks (e.g., spreadsheet, word processor).
  • Special-purpose software — designed for one specific task (e.g., payroll software, air traffic control).
  • Bespoke software — custom-built for a particular client's needs. Expensive but tailored exactly to requirements.

4.6.3 System Software

Operating System (OS)

An operating system is system software that manages hardware resources, provides services to application software, and acts as an interface between the user and the computer hardware.

Utility Programs

Utility programs perform maintenance and housekeeping tasks:

  • Disk defragmenter — reorganises fragmented files on a hard drive so they are stored in contiguous blocks, improving read/write speed.
  • File manager — allows users to browse, copy, move, rename, and delete files and directories.
  • Backup software — creates copies of data to protect against loss (full, incremental, differential).
  • Antivirus/anti-malware — scans for and removes malicious software.
  • Compression utilities — reduce file sizes for storage and transmission.
  • Encryption utilities — protect data by converting it to an unreadable form.

Libraries

A library is a collection of pre-compiled routines (functions/procedures) that programs can use. Libraries provide reusable code, saving development time and reducing errors. They are linked to programs at compile time (static linking) or at run time (dynamic linking).
  • Static libraries — code is copied into the executable at compile time. Results in a larger executable but no external dependencies.
  • Dynamic libraries (DLLs / shared objects) — loaded into memory at runtime and shared between programs. Saves memory and disk space; updates to the library benefit all programs that use it.

Translators

4.6.4 Role of the Operating System

Memory Management

  • Allocation and deallocation of memory to processes.
  • Virtual memory — uses a section of the hard drive as if it were RAM when physical RAM is full. Pages of data are swapped between RAM and disk. This allows programs larger than physical RAM to run, but is much slower than real RAM.
  • Paging — memory is divided into fixed-size blocks called pages. The OS maintains a page table mapping virtual addresses to physical addresses.
  • Segmentation — memory is divided into variable-size segments based on logical divisions (code, data, stack).
  • Memory protection — prevents one process from accessing memory allocated to another process.

Process Scheduling

Scheduling is the method by which the OS decides which process gets to use the CPU and for how long. In a multitasking system, the OS rapidly switches between processes to give the illusion of parallelism.

Scheduling algorithms:

AlgorithmDescriptionProsCons
First Come First Served (FCFS)Processes run in the order they arriveSimple, fairLong processes block short ones (convoy effect)
Shortest Job First (SJF)Shortest process runs nextMinimises average waiting timeRequires knowing process length; starvation of long processes
Round Robin (RR)Each process gets a fixed time slice (quantum), then goes to back of queueFair, no starvation, good response timeContext switching overhead; choosing quantum size is critical
Priority SchedulingEach process has a priority; highest priority runs firstImportant processes run firstLow priority processes may starve
Multilevel Feedback QueueMultiple queues with different priorities; processes can move between queuesAdaptive, balances responsiveness and throughputComplex to implement

Key concepts:

  • Pre-emptive scheduling — the OS can interrupt a running process and give the CPU to another (e.g., Round Robin, Priority).
  • Non-pre-emptive scheduling — a process runs until it finishes or voluntarily yields (e.g., FCFS, SJF).
  • Context switching — saving the state of the current process and loading the state of the next. This has a time overhead.
  • Starvation — a low-priority process never gets to run because higher-priority processes are always available.

I/O Management

  • The OS manages communication between the CPU and input/output devices via device drivers.
  • Buffering — data is temporarily held in a buffer to compensate for speed differences between devices (e.g., printer buffer).
  • Spooling — jobs are queued (e.g., print jobs in a print spooler) so the CPU doesn't have to wait for slow devices.
  • Interrupts — I/O devices signal the CPU when they need attention, allowing the CPU to do other work rather than polling.

File Management

  • The OS provides a file system that organises data into files and directories on storage devices.
  • It manages file naming, access control, storage allocation, and maintains a directory structure.
  • Access control — permissions determine who can read, write, or execute files (e.g., read-only, owner/group/other permissions).
  • File allocation — the OS tracks which disk blocks belong to which files (using methods like FAT, NTFS, ext4).

Security Management

  • User authentication — verifying identity via passwords, biometrics, or multi-factor authentication.
  • Access control — restricting what resources each user or process can access.
  • Encryption — protecting data at rest and in transit.
  • Firewall management — monitoring and controlling network traffic.
  • Audit logging — recording system events for security review.
When asked about the roles of an OS, structure your answer around these five areas: memory management, process scheduling, I/O management, file management, and security. Give specific examples for each.

4.6.5 Boolean Algebra

Boolean algebra is a mathematical system for manipulating logical expressions. Variables can only be TRUE (1) or FALSE (0). It is fundamental to digital circuit design and simplification.

Basic Operations

OperationNotationMeaning
ANDA . B or A ∧ B or ABTrue if both A and B are true
ORA + B or A ∨ BTrue if A or B (or both) are true
NOT¬A or A̅ or NOT AInverts the value

Laws of Boolean Algebra

LawAND formOR form
IdentityA . 1 = AA + 0 = A
Null (Annulment)A . 0 = 0A + 1 = 1
IdempotentA . A = AA + A = A
ComplementA . ¬A = 0A + ¬A = 1
Double Negation¬(¬A) = A
CommutativeA . B = B . AA + B = B + A
Associative(A . B) . C = A . (B . C)(A + B) + C = A + (B + C)
DistributiveA . (B + C) = A.B + A.CA + (B . C) = (A + B) . (A + C)
AbsorptionA . (A + B) = AA + (A . B) = A

De Morgan's Laws

De Morgan's Laws provide rules for complementing compound expressions:
¬(A . B) = ¬A + ¬B — "NOT (A AND B) = (NOT A) OR (NOT B)"
¬(A + B) = ¬A . ¬B — "NOT (A OR B) = (NOT A) AND (NOT B)"

In words: to complement a compound expression, complement each variable AND swap AND with OR (and vice versa).

Simplification Example

  Simplify: Q = A.B + A.B.C

  Factor out A.B:
  Q = A.B.(1 + C)        [Distributive law]
  Q = A.B.1              [A + 1 = 1, so 1 + C = 1]
  Q = A.B                [Identity law: A.1 = A]

  Simplify: Q = (A + B).(A + ¬B)

  Q = A.A + A.¬B + B.A + B.¬B    [Expand using distributive law]
  Q = A + A.¬B + A.B + 0         [A.A = A, B.¬B = 0]
  Q = A + A.(¬B + B)              [Factor out A]
  Q = A + A.1                      [¬B + B = 1]
  Q = A + A                        [A.1 = A]
  Q = A                            [A + A = A]
When simplifying, look for absorption (A + A.B = A), complementary pairs (A.¬A = 0), and common factors. De Morgan's laws are essential for converting between AND/OR forms and for simplifying NAND/NOR expressions.

Universal Gates

NAND and NOR are called universal gates because any other logic gate can be built using only NAND gates (or only NOR gates).

  NOT A  = A NAND A
  A AND B = (A NAND B) NAND (A NAND B)
  A OR B  = (A NAND A) NAND (B NAND B)

This is important in circuit design because manufacturing a chip with only one type of gate is simpler and cheaper.

4.6.7 Karnaugh Maps (K-Maps)

A Karnaugh map is a graphical method for simplifying Boolean expressions. It arranges truth table values in a grid where adjacent cells differ by exactly one variable (Gray code ordering), making it easy to identify groups that can be simplified.

2-Variable K-Map

B=0B=1
A=0m0m1
A=1m2m3

3-Variable K-Map

BC=00BC=01BC=11BC=10
A=0m0m1m3m2
A=1m4m5m7m6

4-Variable K-Map

CD=00CD=01CD=11CD=10
AB=00m0m1m3m2
AB=01m4m5m7m6
AB=11m12m13m15m14
AB=10m8m9m11m10

Note the Gray code ordering: 00, 01, 11, 10. Adjacent cells (including wrap-around from edges) differ by exactly one bit.

Grouping Rules

  1. Groups must contain 1s only (for Sum of Products / SOP).
  2. Group sizes must be powers of 2: 1, 2, 4, 8, 16.
  3. Groups must be rectangular (including squares).
  4. Groups can wrap around edges (top-bottom, left-right).
  5. Make groups as large as possible.
  6. Every 1 must be in at least one group.
  7. Groups can overlap (a cell can be in more than one group).
  8. Use the fewest groups possible.

Reading the Simplified Expression

For each group, identify the variables that remain constant across all cells in the group. Variables that change within the group are eliminated.

Worked Example

  4-variable K-map for F(A,B,C,D):

           CD=00  CD=01  CD=11  CD=10
  AB=00  |  0  |  1  |  1  |  0  |
  AB=01  |  0  |  1  |  1  |  0  |
  AB=11  |  0  |  1  |  1  |  0  |
  AB=10  |  0  |  0  |  0  |  0  |

  Group 1: All six 1s form a group of columns CD=01 and CD=11,
           rows AB=00, AB=01, AB=11 (but that's 6, not a power of 2).

  Better grouping:
  Group 1: 4 cells — AB=00/01, CD=01/11 → ¬A.D
  Wait, let's re-examine. Rows AB=00, AB=01 (A=0), cols CD=01, CD=11 (D=1):
  → ¬A.D

  Group 2: 4 cells — AB=01/11, CD=01/11 (B=1, D=1):
  → B.D

  F = ¬A.D + B.D = D.(¬A + B)

Don't Care Conditions

Don't care conditions (marked as X or d) represent input combinations that either cannot occur or whose output doesn't matter. When simplifying with K-maps, don't cares can be treated as either 0 or 1, whichever produces a simpler expression.

Using don't cares allows larger groups, which leads to simpler expressions and fewer logic gates in the circuit.

Always use Gray code ordering (00, 01, 11, 10) on K-map axes. Groups can wrap around edges. Include don't cares in groups if it makes the group larger. Practice 3-variable and 4-variable maps thoroughly.

4.6.8 Flip-Flops

A flip-flop is a sequential logic circuit that can store one bit of data. Unlike combinational circuits (whose outputs depend only on current inputs), flip-flops have memory — their output depends on both current inputs and previous state.

SR Flip-Flop (Set-Reset)

The SR flip-flop has two inputs: S (Set) and R (Reset), and two outputs: Q and ¬Q.

SRQ (next)Action
00Q (no change)Hold / memory state
010Reset
101Set
11InvalidNot allowed — both Q and ¬Q would be forced to 0, which is contradictory

An SR flip-flop can be constructed from two cross-coupled NAND or NOR gates.

D-Type Flip-Flop (Data)

The D flip-flop has one data input D and a clock input. On the clock edge (rising or falling), the output Q takes the value of D.

Clock EdgeDQ (next)Action
Rising edge00Stores 0
Rising edge11Stores 1
No edgeXQ (no change)Output held

Key properties of the D flip-flop:

  • Eliminates the invalid state of the SR flip-flop (since S and R are never both 1).
  • The output only changes on the clock edge, making it suitable for synchronous circuits.
  • Used in registers to store data — a group of D flip-flops forms a register (e.g., 8 D flip-flops = an 8-bit register).
  • Used in counters and shift registers.
Be able to trace through timing diagrams for both SR and D flip-flops. The D flip-flop is edge-triggered (the value of D matters only at the clock edge, not between edges).

4.6.9 Adders

Half Adder

A half adder adds two single-bit binary numbers and produces a sum (S) and a carry (C). It does NOT handle a carry input from a previous stage.

Boolean expressions:

  S = A XOR B    (A ⊕ B)
  C = A AND B    (A . B)

The half adder is built from one XOR gate and one AND gate.

Full Adder

A full adder adds two single-bit binary numbers plus a carry-in (Cin) from a previous stage. It produces a sum (S) and a carry-out (Cout).
ABCinSum (S)Cout
00000

Boolean expressions:

  S    = A XOR B XOR C_in
  C_out = (A AND B) OR (C_in AND (A XOR B))

A full adder can be built from two half adders and an OR gate:

  1. First half adder: adds A and B, producing intermediate sum (S1) and carry (C1).
  2. Second half adder: adds S1 and Cin, producing final sum (S) and carry (C2).
  3. OR gate: Cout = C1 OR C2.

Multi-Bit Adders

To add multi-bit binary numbers, full adders are chained together in a ripple carry adder:

  • The Cout of each full adder feeds into the Cin of the next (more significant) full adder.
  • The least significant bit position can use a half adder (or a full adder with Cin = 0).
  • An n-bit ripple carry adder uses n full adders.
  4-bit ripple carry adder:

  A3 B3   A2 B2   A1 B1   A0 B0
   |  |    |  |    |  |    |  |
  [FA3]---[FA2]---[FA1]---[FA0]--- C_in (0)
   |  \    |  \    |  \    |  \
  C_out S3     S2     S1     S0

Limitation: In a ripple carry adder, each full adder must wait for the carry from the previous stage, causing delay proportional to the number of bits. This is addressed by more advanced designs like carry-lookahead adders.

You should be able to draw the circuits for half and full adders, write their Boolean expressions, and explain how they chain together for multi-bit addition. Know the difference: a half adder has no carry input; a full adder does.
