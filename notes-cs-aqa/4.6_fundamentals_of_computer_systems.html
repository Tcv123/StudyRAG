4.6.1 Hardware and Software

Hardware is the physical components of a computer system (CPU, RAM, hard drive, keyboard, monitor, GPU). Software is the set of instructions (programs) that tell the hardware what to do.
  • Hardware is useless without software to control it; software cannot run without hardware to execute it
  • Software provides the interface between the user and the hardware
  • The same hardware can run different software for different tasks — the key insight of the stored-program concept
  • Hardware determines the capability ceiling; software determines what the hardware actually does

4.6.3 System Software

Operating System (OS)

An operating system manages hardware resources, provides services to application software, and acts as an intermediary between user and hardware. Loaded from secondary storage into RAM during boot (initiated by BIOS/UEFI firmware in ROM).

Libraries

A library is a collection of pre-compiled routines that programs can reuse, saving development time and reducing errors.
  • Static libraries — code copied into executable at compile time. Larger executable, but self-contained, no external dependencies.
  • Dynamic libraries (DLLs / shared objects) — loaded at runtime, shared between programs. Saves memory and disk; updates benefit all users. Risk of version conflicts ("DLL hell").

Stages of Compilation

  1. Lexical analysis — source code broken into tokens (keywords, identifiers, operators, literals). Whitespace and comments removed. Symbol table created.
  2. Syntax analysis (parsing) — tokens checked against grammar rules (BNF). Abstract Syntax Tree (AST) built. Syntax errors reported.
  3. Semantic analysis — checks for type mismatches, undeclared variables, incorrect arguments.
  4. Code generation — AST translated into machine code or intermediate code.
  5. Optimisation — code improved for speed or size (dead code removal, loop unrolling, register allocation).
Many modern languages use both: compile to intermediate bytecode, then interpret or JIT-compile at runtime. Java → bytecode → JVM; Python → bytecode → CPython interpreter; C# → CIL → .NET CLR. Know all five stages of compilation.

4.6.4 Role of the Operating System

Five major management functions, each operating continuously:

1. Memory Management

  • Allocation/deallocation of RAM to processes
  • Virtual memory — uses hard drive as overflow when RAM is full (swap/page file). Allows programs larger than physical RAM but much slower.
  • Paging — fixed-size blocks (pages, typically 4 KB). Page table maps virtual to physical addresses. Non-contiguous physical memory appears contiguous to each process.
  • Segmentation — variable-size segments based on logical program divisions (code, data, stack, heap).
  • Memory protection — prevents processes from accessing each other's memory.
  • Thrashing — excessive page swapping; system spends more time swapping than doing useful work.

2. Process Scheduling

Scheduling decides which process gets the CPU and for how long. In multitasking, the OS rapidly switches between processes to give the illusion of simultaneous execution.
AlgorithmDescriptionProsCons
FCFSProcesses run in arrival orderSimple, fair (FIFO)Convoy effect; poor for interactive
SJFShortest estimated time runs nextMinimises average waitRequires time estimates; starvation of long jobs
Round RobinFixed time slice (quantum), then back of queueFair, no starvation, good responseContext switch overhead; quantum size critical
PriorityHighest priority runs firstUrgent tasks prioritisedStarvation of low priority (fix: aging)
Multilevel FeedbackMultiple queues; processes move between themAdaptive, balances responsivenessComplex to implement

Key concepts:

  • Pre-emptive — OS can interrupt a running process (RR, Priority). Essential for responsive multitasking.
  • Non-pre-emptive — process runs until it finishes or yields (FCFS, SJF). A misbehaving process can monopolise the CPU.
  • Context switching — saving current process state (registers, PC, stack pointer) and loading the next. Has time overhead.
  • Starvation — low-priority process never runs because higher-priority processes are always available.

3. I/O Management

  • Device drivers — translate generic OS commands into device-specific instructions
  • Buffering — temporary data storage in RAM to compensate for speed differences between devices
  • Spooling — jobs queued on disk for slow devices (e.g., print spooler). CPU does not wait.
  • Interrupts — devices signal the CPU via control bus when needing attention, avoiding wasteful polling

4. File Management

  • File system — organises data into files and directories (FAT, NTFS, ext4, APFS)
  • Access control — permissions determine who can read, write, or execute files
  • Storage allocation — tracks which disk blocks belong to which files

5. Security Management

  • User authentication — passwords, biometrics, multi-factor
  • Access control — principle of least privilege
  • Encryption — data at rest and in transit
  • Firewall management — monitoring network traffic
  • Audit logging — recording events for security review
Structure OS role answers around these five areas: memory, scheduling, I/O, files, security. For scheduling, compare algorithms and explain which suits different scenarios (Round Robin for desktops; Priority for real-time systems; SJF minimises average waiting time).

Simplification Examples

Universal Gates

NAND and NOR are universal — any gate can be built from just NAND (or just NOR). This simplifies chip manufacturing.

From NAND only:
NOT A   = A NAND A
A AND B = (A NAND B) NAND (A NAND B)
A OR B  = (A NAND A) NAND (B NAND B)

From NOR only:
NOT A   = A NOR A
A OR B  = (A NOR B) NOR (A NOR B)
A AND B = (A NOR A) NOR (B NOR B)

Deriving Expressions from Truth Tables

Sum of Products (SOP): for each row where output=1, write an AND term (variable if 1, complement if 0). OR all terms together.

4.6.7 Karnaugh Maps (K-Maps)

A Karnaugh map simplifies Boolean expressions visually. Truth table values are arranged in a grid where adjacent cells differ by one variable (Gray code ordering), making it easy to identify groups of 1s to combine.

K-Map Layouts

3-variable (Gray code order: 00, 01, 11, 10):

BC=00BC=01BC=11BC=10
A=0m0m1m3m2
A=1m4m5m7m6

4-variable:

CD=00CD=01CD=11CD=10
AB=00m0m1m3m2
AB=01m4m5m7m6
AB=11m12m13m15m14
AB=10m8m9m11m10

Grouping Rules

  1. Groups contain 1s only
  2. Group sizes must be powers of 2 (1, 2, 4, 8, 16)
  3. Groups must be rectangular
  4. Groups should be as large as possible
  5. Groups can wrap around edges (the map is a torus)
  6. Every 1 must be in at least one group
  7. Overlapping groups are allowed (and often necessary)
  8. Use fewest groups possible

Worked Example

Boolean expression: Q = ¬A.¬B.C + ¬A.B.C + A.¬B.C + A.B.C

3-variable K-map:
        BC=00  BC=01  BC=11  BC=10
A=0  [  0      1      1      0   ]
A=1  [  0      1      1      0   ]

Group all four 1s (2x2 block in columns 01 and 11):
This eliminates A and B (they both change within the group).
Result: Q = C

Simplified from 4 terms with 3 variables to just C!

Another Example

4-variable K-map:
        CD=00  CD=01  CD=11  CD=10
AB=00 [  1      0      0      1   ]
AB=01 [  1      0      0      1   ]
AB=11 [  0      0      0      0   ]
AB=10 [  1      0      0      1   ]

Group the six 1s:
- All cells in CD=00 column (except AB=11): wrap rows 00,01,10 = group of 4 if we
  consider corners wrapping. Actually: column CD=00 rows 00,01,10 = 3 cells (not power of 2).
  Better: pair (00,00)+(01,00) = 2, pair (00,00)+(10,00) wrap = 2.
- Column CD=10 same pattern.
- Larger group: all 1s form an L-shape; cover with two groups of 4 wrapping edges.
  Group 1: CD=00 column rows 00,01,10 + CD=10 column row 00 (wrap) = 4 cells → ¬B.¬D
  Group 2: CD=10 column rows 00,01,10 + CD=00 column row 00 (wrap) = ¬B.¬C...

  Simpler: two groups of 4:
  Corners wrap: (00,00),(01,00),(00,10),(01,10) = ¬A.¬D (4 cells)
  And: (00,00),(10,00),(00,10),(10,10) = ¬B.¬D (4 cells)
  Result: Q = ¬A.¬D + ¬B.¬D = ¬D.(¬A + ¬B)
In K-map questions, remember Gray code ordering (00, 01, 11, 10 — NOT 00, 01, 10, 11). Groups can wrap around edges. A group of 2 eliminates 1 variable; a group of 4 eliminates 2 variables; a group of 8 eliminates 3. Always make groups as large as possible to maximise simplification.

Half Adder

Adds two single-bit inputs A and B. Produces a Sum and a Carry.

Adds three single-bit inputs: A, B, and Carry In (Cin). Produces Sum and Carry Out (Cout).

A full adder can be built from two half adders and an OR gate. Multiple full adders are chained together (the carry out of one feeds the carry in of the next) to create an n-bit ripple-carry adder.

D-type Flip-Flop

A D-type flip-flop is a sequential logic circuit that stores one bit of data. It has a data input (D) and a clock input. On the rising edge of the clock signal, the output Q takes the value of D and holds it until the next clock edge. This is the fundamental building block of registers and memory.

Know half adder (2 inputs, no carry in) vs full adder (3 inputs including carry in). Full adders are chained for multi-bit addition. The D-type flip-flop stores 1 bit on the clock edge — it is the basis of all registers and memory. Be able to draw and explain the truth tables for all three circuits.
