4.2.1 Arrays

Array: An ordered, fixed-size collection of elements of the same data type, stored in contiguous (adjacent) memory locations. Elements are accessed by their index in O(1) constant time because the memory address can be calculated directly: address = base_address + (index × element_size).

One-Dimensional (1D) Arrays

A simple list of elements accessed by a single index. Most languages use zero-based indexing (first element at index 0). Some languages (Pascal) and AQA pseudocode may use one-based indexing.

# Python
names = ["Alice", "Bob", "Charlie", "Diana", "Eve"]
# Index:    0        1        2         3       4

print(names[0])     # Alice (first element)
print(names[4])     # Eve (last element)
print(names[-1])    # Eve (Python shorthand for last element)
names[1] = "Ben"    # Modify element at index 1

# Traversing an array
for i in range(len(names)):
    print(f"Index {i}: {names[i]}")

# Pseudocode
names ← ["Alice", "Bob", "Charlie", "Diana", "Eve"]
FOR i ← 0 TO LEN(names) - 1
    OUTPUT names[i]
NEXT i

Two-Dimensional (2D) Arrays

A table or grid of elements accessed by two indices: row and column. Conceptually an "array of arrays". Useful for representing grids, matrices, tables, and game boards.

# 2D array (3 rows x 4 columns)
grid = [
    [1,  2,  3,  4],      # row 0
    [5,  6,  7,  8],      # row 1
    [9, 10, 11, 12]       # row 2
]

print(grid[1][2])          # 7 (row 1, column 2)
grid[0][3] = 99            # modify element at row 0, col 3

# Iterating through all elements
for row in range(3):
    for col in range(4):
        print(grid[row][col], end=" ")
    print()

# Pseudocode: summing all elements in a 2D array
total ← 0
FOR row ← 0 TO ROWS - 1
    FOR col ← 0 TO COLS - 1
        total ← total + grid[row][col]
    NEXT col
NEXT row

Multidimensional Arrays

DimensionAnalogyAccess PatternExample Use
1DA single row of boxesa[i]List of names, sensor readings
2DA spreadsheet / tablea[row][col]Chess board, pixel grid, timetable
3DA stack of spreadsheetsa[layer][row][col]RGB image (3 colour channels), 3D game world

Key properties of arrays:

  • Fixed size — declared at creation, cannot grow or shrink (in most languages; Python lists are dynamic)
  • Homogeneous — all elements must be the same type
  • Random access — any element accessed in O(1) time using base + index * size
  • Contiguous memory — elements stored next to each other in RAM, enabling efficient CPU cache utilisation
  • Insertion/deletion is expensive — inserting or deleting in the middle requires shifting elements: O(n)
In AQA pseudocode, arrays are declared as: myArray : ARRAY[0..9] OF INTEGER. This declares an array of 10 integers (indices 0 to 9). Be careful with the upper bound — [0..9] has 10 elements, not 9. For 2D arrays: grid : ARRAY[0..2, 0..3] OF INTEGER declares a 3x4 array.

4.2.2 Records

Record: A composite data type that groups together related data items (fields) which may be of different types. Each field has a name and a data type. Records are also called structs (C), data classes (Kotlin/Python), or named tuples. Fields are accessed using dot notation. 
# Pseudocode record definition
TYPE Student
    firstName : STRING
    lastName  : STRING
    age       : INTEGER
    grade     : REAL
    enrolled  : BOOLEAN
ENDTYPE

pupil : Student
pupil.firstName ← "Alice"
pupil.lastName  ← "Smith"
pupil.age       ← 17
pupil.grade     ← 88.5
pupil.enrolled  ← True

# Python equivalent
from dataclasses import dataclass

@dataclass
class Student:
    first_name: str
    last_name: str
    age: int
    grade: float
    enrolled: bool

pupil = Student("Alice", "Smith", 17, 88.5, True)
print(pupil.first_name)   # Alice

Records are often used as the elements of arrays to create a simple database-like structure:

# Array of records
students : ARRAY[0..29] OF Student

students[0].firstName ← "Alice"
students[0].age ← 17
students[1].firstName ← "Bob"
students[1].age ← 18

# Searching through an array of records
FOR i ← 0 TO 29
    IF students[i].grade >= 90 THEN
        OUTPUT students[i].firstName + " achieved an A*"
    ENDIF
NEXT i

Records vs Arrays: Arrays store multiple items of the same type (a column of data); records store a single entity with multiple fields of potentially different types (a row of data). An array of records combines both concepts to create a simple table.

4.2.3 Stacks

Stack: A Last-In, First-Out (LIFO) abstract data type. The last item added is the first to be removed. Think of a stack of plates — you can only add to or remove from the top. All operations occur at one end (the top).

Stack Operations

OperationDescriptionTimeError Condition
push(item)Add an item to the top of the stackO(1)Stack overflow (stack is full)
pop()Remove and return the item from the topO(1)Stack underflow (stack is empty)
peek() / top()View the top item without removing itO(1)Stack is empty
isEmpty()Check if the stack is emptyO(1)None
isFull()Check if the stack is full (array-based only)O(1)None
size()Return the number of items on the stackO(1)None

Array-Based Implementation

# Pseudocode
CLASS Stack
    PRIVATE items : ARRAY[0..MAX-1] OF DATA
    PRIVATE top : INTEGER ← -1

    PUBLIC PROCEDURE push(item)
        IF top = MAX - 1 THEN
            ERROR "Stack overflow"
        ENDIF
        top ← top + 1
        items[top] ← item
    ENDPROCEDURE

    PUBLIC FUNCTION pop()
        IF top = -1 THEN
            ERROR "Stack underflow"
        ENDIF
        item ← items[top]
        top ← top - 1
        RETURN item
    ENDFUNCTION

    PUBLIC FUNCTION peek()
        IF top = -1 THEN
            ERROR "Stack is empty"
        ENDIF
        RETURN items[top]
    ENDFUNCTION

    PUBLIC FUNCTION isEmpty()
        RETURN top = -1
    ENDFUNCTION
ENDCLASS
# Python implementation
class Stack:
    def __init__(self, max_size):
        self.stack = [None] * max_size   # pre-allocate array
        self.max_size = max_size
        self.top = -1                    # -1 indicates empty stack

    def push(self, item):
        if self.top == self.max_size - 1:
            raise Exception("Stack overflow - stack is full")
        self.top += 1                    # move top pointer up
        self.stack[self.top] = item      # place item at new top

    def pop(self):
        if self.top == -1:
            raise Exception("Stack underflow - stack is empty")
        item = self.stack[self.top]      # get item at top
        self.top -= 1                    # move top pointer down
        return item

    def peek(self):
        if self.top == -1:
            raise Exception("Stack is empty")
        return self.stack[self.top]

    def is_empty(self):
        return self.top == -1

    def is_full(self):
        return self.top == self.max_size - 1

    def size(self):
        return self.top + 1

Applications of Stacks

  • Function call stack — the system pushes a stack frame (return address + local variables) for each subroutine call; pops on return
  • Undo functionality — each action is pushed; Ctrl+Z pops and reverses the last action
  • Expression evaluation — evaluating postfix (Reverse Polish Notation) expressions
  • Bracket matching — push opening brackets, pop when closing bracket encountered and check they match
  • Backtracking algorithms — depth-first search, maze solving, puzzle solving
  • Browser back button — visited pages pushed onto stack; back pops the most recent page
  • Reversing data — push all items, then pop them out in reverse order
  • Infix to postfix conversion — the Shunting Yard algorithm uses an operator stack
# Bracket matching using a stack
def is_balanced(expression):
    stack = []
    matching = {')': '(', ']': '[', '}': '{'}
    for char in expression:
        if char in '([{':
            stack.append(char)         # push opening bracket
        elif char in ')]}':
            if len(stack) == 0:
                return False           # no matching opening bracket
            if stack.pop() != matching[char]:
                return False           # mismatched bracket types
    return len(stack) == 0             # empty = all matched

print(is_balanced("((2+3)*[4-1])"))    # True
print(is_balanced("((2+3)*[4-1)"))     # False - mismatch
When tracing stack operations, draw the stack vertically with the top at the top. Show the top pointer moving up for push and down for pop. Always check for overflow (push to full stack) and underflow (pop from empty stack). The top pointer = -1 means empty; top pointer = maxSize - 1 means full.

4.2.4 Queues

Queue: A First-In, First-Out (FIFO) abstract data type. The first item added is the first to be removed. Items are added at the rear and removed from the front.

Queue Operations

OperationDescriptionTime
enqueue(item)Add an item to the rear of the queueO(1)
dequeue()Remove and return the item from the frontO(1)
peek() / front()View the front item without removing itO(1)
isEmpty()Check if the queue is emptyO(1)
isFull()Check if the queue is fullO(1)

Linear Queue Problem

A simple array-based queue with front and rear pointers. The problem: as items are dequeued, space at the front is wasted and cannot be reused. Eventually the rear reaches the end even though free space exists at the front.

# After several enqueue and dequeue operations:
# [_, _, _, D, E, F, G, _]
#           ^front     ^rear
# Indices 0-2 are wasted even though empty!

Circular Queue

Solves the wasted space problem by treating the array as circular — when a pointer reaches the end, it wraps around to the beginning using modulo: next = (current + 1) % maxSize.

# Pseudocode
CLASS CircularQueue
    PRIVATE queue : ARRAY[0..MAX-1] OF DATA
    PRIVATE front : INTEGER ← 0
    PRIVATE rear : INTEGER ← -1
    PRIVATE size : INTEGER ← 0

    PUBLIC PROCEDURE enqueue(item)
        IF size = MAX THEN
            ERROR "Queue is full"
        ENDIF
        rear ← (rear + 1) MOD MAX     # wrap around
        queue[rear] ← item
        size ← size + 1
    ENDPROCEDURE

    PUBLIC FUNCTION dequeue()
        IF size = 0 THEN
            ERROR "Queue is empty"
        ENDIF
        item ← queue[front]
        front ← (front + 1) MOD MAX   # wrap around
        size ← size - 1
        RETURN item
    ENDFUNCTION
ENDCLASS
# Python implementation
class CircularQueue:
    def __init__(self, max_size):
        self.queue = [None] * max_size
        self.max_size = max_size
        self.front = 0
        self.rear = -1
        self.size = 0

    def enqueue(self, item):
        if self.size == self.max_size:
            raise Exception("Queue is full")
        self.rear = (self.rear + 1) % self.max_size
        self.queue[self.rear] = item
        self.size += 1

    def dequeue(self):
        if self.size == 0:
            raise Exception("Queue is empty")
        item = self.queue[self.front]
        self.front = (self.front + 1) % self.max_size
        self.size -= 1
        return item

    def peek(self):
        if self.size == 0:
            raise Exception("Queue is empty")
        return self.queue[self.front]

    def is_empty(self):
        return self.size == 0

Priority Queue

Each item has an associated priority. Items with higher priority are dequeued before items with lower priority, regardless of insertion order. If two items have equal priority, they follow FIFO order.

Implementation options:

  • Sorted array/list: O(n) insert (find correct position), O(1) dequeue (highest priority always at front)
  • Unsorted array/list: O(1) insert (add at end), O(n) dequeue (must search for highest priority)
  • Binary heap: O(log n) for both insert and dequeue (most efficient)

Applications of Queues

  • Print spooling — print jobs processed in order received (FIFO)
  • CPU scheduling — processes waiting in round-robin scheduling
  • Breadth-first search (BFS) — traversing graphs level by level
  • Keyboard buffer — storing keystrokes in order
  • Message queues — asynchronous communication between systems
  • Priority queue: hospital A&E triage, OS task scheduling, network packet prioritisation
Queue TypeAdvantageDisadvantage
LinearSimplest to implementWastes space as items dequeued; eventually runs out of space
CircularReuses space efficiently; no wasted memorySlightly more complex (modulo logic); still fixed size
PriorityHandles urgency-based orderingMore complex; insert or dequeue is O(n) unless using a heap
When tracing circular queue operations, always show the modulo calculation explicitly: (rear + 1) % maxSize. Draw the array and clearly label front, rear, and size after each operation. A common exam mistake is not wrapping front when dequeuing. Remember: size is tracked separately so you can distinguish full from empty (both would have front == (rear+1)%max without a size counter).

4.2.5 Linked Lists

Linked list: A dynamic data structure consisting of nodes, where each node contains a data field and a pointer (reference) to the next node in the sequence. The last node's pointer is null. The list is accessed through a head pointer that references the first node.

Types of Linked Lists

TypeDescriptionPointers per Node
Singly linkedEach node points to the next only; traverse forwards only1 (next)
Doubly linkedEach node points to both next and previous; traverse both ways2 (next, prev)
Circular linkedLast node points back to first (instead of null); forms a ring1 or 2

Linked List Operations

Inserting at the beginning (O(1)):

  1. Create a new node with the data
  2. Set new node's next pointer to the current head
  3. Update head to point to the new node

Inserting at a specific position (O(n) to find, O(1) to insert):

  1. Traverse to the node before the insertion point
  2. Set new node's next to the current node's next
  3. Set current node's next to the new node

Deleting a node (O(n) to find, O(1) to remove):

  1. Traverse to the node before the one to delete
  2. Set its next to the deleted node's next (bypassing it)
  3. Free the deleted node's memory
# Python implementation
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None

    def insert_at_start(self, data):
        new_node = Node(data)
        new_node.next = self.head    # point to current head
        self.head = new_node         # update head

    def insert_at_end(self, data):
        new_node = Node(data)
        if self.head is None:
            self.head = new_node
            return
        current = self.head
        while current.next is not None:
            current = current.next
        current.next = new_node

    def delete(self, target):
        if self.head is None:
            return
        if self.head.data == target:
            self.head = self.head.next
            return
        current = self.head
        while current.next is not None:
            if current.next.data == target:
                current.next = current.next.next
                return
            current = current.next

    def search(self, target):
        current = self.head
        index = 0
        while current is not None:
            if current.data == target:
                return index
            current = current.next
            index += 1
        return -1

    def display(self):
        current = self.head
        while current is not None:
            print(current.data, end=" -> ")
            current = current.next
        print("None")

# Usage
ll = LinkedList()
ll.insert_at_start("Charlie")
ll.insert_at_start("Bob")
ll.insert_at_start("Alice")
ll.display()                 # Alice -> Bob -> Charlie -> None
ll.delete("Bob")
ll.display()                 # Alice -> Charlie -> None

Static Array Implementation of a Linked List

When dynamic memory is unavailable, a linked list can be simulated using an array. Each element has data and a "pointer" (array index). A free list tracks available positions.

# Static linked list using an array
# Index:  0      1       2       3       4       5
# Data:  "Bob"  "Eve"  "Alice"  [free]  "Diana" [free]
# Next:   4      -1      0       5       1       -1
# Head pointer: 2   |   Free list pointer: 3

# Following the chain from head (index 2):
# Index 2 ("Alice") -next=0-> Index 0 ("Bob") -next=4->
# Index 4 ("Diana") -next=1-> Index 1 ("Eve") -next=-1 (end)
# Logical order: Alice -> Bob -> Diana -> Eve

When asked to implement a linked list statically (using an array), each array element has a data field and a pointer (index) field. You also need a head pointer and a free list pointer. Insertions take a node from the free list; deletions return a node to it. Be prepared to trace through operations showing both the array and the head/free pointers changing at each step. In diagrams, always draw the pointer arrows clearly.

4.2.6 Trees

Tree: A connected, acyclic (no cycles), hierarchical data structure consisting of nodes connected by edges. It has a single root node, and every other node has exactly one parent. A tree with n nodes has exactly n−1 edges.

Tree Terminology

TermDefinition
RootThe topmost node with no parent
ChildA node directly connected below another node
ParentA node directly connected above another node
LeafA node with no children (terminal node)
EdgeThe connection between a parent and child
SubtreeA node and all its descendants
Depth / LevelNumber of edges from the root to that node (root = depth 0)
HeightNumber of edges on the longest path from root to any leaf
SiblingNodes that share the same parent

Binary Search Trees (BST)

Binary Search Tree: A binary tree (each node has at most two children) with the ordering property: for every node, all values in the left subtree are less than the node's value, and all values in the right subtree are greater. This property must hold at every node, not just the root.

BST Search: Compare target with current node. If equal, found. If smaller, go left. If larger, go right.

# Searching for 12 in the BST above:
# Start at 15: 12 < 15, go LEFT
# At 8: 12 > 8, go RIGHT
# At 12: FOUND!

# Searching for 20:
# Start at 15: 20 > 15, go RIGHT
# At 23: 20 < 23, go LEFT
# At 18: 20 > 18, go RIGHT
# null → NOT FOUND

BST Insert: Follow the search path until a null pointer is reached, then place the new node there.

# Inserting 10:
# Start at 15: 10 < 15, go LEFT
# At 8: 10 > 8, go RIGHT
# At 12: 10 < 12, go LEFT
# null → Insert 10 as left child of 12

BST Delete: Three cases:

  1. Leaf node — simply remove it (set parent's pointer to null)
  2. One child — replace the node with its only child
  3. Two children — replace with the in-order successor (smallest value in right subtree) or in-order predecessor (largest in left subtree), then delete that successor node

Time complexity:

OperationAverage (balanced)Worst (degenerate)
SearchO(log n)O(n)
InsertO(log n)O(n)
DeleteO(log n)O(n)

The worst case occurs when the tree degenerates into a linked list (e.g., inserting 1, 2, 3, 4, 5 in order creates a chain going only right).

Tree Traversals

Three depth-first traversal methods, each visiting nodes in a different order. All use recursion:

TraversalOrderMnemonicResult for BST above
Pre-orderRoot, Left, RightVisit root before children15, 8, 4, 2, 12, 23, 18, 27
In-orderLeft, Root, RightVisit root between children2, 4, 8, 12, 15, 18, 23, 27
Post-orderLeft, Right, RootVisit root after children2, 4, 12, 8, 18, 27, 23, 15
# Pseudocode
PROCEDURE preOrder(node)
    IF node <> null THEN
        OUTPUT node.data       # VISIT root first
        preOrder(node.left)
        preOrder(node.right)
    ENDIF
ENDPROCEDURE

PROCEDURE inOrder(node)
    IF node <> null THEN
        inOrder(node.left)     # left subtree first
        OUTPUT node.data       # VISIT root in middle
        inOrder(node.right)
    ENDIF
ENDPROCEDURE

PROCEDURE postOrder(node)
    IF node <> null THEN
        postOrder(node.left)
        postOrder(node.right)  # right subtree
        OUTPUT node.data       # VISIT root last
    ENDIF
ENDPROCEDURE
# Python implementations
def pre_order(node):
    if node is not None:
        print(node.data, end=" ")
        pre_order(node.left)
        pre_order(node.right)

def in_order(node):
    if node is not None:
        in_order(node.left)
        print(node.data, end=" ")
        in_order(node.right)

def post_order(node):
    if node is not None:
        post_order(node.left)
        post_order(node.right)
        print(node.data, end=" ")

Critical fact: An in-order traversal of a BST produces the values in ascending sorted order.

Uses of each traversal:

  • Pre-order: Copying/cloning a tree, serialising a tree to a file, prefix expression evaluation
  • In-order: Outputting sorted data from a BST, converting BST to sorted array
  • Post-order: Safely deleting a tree (children before parent), evaluating postfix expression trees, calculating directory sizes
To trace a traversal, use the "hand-hugging" technique: draw a continuous line around the outside of the tree. For pre-order, record a node when passing its left side. For in-order, record when passing underneath. For post-order, record when passing its right side. This is faster and more reliable than tracing recursion. Always remember: in-order on a BST = sorted output.

4.2.7 Graphs

Graph: A data structure consisting of a set of vertices (nodes) connected by edges (arcs). Unlike trees, graphs can contain cycles and vertices can have any number of connections. Formally: G = (V, E) where V is the set of vertices and E is the set of edges.

Adjacency Matrix

A 2D array where matrix[i][j] represents the edge from vertex i to vertex j. For weighted graphs, the value is the weight; for unweighted, 1 (connected) or 0 (not). The matrix is symmetric for undirected graphs.

#     A    B    C    D
# A [ 0,   5,   3,   8 ]
# B [ 5,   0,   2,   7 ]
# C [ 3,   2,   0, inf ]
# D [ 8,   7, inf,   0 ]

# Unweighted version:
#     A  B  C  D
# A [ 0, 1, 1, 1 ]
# B [ 1, 0, 1, 1 ]
# C [ 1, 1, 0, 0 ]
# D [ 1, 1, 0, 0 ]

Adjacency List

Each vertex stores a list of its adjacent vertices (and weights if applicable). More memory-efficient for sparse graphs.

# Weighted adjacency list for graph above
graph = {
    'A': [('B', 5), ('C', 3), ('D', 8)],
    'B': [('A', 5), ('C', 2), ('D', 7)],
    'C': [('A', 3), ('B', 2)],
    'D': [('A', 8), ('B', 7)]
}

# Unweighted version
graph = {
    'A': ['B', 'C', 'D'],
    'B': ['A', 'C', 'D'],
    'C': ['A', 'B'],
    'D': ['A', 'B']
}

You must be able to: (1) draw both matrix and list representations from a graph diagram, (2) construct a graph from either representation, (3) decide which is more appropriate for a scenario. An adjacency matrix for an undirected graph is always symmetric. For a directed graph, it is NOT symmetric. A sparse graph with many vertices but few edges should use an adjacency list to save memory.

4.2.8 Hash Tables

Hash table: A data structure that maps keys to values using a hash function. The hash function converts a key into an array index, allowing near-constant-time O(1) average access for insertion, deletion, and lookup.

Hashing

A hash function takes a key and computes an integer index. A good hash function should:

  • Distribute keys uniformly across the table (minimise clustering)
  • Be deterministic — same key always produces same hash
  • Be fast to compute
  • Minimise collisions (different keys mapping to the same index)
# Simple hash function example
def hash_function(key, table_size):
    total = 0
    for char in str(key):
        total += ord(char)
    return total % table_size

# hash_function("Cat", 10) = (67+97+116) % 10 = 280 % 10 = 0
# hash_function("Dog", 10) = (68+111+103) % 10 = 282 % 10 = 2
# hash_function("Act", 10) = (65+99+116) % 10 = 280 % 10 = 0  COLLISION!

Collision Resolution

A collision occurs when two different keys hash to the same index. Collisions are inevitable (pigeonhole principle). Two main resolution strategies:

1. Open Addressing (Linear Probing)

  • If target slot is occupied, check the next slot, wrapping around if needed
  • Simple but causes clustering — groups of consecutive occupied slots
  • Variants: quadratic probing (1, 4, 9, 16... ahead), double hashing
# Linear probing: table size = 7
# Insert keys: A(hash=2), B(hash=2), C(hash=3), D(hash=2)

# Insert A (hash=2): slot 2 empty, place A
# [_, _, A, _, _, _, _]

# Insert B (hash=2): slot 2 taken, try 3 -> empty, place B
# [_, _, A, B, _, _, _]

# Insert C (hash=3): slot 3 taken, try 4 -> empty, place C
# [_, _, A, B, C, _, _]

# Insert D (hash=2): slots 2,3,4 taken, try 5 -> place D
# [_, _, A, B, C, D, _]   <-- clustering at indices 2-5!

2. Chaining (Separate Chaining)

  • Each slot holds a linked list of all key-value pairs that hash to that index
  • No clustering; multiple items share a slot via the linked list
  • Requires extra memory for linked list pointers
# Chaining: table size = 5
# Insert: "Cat"->0, "Dog"->3, "Act"->0 (collision!)

# Index 0: [Cat: "meow"] -> [Act: "perform"] -> null
# Index 1: null
# Index 2: null
# Index 3: [Dog: "woof"] -> null
# Index 4: null

Load Factor and Rehashing

Load factor = number of items / table size. A higher load factor means more collisions. When load factor exceeds a threshold (commonly 0.7), the table is resized (usually doubled) and all items rehashed with the new table size. This is called rehashing.

Hash Table Performance

OperationAverageWorst Case

InsertO(1)O(n)
DeleteO(1)O(n)

For linear probing, show each probe position. For chaining, show items being added to the linked list. Remember that the load factor determines when to rehash — know the formula and typical threshold.

4.2.9 Vectors

Vector: A mathematical object with both magnitude (size/length) and direction. In CS, vectors are represented as ordered lists of numbers and can be treated as: a 1D array, a function mapping indices to values, or a point in n-dimensional space.

Representations

  • A list of numbers: v = [3, 4] (2D), w = [1, 2, 5] (3D)
  • An arrow from the origin to a point in space
  • A function: f(0) = 3, f(1) = 4
  • A dictionary mapping indices to values (useful for sparse vectors)

Vector Operations

Addition (both vectors must have the same dimension):

[3, 4] + [1, 2] = [3+1, 4+2] = [4, 6]
[1, 5, 3] + [2, -1, 4] = [3, 4, 7]

Scalar multiplication (multiply every component by a scalar):

2 * [3, 4] = [6, 8]          # doubled in length
-1 * [3, 4] = [-3, -4]       # reversed direction
0.5 * [6, 10] = [3, 5]       # halved in length

Dot product (multiply corresponding components and sum; result is a scalar):

[3, 4] . [1, 2] = (3*1) + (4*2) = 3 + 8 = 11
[1, 0] . [0, 1] = (1*0) + (0*1) = 0   (perpendicular vectors!)

# Formula: u . v = u[0]*v[0] + u[1]*v[1] + ... + u[n-1]*v[n-1]
# If dot product = 0, vectors are perpendicular (orthogonal)

Magnitude (length, using Pythagoras):

|[3, 4]| = sqrt(3^2 + 4^2) = sqrt(9 + 16) = sqrt(25) = 5
|[1, 2, 2]| = sqrt(1^2 + 2^2 + 2^2) = sqrt(9) = 3

Convex combination (weighted average, point on line between two vectors):

result = (1-t) * u + t * v    where 0 ≤ t ≤ 1

# t=0: result = u,  t=1: result = v,  t=0.5: midpoint

# Example: u=[2,8], v=[10,4], t=0.25
# result = 0.75*[2,8] + 0.25*[10,4] = [1.5,6] + [2.5,1] = [4, 7]

Worked Example

# Given u = [3, -1, 2] and v = [1, 4, -2]

# Addition:   u + v = [3+1, -1+4, 2+(-2)] = [4, 3, 0]
# Subtraction: u - v = [3-1, -1-4, 2-(-2)] = [2, -5, 4]
# Scalar mult: 3u = [9, -3, 6]
# Dot product: u . v = (3*1) + (-1*4) + (2*-2) = 3 - 4 - 4 = -5
# Magnitude of u: |u| = sqrt(9 + 1 + 4) = sqrt(14) = 3.74
AQA may ask you to perform vector arithmetic by hand. Remember: addition requires vectors of the same dimension, the dot product always produces a scalar (not a vector), and magnitude uses Pythagoras. Be able to explain vectors as both a mathematical concept and a data structure (as a function from indices to values). Know all four operations: addition, scalar multiplication, dot product, and convex combination.

4.2.10 Dictionaries

Dictionary: An abstract data type that stores key-value pairs, where each unique key maps to exactly one value. Also known as an associative array, map, or symbol table. The key is used to look up the associated value.

Key Properties

  • Keys must be unique — inserting with an existing key overwrites the value
  • Keys must be immutable and hashable (strings, numbers, tuples — not lists)
  • Values can be any type and can be duplicated
  • Internally implemented using hash tables, giving O(1) average operations

students = {

        print(f"{info['name']} ({student_id}): {info['grade']}")

# Word frequency counter
def word_count(text):
    words = text.lower().split()
    freq = {}
    for word in words:
        if word in freq:
            freq[word] += 1
        else:
            freq[word] = 1
    return freq

print(word_count("the cat sat on the mat"))
# {'the': 2, 'cat': 1, 'sat': 1, 'on': 1, 'mat': 1}
Dictionaries are implemented using hash tables, which is why they offer O(1) average-case lookup. Keys must be hashable (immutable). You should be able to use dictionaries for common tasks: frequency counting, lookup tables, caching, and representing relationships.
