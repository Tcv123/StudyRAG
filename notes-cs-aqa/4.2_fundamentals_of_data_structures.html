4.2.1 Arrays

Array: An ordered, fixed-size collection of elements of the same data type, stored in contiguous (adjacent) memory locations. Elements are accessed by their index in O(1) constant time because the memory address can be calculated directly: address = base_address + (index × element_size).

One-Dimensional (1D) Arrays

A simple list of elements accessed by a single index. Most languages use zero-based indexing (first element at index 0). Some languages like Pascal and pseudocode use one-based indexing.

names = ["Alice", "Bob", "Charlie", "Diana", "Eve"]
# Index:    0        1        2         3       4

print(names[0])     # Alice (first element)
print(names[4])     # Eve (last element)
print(names[-1])    # Eve (Python shorthand for last element)
names[1] = "Ben"    # Modify element at index 1

# Traversing an array
for i in range(len(names)):
    print(f"Index {i}: {names[i]}")

# Common operations
length = len(names)              # 5
names.append("Frank")            # Note: Python lists are dynamic, not true fixed arrays
# In a true fixed-size array, you cannot add beyond the declared size

Two-Dimensional (2D) Arrays

A table or grid of elements accessed by two indices: row and column. Conceptually an "array of arrays". Useful for representing grids, matrices, tables, and game boards.

# 2D array (3 rows x 4 columns)
grid = [
    [1,  2,  3,  4],      # row 0
    [5,  6,  7,  8],      # row 1
    [9, 10, 11, 12]       # row 2
]

print(grid[1][2])          # 7 (row 1, column 2)
grid[0][3] = 99            # modify element at row 0, col 3

# Iterating through a 2D array
for row in range(3):
    for col in range(4):
        print(grid[row][col], end=" ")
    print()   # newline after each row

# Output:
# 1 2 3 99
# 5 6 7 8
# 9 10 11 12

# Pseudocode for summing all elements in a 2D array
total ← 0
FOR row ← 0 TO ROWS - 1
    FOR col ← 0 TO COLS - 1
        total ← total + grid[row][col]
    NEXT col
NEXT row

Multidimensional Arrays

Arrays with three or more dimensions. A 3D array can be thought of as a collection of 2D tables (like a stack of spreadsheets, or a Rubik's cube of values). Accessed with three indices: array[layer][row][col].

DimensionAnalogyAccess PatternExample Use
1DA single row of boxesa[i]List of names, sensor readings
2DA spreadsheet / tablea[row][col]Chess board, pixel grid, timetable
3DA stack of spreadsheetsa[layer][row][col]RGB image (3 colour channels), 3D game world

Key properties of arrays:

  • Fixed size — declared at creation, cannot grow or shrink (in most languages; Python lists are dynamic)
  • Homogeneous — all elements must be the same type (int, float, string, etc.)
  • Random access — any element accessed in O(1) time using its index, because the address is calculated directly
  • Contiguous memory — elements stored next to each other in RAM, enabling efficient CPU cache utilisation
  • Insertion/deletion is expensive — inserting or deleting in the middle requires shifting elements: O(n)
Arrays are used everywhere: pixel data in images (2D array of colour values), spreadsheet cells, look-up tables in embedded systems, and buffers for streaming data. In game development, a chess board is naturally a 2D array, and a 3D voxel world (like Minecraft) uses 3D arrays.

4.2.2 Records

Record: A composite data type that groups together related data items (fields) which may be of different types. Each field has a name and a data type. Records are also called structs (C), data classes (Kotlin/Python), or named tuples. 
TYPE Student
    firstName : STRING
    lastName  : STRING
    age       : INTEGER
    grade     : REAL
    enrolled  : BOOLEAN
ENDTYPE

pupil : Student
pupil.firstName ← "Alice"
pupil.lastName  ← "Smith"
pupil.age       ← 17
pupil.grade     ← 88.5
pupil.enrolled  ← True

# Python equivalent using a class or dataclass
from dataclasses import dataclass

@dataclass
class Student:
    first_name: str
    last_name: str
    age: int
    grade: float
    enrolled: bool

pupil = Student("Alice", "Smith", 17, 88.5, True)
print(pupil.first_name)   # Alice

Records are often used as the elements of arrays to create a simple database-like structure:

# Array of records - like a simple database table
students : ARRAY[0..29] OF Student

students[0].firstName ← "Alice"
students[0].age ← 17
students[1].firstName ← "Bob"
students[1].age ← 18

# Searching through an array of records
FOR i ← 0 TO 29
    IF students[i].grade >= 90 THEN
        OUTPUT students[i].firstName + " achieved an A*"
    ENDIF
NEXT i

Records vs Arrays: Arrays store multiple items of the same type; records store a single entity with multiple fields of potentially different types. Think of an array as a column in a table, and a record as a single row.

4.2.3 Stacks

Stack: A Last-In, First-Out (LIFO) abstract data type. The last item added is the first to be removed. Think of a stack of plates in a cafeteria — you can only add to or remove from the top.

Stack Operations

OperationDescriptionTimeError Condition
push(item)Add an item to the top of the stackO(1)Stack overflow (stack is full)
pop()Remove and return the item from the topO(1)Stack underflow (stack is empty)
peek() / top()View the top item without removing itO(1)Stack is empty
isEmpty()Check if the stack is emptyO(1)None
isFull()Check if the stack is full (array-based only)O(1)None
size()Return the number of items on the stackO(1)None

Array-Based Implementation

class Stack:
    def __init__(self, maxSize):
        self.stack = [None] * maxSize   # pre-allocate array
        self.maxSize = maxSize
        self.top = -1                   # -1 indicates empty stack

    def push(self, item):
        if self.top == self.maxSize - 1:
            raise Exception("Stack overflow - stack is full")
        self.top += 1                   # move top pointer up
        self.stack[self.top] = item     # place item at new top

    def pop(self):
        if self.top == -1:
            raise Exception("Stack underflow - stack is empty")
        item = self.stack[self.top]     # get item at top
        self.top -= 1                   # move top pointer down
        return item

    def peek(self):
        if self.top == -1:
            raise Exception("Stack is empty")
        return self.stack[self.top]     # return without removing

    def isEmpty(self):
        return self.top == -1

    def isFull(self):
        return self.top == self.maxSize - 1

    def size(self):
        return self.top + 1

# Trace through operations:
s = Stack(5)
s.push(10)     # stack: [10, _, _, _, _]  top=0
s.push(20)     # stack: [10, 20, _, _, _]  top=1
s.push(30)     # stack: [10, 20, 30, _, _]  top=2
print(s.peek())  # 30 (top item, not removed)
print(s.pop())   # 30 (removed), stack: [10, 20, _, _, _]  top=1
print(s.pop())   # 20 (removed), stack: [10, _, _, _, _]  top=0
print(s.isEmpty())  # False (10 still on stack)

Applications of Stacks

  • Function call stack — the system uses a stack to track return addresses and local variables during subroutine calls and recursion. When a function is called, a stack frame is pushed; when it returns, the frame is popped.
  • Undo functionality — each action is pushed onto an undo stack; pressing Ctrl+Z pops and reverses the last action
  • Expression evaluation — evaluating postfix (Reverse Polish Notation) expressions using the operand stack
  • Bracket matching — checking balanced parentheses in code/expressions: push opening brackets, pop when a closing bracket is encountered and check they match
  • Backtracking algorithms — depth-first search, maze solving, puzzle solving
  • Browser back button — visited pages are pushed onto a stack; pressing back pops the most recent page
  • Reversing data — push all items, then pop them out in reverse order
  • Infix to postfix conversion — the Shunting Yard algorithm uses an operator stack
# Bracket matching using a stack
def is_balanced(expression):
    stack = []
    matching = {')': '(', ']': '[', '}': '{'}
    for char in expression:
        if char in '([{':
            stack.append(char)        # push opening bracket
        elif char in ')]}':
            if len(stack) == 0:
                return False          # no matching opening bracket
            if stack.pop() != matching[char]:
                return False          # mismatched bracket types
    return len(stack) == 0            # empty stack = all matched

print(is_balanced("((2+3)*[4-1])"))   # True
print(is_balanced("((2+3)*[4-1)"))    # False - ] expected, got )
When tracing stack operations, draw the stack vertically with the top at the top. Show the top pointer moving up for push and down for pop. Always check for overflow (push to full stack) and underflow (pop from empty stack). A common exam mistake is confusing which end items are added/removed from — stacks always operate from the TOP only.

4.2.4 Queues

Queue: A First-In, First-Out (FIFO) abstract data type. The first item added is the first to be removed. Think of a queue of people at a bus stop — the person who arrived first is served first.

Queue Operations

OperationDescriptionTime
enqueue(item)Add an item to the rear of the queueO(1)
dequeue()Remove and return the item from the frontO(1)
peek() / front()View the front item without removing itO(1)
isEmpty()Check if the queue is emptyO(1)
isFull()Check if the queue is fullO(1)

Linear Queue

A simple array-based queue with front and rear pointers. The problem with a linear queue is that as items are dequeued, space at the front of the array is wasted and cannot be reused. Eventually the rear pointer reaches the end of the array even though there is free space at the front.

# Linear queue problem demonstration
# After several enqueue and dequeue operations:
# [_, _, _, D, E, F, G, _]
#           ^front     ^rear
# Indices 0-2 are wasted even though they are empty!

Circular Queue

Solves the wasted space problem by treating the array as circular — when a pointer reaches the end of the array, it wraps around to the beginning using the modulo operator: next_position = (current + 1) % maxSize.

class CircularQueue:
    def __init__(self, maxSize):
        self.queue = [None] * maxSize
        self.maxSize = maxSize
        self.front = 0
        self.rear = -1
        self.size = 0

    def enqueue(self, item):
        if self.size == self.maxSize:
            raise Exception("Queue is full")
        self.rear = (self.rear + 1) % self.maxSize  # wrap around!
        self.queue[self.rear] = item
        self.size += 1

    def dequeue(self):
        if self.size == 0:
            raise Exception("Queue is empty")
        item = self.queue[self.front]
        self.front = (self.front + 1) % self.maxSize  # wrap around!
        self.size -= 1
        return item

    def peek(self):
        if self.size == 0:
            raise Exception("Queue is empty")
        return self.queue[self.front]

    def isEmpty(self):
        return self.size == 0

# Trace through circular queue (maxSize = 4):
q = CircularQueue(4)
q.enqueue('A')     # [A, _, _, _]  front=0, rear=0, size=1
q.enqueue('B')     # [A, B, _, _]  front=0, rear=1, size=2
q.enqueue('C')     # [A, B, C, _]  front=0, rear=2, size=3
q.dequeue()        # [_, B, C, _]  front=1, rear=2, size=2  returns 'A'
q.dequeue()        # [_, _, C, _]  front=2, rear=2, size=1  returns 'B'
q.enqueue('D')     # [_, _, C, D]  front=2, rear=3, size=2
q.enqueue('E')     # [E, _, C, D]  front=2, rear=0, size=3  WRAPS AROUND!
#                    ^rear   ^front  -- rear has wrapped to index 0

Priority Queue

Each item has an associated priority. Items with higher priority are dequeued before items with lower priority, regardless of insertion order. If two items have equal priority, they follow FIFO order. This is NOT a FIFO structure.

Implementation options:

  • Sorted array/list: Insert in correct position (O(n) insert, O(1) dequeue — highest priority always at front)
  • Unsorted array/list: Insert at end (O(1) insert, O(n) dequeue — must search for highest priority item)
  • Binary heap: O(log n) for both insert and dequeue (most efficient in practice)

Applications of Queues

  • Print spooling — print jobs are processed in the order they are received (FIFO)
  • CPU scheduling — processes waiting to use the processor in round-robin scheduling
  • Breadth-first search (BFS) — traversing graphs level by level using a queue
  • Keyboard buffer — storing keystrokes in order before they are processed
  • Message queues — handling asynchronous communication between systems
  • Priority queue examples: hospital A&E triage (most critical patients first), operating system task scheduling, network packet prioritisation (VoIP over email)
Queue TypeAdvantageDisadvantageBest For
LinearSimplest to implementWastes space as items are dequeued; eventually runs out of spaceTeaching / very simple cases
CircularReuses space efficiently; no wasted memorySlightly more complex (modulo logic); still fixed sizeMost real queue implementations
PriorityHandles urgency-based orderingMore complex; insertion or deletion is O(n) unless using a heapScheduling, simulations
Operating systems use priority queues extensively. Real-time processes (like audio playback) get higher priority than background tasks (like virus scanning). Network routers use queues to buffer packets waiting to be forwarded; quality-of-service (QoS) uses priority queues to ensure voice and video traffic is forwarded before email.
When tracing circular queue operations, always use the modulo operator for both front and rear pointers. Draw the array and clearly label front, rear, and size after each operation. The key insight is that (rear + 1) % maxSize wraps rear back to index 0 when it reaches the end of the array. A common exam mistake is not wrapping front as well when dequeuing.

4.2.5 Linked Lists

Linked list: A dynamic data structure consisting of nodes, where each node contains a data field and a pointer (reference) to the next node in the sequence. The last node points to null. The list is accessed through a head pointer that references the first node.

Structure

Each node contains:
    data  — the value stored in this node
    next  — pointer/reference to the next node (null if last)

Visual representation:
HEAD → [Alice | →] → [Bob | →] → [Charlie | →] → [Diana | null]

The head pointer gives us entry to the list.
To reach Diana, we must traverse through Alice, Bob, and Charlie.

Types of Linked Lists

TypeDescriptionPointers per Node
Singly linkedEach node points to the next only; can only traverse forwards1 (next)
Doubly linkedEach node points to both the next and previous nodes; can traverse both ways2 (next, prev)
Circular linkedThe last node points back to the first (instead of null); forms a ring1 or 2

Static vs Dynamic Implementation

FeatureStatic (Array-based)Dynamic (Pointer-based)
Memory allocationFixed at compile time; uses an array of nodesAllocated at runtime from the heap as needed
SizeMaximum size predeterminedCan grow and shrink freely
Memory usageMay waste space if not fullOnly uses what is needed (plus pointer overhead per node)
Free space managementUses a free list (a linked list of unused array slots)OS manages free memory (heap allocation/deallocation)
Suitable forEmbedded systems, fixed-size poolsGeneral-purpose programming

Linked List Operations

Inserting at the beginning (O(1)):

  1. Create a new node with the data
  2. Set new node's next pointer to the current head
  3. Update head to point to the new node

Inserting at a specific position (O(n) to find position, O(1) to insert):

  1. Traverse to the node before the insertion point
  2. Set new node's next to the current node's next
  3. Set current node's next to the new node

Deleting a node (O(n) to find, O(1) to remove):

  1. Traverse to the node before the one to delete
  2. Set its next to the deleted node's next (bypassing it)
  3. Free the deleted node's memory (in dynamic implementations)

Searching (O(n)): Start at head, traverse node by node, comparing each node's data to the target, until found or null is reached.

class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None

    def insert_at_start(self, data):
        new_node = Node(data)
        new_node.next = self.head   # point new node to current head
        self.head = new_node        # update head to new node

    def insert_at_end(self, data):
        new_node = Node(data)
        if self.head is None:       # empty list
            self.head = new_node
            return
        current = self.head
        while current.next is not None:   # traverse to last node
            current = current.next
        current.next = new_node     # last node now points to new node

    def delete(self, target):
        if self.head is None:
            return                  # empty list, nothing to delete
        if self.head.data == target:
            self.head = self.head.next   # delete head node
            return
        current = self.head
        while current.next is not None:
            if current.next.data == target:
                current.next = current.next.next   # bypass deleted node
                return
            current = current.next

    def search(self, target):
        current = self.head
        index = 0
        while current is not None:
            if current.data == target:
                return index        # found at this index
            current = current.next
            index += 1
        return -1                   # not found

    def display(self):
        current = self.head
        while current is not None:
            print(current.data, end=" -> ")
            current = current.next
        print("None")

# Usage
ll = LinkedList()
ll.insert_at_start("Charlie")
ll.insert_at_start("Bob")
ll.insert_at_start("Alice")
ll.display()                # Alice -> Bob -> Charlie -> None
ll.delete("Bob")
ll.display()                # Alice -> Charlie -> None
print(ll.search("Charlie")) # 1

Static Array Implementation of a Linked List

When dynamic memory allocation is not available (e.g., in some embedded systems), a linked list can be simulated using an array. Each array element has a data field and a "pointer" field (which stores an array index rather than a memory address). A free list tracks which array positions are available.

# Static linked list using an array
# Array index:  0      1       2       3       4       5
# Data:        "Bob"  "Eve"   "Alice" [free]  "Diana" [free]
# Next:         4      -1      0       5       1       -1
# Head pointer: 2 (points to "Alice")
# Free list pointer: 3 (first free slot)

# Following the chain from head:
# Index 2 ("Alice") -> next=0 -> Index 0 ("Bob") -> next=4 ->
# Index 4 ("Diana") -> next=1 -> Index 1 ("Eve") -> next=-1 (end)

Linked Lists vs Arrays

FeatureArrayLinked List
SizeFixed (must be declared in advance)Dynamic (grows/shrinks as needed)
Access by indexO(1) — direct random accessO(n) — must traverse from head
Insertion at startO(n) — all elements must shift rightO(1) — just update head pointer
Insertion at endO(1) if space availableO(n) — must traverse to end (O(1) if tail pointer maintained)
DeletionO(n) — elements must shift to fill gapO(1) once position found (just re-link pointers)
MemoryContiguous block, no pointer overheadNon-contiguous, extra memory for pointer in each node

When implementing a linked list using a static array, each array element has a data field and a pointer (index) field. You also need a free list pointer to track available array positions. Insertions take a node from the free list; deletions return a node to it. Be prepared to trace through insertion and deletion operations, showing both the data array and the head/free pointers changing at each step.

4.2.6 Trees

Tree: A connected, acyclic (no cycles), hierarchical data structure consisting of nodes connected by edges. It has a single root node (the topmost node), and every other node has exactly one parent. A tree with n nodes has exactly n-1 edges.

Tree Terminology

TermDefinitionExample
RootThe topmost node with no parentThe CEO in an org chart
ChildA node directly connected below another nodeDepartments under the CEO
ParentA node directly connected above another nodeCEO is parent of each department
LeafA node with no children (terminal node)Individual employees
EdgeThe connection (link) between a parent and childThe line in an org chart
SubtreeA node and all its descendantsA department and all its members
Depth / LevelThe number of edges from the root to that node (root is depth 0)CEO=0, Dept=1, Team=2
HeightThe number of edges on the longest path from root to any leafHeight of org chart
SiblingNodes that share the same parentDepartments under the same CEO

Binary Trees

A tree where each node has at most two children: a left child and a right child. A node with zero children is a leaf. A complete binary tree has all levels fully filled except possibly the last, which is filled from left to right.

Binary Search Trees (BST)

Binary Search Tree: A binary tree with the ordering property: for every node, all values in the left subtree are less than the node's value, and all values in the right subtree are greater than the node's value. This property enables efficient searching. 
         15
        /  \
       8    23
      / \   / \
     4  12 18  27
    /
   2

All values left of 15 are < 15: {2, 4, 8, 12}
All values right of 15 are > 15: {18, 23, 27}
This property holds at EVERY node, not just the root.

BST operations:

Search: Compare target with current node. If equal, found. If target is smaller, go left. If larger, go right. Repeat until found or reach null.

Searching for 12:
  Start at 15: 12 < 15, go LEFT
  At 8: 12 > 8, go RIGHT
  At 12: FOUND!

Searching for 20:
  Start at 15: 20 > 15, go RIGHT
  At 23: 20 < 23, go LEFT
  At 18: 20 > 18, go RIGHT
  null → NOT FOUND

Insert: Follow the search path until you reach a null pointer, then place the new node there.

Inserting 10 into the BST above:
  Start at 15: 10 < 15, go LEFT
  At 8: 10 > 8, go RIGHT
  At 12: 10 < 12, go LEFT
  null → Insert 10 as left child of 12

         15
        /  \
       8    23
      / \   / \
     4  12 18  27
    /  /
   2  10

Delete: Three cases:

  1. Leaf node — simply remove it (set parent's pointer to null)
  2. One child — replace the node with its only child
  3. Two children — replace with the in-order successor (smallest value in the right subtree) or in-order predecessor (largest value in left subtree), then delete that successor/predecessor node

Time complexity:

OperationAverage (balanced tree)Worst (degenerate/unbalanced)
SearchO(log n)O(n)
InsertO(log n)O(n)
DeleteO(log n)O(n)

The worst case occurs when the tree degenerates into a linked list (e.g., inserting values in sorted order: 1, 2, 3, 4, 5 creates a chain going only right).

Tree Traversals

Three depth-first traversal methods, each visiting nodes in a different order. All three use recursion:

TraversalOrderMnemonicResult for BST above
Pre-orderRoot, Left, RightVisit the root before its children15, 8, 4, 2, 12, 23, 18, 27
In-orderLeft, Root, RightVisit the root between its children2, 4, 8, 12, 15, 18, 23, 27
Post-orderLeft, Right, RootVisit the root after its children2, 4, 12, 8, 18, 27, 23, 15
# Recursive implementations
def pre_order(node):
    if node is not None:
        print(node.data)       # VISIT root first
        pre_order(node.left)   # then traverse left subtree
        pre_order(node.right)  # then traverse right subtree

def in_order(node):
    if node is not None:
        in_order(node.left)    # traverse left subtree first
        print(node.data)       # VISIT root in the middle
        in_order(node.right)   # then traverse right subtree

def post_order(node):
    if node is not None:
        post_order(node.left)  # traverse left subtree first
        post_order(node.right) # then traverse right subtree
        print(node.data)       # VISIT root last

Critical fact: An in-order traversal of a BST produces the values in ascending sorted order. This is because it visits all smaller values (left) before the node, and all larger values (right) after.

Uses of each traversal:

  • Pre-order: Copying/cloning a tree, serialising a tree to a file (can reconstruct the tree from pre-order data), prefix expression evaluation
  • In-order: Outputting sorted data from a BST, converting BST to sorted array
  • Post-order: Safely deleting a tree (children deleted before parent), evaluating postfix expression trees, calculating directory sizes (process children before parent folder)
To trace a traversal quickly, use the "hand-hugging" technique: imagine drawing a continuous line around the outside of the tree, hugging each node. Start from the left side of the root. For pre-order, record a node when the line passes its left side. For in-order, record it when passing underneath. For post-order, record it when passing its right side. This works every time and is faster than tracing recursion.

4.2.7 Graphs

Graph: A data structure consisting of a set of vertices (nodes) connected by edges (arcs/links). Unlike trees, graphs can contain cycles (paths that loop back to a starting vertex) and vertices can have any number of connections. Formally: G = (V, E) where V is the set of vertices and E is the set of edges.

Types of Graphs

TypeDescriptionReal-World Example
Directed (digraph)Edges have a direction (one-way arrow from A to B does not imply B to A)Web page hyperlinks, Twitter follows, one-way streets
UndirectedEdges have no direction (connection goes both ways)Facebook friendships, road networks (two-way streets)
WeightedEach edge has an associated numerical cost, distance, or capacityRoad network with distances, flight routes with prices
UnweightedAll edges are treated equally (no cost associated)Simple social network connections
ConnectedThere is a path between every pair of verticesA fully connected road network

AcyclicContains no cycles (a tree is a connected acyclic graph)Dependency graphs, file system hierarchy

Adjacency Matrix

A 2D array where matrix[i][j] represents the edge from vertex i to vertex j. For weighted graphs, the value is the weight; for unweighted, it is 1 (connected) or 0 (not connected). A special value (e.g., 0 or infinity) indicates no edge.

# Undirected, unweighted graph: A-B, A-C, B-C, B-D
#     A  B  C  D
# A [ 0, 1, 1, 0 ]   A connects to B and C
# B [ 1, 0, 1, 1 ]   B connects to A, C, D
# C [ 1, 1, 0, 0 ]   C connects to A, B
# D [ 0, 1, 0, 0 ]   D connects to B only

# Note: the matrix is SYMMETRIC for undirected graphs
# matrix[i][j] = matrix[j][i] always

# Weighted version (distances in km):
#     A    B    C    D
# A [ 0,   5,   3, inf ]
# B [ 5,   0,   2,   7 ]
# C [ 3,   2,   0, inf ]
# D [inf,  7, inf,   0 ]

Adjacency List

Each vertex stores a list of its adjacent vertices (and edge weights if applicable). More memory-efficient for sparse graphs (graphs with relatively few edges compared to the maximum possible).

# Same graph as above
adjacency_list = {
    'A': ['B', 'C'],
    'B': ['A', 'C', 'D'],
    'C': ['A', 'B'],
    'D': ['B']
}

# Weighted version
weighted_list = {
    'A': [('B', 5), ('C', 3)],
    'B': [('A', 5), ('C', 2), ('D', 7)],
    'C': [('A', 3), ('B', 2)],
    'D': [('B', 7)]
}

4.2.8 Hash Tables

Hash table: A data structure that maps keys to values using a hash function. The hash function converts a key into an array index, allowing near-constant-time (O(1) average) access for insertion, deletion, and lookup.

Hashing

A hash function takes a key (e.g., a string, number, or any data) and computes an integer index into the hash table's underlying array. A good hash function should:

  • Distribute keys uniformly across the table (minimising clustering)
  • Be deterministic — the same key must always produce the same hash
  • Be fast to compute (O(1) time)
  • Minimise collisions (different keys mapping to the same index)
# Simple hash function examples
def hash_mod(key, table_size):
    """Sum ASCII values and take modulo"""
    total = 0
    for char in str(key):
        total += ord(char)
    return total % table_size

print(hash_mod("Cat", 10))   # (67+97+116) % 10 = 280 % 10 = 0
print(hash_mod("Dog", 10))   # (68+111+103) % 10 = 282 % 10 = 2
print(hash_mod("Bat", 10))   # (66+97+116) % 10 = 279 % 10 = 9

# A better hash function uses position-weighting to avoid anagrams colliding:
def hash_weighted(key, table_size):
    total = 0
    for i, char in enumerate(str(key)):
        total += ord(char) * (i + 1)   # multiply by position
    return total % table_size

Collisions

A collision occurs when two different keys hash to the same index. Since the hash function maps a large key space to a small index space, collisions are inevitable (by the pigeonhole principle). Two main collision resolution strategies:

1. Open Addressing (Linear Probing):

  • If the target slot is occupied, check the next slot, then the next, wrapping around to the beginning if necessary
  • To search: hash the key, then probe forward until the key is found or an empty slot is reached
  • Simple to implement but can cause clustering — groups of consecutive occupied slots that force long probe sequences
  • Other variants: quadratic probing (probe 1, 4, 9, 16... slots ahead), double hashing (use a second hash function to determine step size)
# Linear probing example: table size = 7
# Insert keys with hashes: A->2, B->2, C->3, D->2

# Insert A (hash=2): slot 2 empty, place A
# [_, _, A, _, _, _, _]

# Insert B (hash=2): slot 2 occupied, try 3 -> empty, place B
# [_, _, A, B, _, _, _]

# Insert C (hash=3): slot 3 occupied, try 4 -> empty, place C
# [_, _, A, B, C, _, _]

# Insert D (hash=2): slots 2,3,4 occupied, try 5 -> empty, place D
# [_, _, A, B, C, D, _]
# Notice the cluster forming at indices 2-5!

2. Chaining (Separate Chaining):

  • Each slot in the table holds a linked list (or other collection) of all key-value pairs that hash to that index
  • No clustering — multiple items simply share a slot via the linked list
  • Requires extra memory for linked list pointers
  • Performance degrades if lists become very long (worst case: all items in one chain = O(n))
# Chaining example: table size = 5
# Insert: "Cat"->index 0, "Dog"->index 3, "Bat"->index 0 (collision!)

# Index 0: [Cat: "meow"] -> [Bat: "squeak"] -> None
# Index 1: None
# Index 2: None
# Index 3: [Dog: "woof"] -> None
# Index 4: None

# To find "Bat": hash("Bat")=0, go to index 0, traverse linked list
# until "Bat" is found

Load Factor and Rehashing

Load factor = number of items stored / table size. A higher load factor means more collisions and slower performance. When the load factor exceeds a threshold (commonly 0.7 or 70%), the table should be resized (usually doubled to the next prime number) and all items rehashed with the new table size.

Hash tables are used to implement Python dictionaries, JavaScript objects, database indexing, caching systems (like memcached/Redis), compiler symbol tables (mapping variable names to memory addresses), and DNS lookup (mapping domain names to IP addresses). They are one of the most commonly used data structures in practice.

4.2.9 Vectors

Vector: A mathematical object that has both magnitude (size/length) and direction. In computer science, vectors are represented as ordered lists of numbers and can be treated as a 1D array, a function mapping indices to values, or a point in n-dimensional space.

Representations

Vectors can be viewed in multiple ways:

  • A list of numbers: v = [3, 4] (a 2D vector), w = [1, 2, 5] (a 3D vector)
  • An arrow from the origin to a point in space (geometric interpretation)
  • A function from index set {0, 1, ..., n-1} to real numbers: f(0) = 3, f(1) = 4
  • A dictionary mapping indices to values (useful for sparse vectors where most values are 0)

Vector Operations

Addition: Add corresponding components. Both vectors must have the same dimension.

[3, 4] + [1, 2] = [3+1, 4+2] = [4, 6]
[1, 5, 3] + [2, -1, 4] = [3, 4, 7]

Scalar multiplication: Multiply every component by a scalar (single number). Scales the vector's magnitude without changing direction (unless the scalar is negative, which reverses direction).

2 * [3, 4] = [6, 8]          # doubled in length
-1 * [3, 4] = [-3, -4]       # same length, opposite direction
0.5 * [6, 10] = [3, 5]       # halved in length

Dot product (scalar product): Multiply corresponding components and sum the results. The result is a single scalar (number), not a vector. The dot product measures how "aligned" two vectors are.

[3, 4] . [1, 2] = (3*1) + (4*2) = 3 + 8 = 11
[1, 0] . [0, 1] = (1*0) + (0*1) = 0    (perpendicular vectors!)

# General formula:
# u . v = u[0]*v[0] + u[1]*v[1] + ... + u[n-1]*v[n-1]

# If dot product = 0, the vectors are perpendicular (orthogonal)

Magnitude (length): The length of a vector, calculated using Pythagoras' theorem.

|[3, 4]| = sqrt(3^2 + 4^2) = sqrt(9 + 16) = sqrt(25) = 5
|[1, 2, 2]| = sqrt(1^2 + 2^2 + 2^2) = sqrt(9) = 3

Convex combination: A weighted average of two vectors where the weights are non-negative and sum to 1. Produces a point that lies on the straight line between the two vectors.

result = (1-t) * u + t * v    where 0 ≤ t ≤ 1

# When t = 0: result = u (start point)
# When t = 1: result = v (end point)
# When t = 0.5: result = midpoint of u and v

# Example: u = [2, 8], v = [10, 4], t = 0.25
# result = 0.75 * [2, 8] + 0.25 * [10, 4]
#        = [1.5, 6] + [2.5, 1]
#        = [4, 7]   (a point 25% of the way from u to v)

Applications of Vectors

  • Computer graphics — positions, directions, velocities, forces, normals for lighting
  • Machine learning — feature vectors representing data points; weight vectors in neural networks
  • Physics simulations — velocity, acceleration, gravitational force
  • Natural language processing — word embeddings (words represented as vectors; similar words have similar vectors)
  • Recommendation systems — users and items as vectors; dot product measures compatibility
AQA may ask you to perform vector arithmetic by hand. Remember: addition requires vectors of the same dimension, and the dot product always produces a scalar (not a vector). Be able to explain vectors as both a mathematical concept and a data structure (as a function from indices to values).

4.2.10 Dictionaries

Dictionary: An abstract data type that stores key-value pairs, where each unique key maps to exactly one value. Also known as an associative array, map, or symbol table. The key is used to look up the associated value, much like looking up a word in an English dictionary to find its definition.

Key Properties

  • Keys must be unique — no duplicate keys (inserting with an existing key overwrites the value)
  • Keys must be immutable and hashable (in hash-based implementations) — strings, numbers, and tuples can be keys; lists and other mutable objects cannot
  • Values can be of any type and can be duplicated across different keys
  • No inherent ordering in the abstract definition (though Python 3.7+ dicts maintain insertion order)

Implementation

Dictionaries are most commonly implemented using hash tables, which is why lookup, insertion, and deletion are all O(1) on average. The key is hashed to determine the array index where the value is stored. This is why keys must be hashable.

# Practical dictionary example: student database
students = {
    "S001": {"name": "Alice", "age": 17, "grade": 92},
    "S002": {"name": "Bob", "age": 18, "grade": 85},
    "S003": {"name": "Charlie", "age": 17, "grade": 78}
}

# Access nested data
print(students["S001"]["name"])     # Alice
print(students["S002"]["grade"])    # 85

# Add a new student
students["S004"] = {"name": "Diana", "age": 17, "grade": 95}

# Check if a student exists before accessing
student_id = "S005"
if student_id in students:
    print(students[student_id]["name"])
else:
    print(f"Student {student_id} not found")

# Iterate through all students
for sid, info in students.items():
    if info["grade"] >= 90:
        print(f"{info['name']} ({sid}): A* student")

# Using .get() for safe access with a default value
grade = students.get("S999", {}).get("grade", "N/A")
print(grade)   # "N/A" (student doesn't exist)
