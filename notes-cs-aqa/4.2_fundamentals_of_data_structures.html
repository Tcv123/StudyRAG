4.2.1 Arrays

Array: An ordered, fixed-size collection of elements of the same data type, stored in contiguous memory locations. Elements are accessed by their index.

One-Dimensional (1D) Arrays

A simple list of elements accessed by a single index. Most languages use zero-based indexing (first element at index 0).

names = ["Alice", "Bob", "Charlie", "Diana"]
# Index:    0        1        2         3

print(names[0])     # Alice
print(names[2])     # Charlie
names[1] = "Ben"    # Modify element at index 1

Two-Dimensional (2D) Arrays

A table or grid of elements accessed by two indices: row and column. Conceptually an "array of arrays".

# 2D array (3 rows x 4 columns)
grid = [
    [1, 2, 3, 4],      # row 0
    [5, 6, 7, 8],      # row 1
    [9, 10, 11, 12]    # row 2
]

print(grid[1][2])       # 7 (row 1, column 2)

# Iterating through a 2D array
for row in range(3):
    for col in range(4):
        print(grid[row][col], end=" ")
    print()

Multidimensional Arrays

Arrays with three or more dimensions. A 3D array can be thought of as a collection of 2D tables. Accessed with three indices: array[layer][row][col].

DimensionAnalogyAccess Pattern
1DA single row of boxesa[i]
2DA spreadsheet / tablea[row][col]
3DA stack of spreadsheetsa[layer][row][col]

Key properties of arrays:

  • Fixed size — declared at creation, cannot grow or shrink
  • Homogeneous — all elements must be the same type
  • Random access — any element accessed in O(1) time using its index
  • Contiguous memory — elements stored next to each other, allowing efficient access

4.2.2 Records

Record: A composite data type that groups together related data items (fields) which may be of different types. Each field has a name and a data type. 
TYPE Student
    firstName : STRING
    lastName  : STRING
    age       : INTEGER
    grade     : REAL
    enrolled  : BOOLEAN
ENDTYPE

pupil : Student
pupil.firstName ← "Alice"
pupil.lastName  ← "Smith"
pupil.age       ← 17
pupil.grade     ← 88.5
pupil.enrolled  ← True

Records are often used as the elements of arrays to create a simple database-like structure:

# Array of records
students : ARRAY[0..29] OF Student

students[0].firstName ← "Alice"
students[0].age ← 17

Records vs Arrays: Arrays store multiple items of the same type; records store a single entity with multiple fields of potentially different types.

4.2.3 Stacks

Stack: A Last-In, First-Out (LIFO) data structure. The last item added is the first to be removed. Think of a stack of plates.

Stack Operations

OperationDescriptionTime Complexity
push(item)Add an item to the top of the stackO(1)
pop()Remove and return the item from the topO(1)
peek() / top()View the top item without removing itO(1)
isEmpty()Check if the stack is emptyO(1)
isFull()Check if the stack is full (array-based)O(1)
size()Return the number of itemsO(1)

Array-Based Implementation

class Stack:
    def __init__(self, maxSize):
        self.stack = [None] * maxSize
        self.maxSize = maxSize
        self.top = -1              # -1 indicates empty stack

    def push(self, item):
        if self.top == self.maxSize - 1:
            raise Exception("Stack overflow")
        self.top += 1
        self.stack[self.top] = item

    def pop(self):
        if self.top == -1:
            raise Exception("Stack underflow")
        item = self.stack[self.top]
        self.top -= 1
        return item

    def peek(self):
        if self.top == -1:
            raise Exception("Stack is empty")
        return self.stack[self.top]

    def isEmpty(self):
        return self.top == -1

Applications of Stacks

  • Function call stack — tracking return addresses and local variables during subroutine calls and recursion
  • Undo functionality — each action is pushed; undo pops the last action
  • Expression evaluation — evaluating postfix (Reverse Polish Notation) expressions
  • Bracket matching — checking balanced parentheses in code/expressions
  • Backtracking algorithms — depth-first search, maze solving
  • Reversing data — push all items, then pop them out in reverse order
When tracing stack operations, draw the stack vertically with the top at the top. Show the top pointer moving up for push and down for pop. Always check for overflow (push to full stack) and underflow (pop from empty stack).

4.2.4 Queues

Queue: A First-In, First-Out (FIFO) data structure. The first item added is the first to be removed. Think of a queue of people waiting.

Queue Operations

OperationDescription
enqueue(item)Add an item to the rear of the queue
dequeue()Remove and return the item from the front
peek() / front()View the front item without removing it
isEmpty()Check if the queue is empty
isFull()Check if the queue is full

Linear Queue

A simple array-based queue with front and rear pointers. The problem with a linear queue is that as items are dequeued, space at the front of the array is wasted and cannot be reused. Eventually the rear pointer reaches the end even though there is free space at the front.

Circular Queue

Solves the wasted space problem by treating the array as circular. When the rear pointer reaches the end, it wraps around to the beginning using the modulo operator.

class CircularQueue:
    def __init__(self, maxSize):
        self.queue = [None] * maxSize
        self.maxSize = maxSize
        self.front = 0
        self.rear = -1
        self.size = 0

    def enqueue(self, item):
        if self.size == self.maxSize:
            raise Exception("Queue is full")
        self.rear = (self.rear + 1) % self.maxSize  # wrap around
        self.queue[self.rear] = item
        self.size += 1

    def dequeue(self):
        if self.size == 0:
            raise Exception("Queue is empty")
        item = self.queue[self.front]
        self.front = (self.front + 1) % self.maxSize  # wrap around
        self.size -= 1
        return item

    def isEmpty(self):
        return self.size == 0

Priority Queue

Each item has an associated priority. Items with higher priority are dequeued before items with lower priority, regardless of insertion order. If two items have equal priority, they follow FIFO order.

Implementation options:

  • Sorted array/list: Insert in correct position (O(n) insert, O(1) dequeue)
  • Unsorted array/list: Insert at end (O(1) insert, O(n) dequeue — must search for highest priority)
  • Heap: O(log n) for both insert and dequeue (most efficient)

Applications of Queues

  • Print spooling — print jobs processed in order received
  • CPU scheduling — processes waiting to use the processor
  • Breadth-first search — traversing graphs level by level
  • Keyboard buffer — storing keystrokes in order
  • Priority queue: hospital A&E triage, operating system task scheduling
Queue TypeAdvantageDisadvantage
LinearSimple to implementWastes space as items dequeue
CircularReuses space efficientlySlightly more complex (modulo logic)
PriorityHandles urgency-based orderingMore complex; insertion or deletion is O(n) unless using a heap

4.2.5 Linked Lists

Linked list: A dynamic data structure consisting of nodes, where each node contains a data field and a pointer (reference) to the next node in the sequence.

Structure

Node:
    data  — the value stored
    next  — pointer to the next node (or null if last node)

[Alice | →] → [Bob | →] → [Charlie | null]
  head

Static vs Dynamic Implementation

FeatureStatic (Array-based)Dynamic (Pointer-based)
Memory allocationFixed at compile timeAllocated at runtime as needed
SizeMaximum size predeterminedCan grow and shrink freely
Memory usageMay waste space if not fullOnly uses what is needed (plus pointer overhead)
Free space managementUses a free list within the arrayOS manages free memory (heap)

Linked List Operations

Inserting at the beginning:

  1. Create a new node with the data
  2. Set new node's next to the current head
  3. Update head to point to the new node

Inserting at a specific position:

  1. Traverse to the node before the insertion point
  2. Set new node's next to the current node's next
  3. Set current node's next to the new node

Deleting a node:

  1. Traverse to the node before the one to delete
  2. Set its next to the deleted node's next (bypassing the deleted node)
  3. Free the deleted node's memory (in dynamic implementations)

Searching: Start at head, traverse node by node until the target is found or the end is reached. Time complexity: O(n).

class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None

    def insert_at_start(self, data):
        new_node = Node(data)
        new_node.next = self.head
        self.head = new_node

    def delete(self, target):
        if self.head is None:
            return
        if self.head.data == target:
            self.head = self.head.next
            return
        current = self.head
        while current.next is not None:
            if current.next.data == target:
                current.next = current.next.next
                return
            current = current.next

    def display(self):
        current = self.head
        while current is not None:
            print(current.data, end=" -> ")
            current = current.next
        print("None")

Linked Lists vs Arrays

FeatureArrayLinked List
SizeFixedDynamic
Access by indexO(1) — directO(n) — must traverse
Insertion / deletionO(n) — elements must shiftO(1) once position found
MemoryContiguous, no pointer overheadNon-contiguous, extra memory for pointers
Cache performanceBetter (locality)Worse (scattered in memory)
When implementing a linked list using a static array, each array element has a data field and a pointer (index) field. You also need a free list pointer to track available array positions. Insertions take a node from the free list; deletions return a node to it.

4.2.6 Trees

Tree: A connected, hierarchical data structure with no cycles, consisting of nodes connected by edges. It has a single root node, and every other node has exactly one parent.

Tree Terminology

TermDefinition
RootThe topmost node (no parent)
ChildA node directly connected below another node
ParentA node directly connected above another node
LeafA node with no children (also called a terminal node)
EdgeThe connection between a parent and child node
SubtreeA node and all its descendants
Depth / LevelThe number of edges from the root to a node
HeightThe number of edges on the longest path from root to leaf

Binary Trees

A tree where each node has at most two children: a left child and a right child.

Binary Search Trees (BST)

Binary Search Tree: A binary tree where for every node, all values in the left subtree are less than the node's value, and all values in the right subtree are greater than the node's value. 
         15
        /  \
       8    23
      / \   / \
     4  12 18  27

Searching for 18:
  15 → 18 > 15, go right
  23 → 18 < 23, go left
  18 → found!

BST operations:

  • Search: Compare target with current node; go left if smaller, right if larger. Average O(log n), worst O(n) if tree is unbalanced.
  • Insert: Search for the correct position (where a null pointer is), then place the new node there.
  • Delete: Three cases: (1) leaf node — simply remove; (2) one child — replace with child; (3) two children — replace with in-order successor (smallest value in right subtree) or in-order predecessor.

Tree Traversals

Three depth-first traversal methods, each visiting nodes in a different order:

TraversalOrderMnemonicResult for tree above
Pre-orderRoot, Left, RightVisit before children15, 8, 4, 12, 23, 18, 27
In-orderLeft, Root, RightVisit between children4, 8, 12, 15, 18, 23, 27
Post-orderLeft, Right, RootVisit after children4, 12, 8, 18, 27, 23, 15
# Recursive implementations
def pre_order(node):
    if node is not None:
        print(node.data)       # visit root
        pre_order(node.left)   # traverse left
        pre_order(node.right)  # traverse right

def in_order(node):
    if node is not None:
        in_order(node.left)    # traverse left
        print(node.data)       # visit root
        in_order(node.right)   # traverse right

def post_order(node):
    if node is not None:
        post_order(node.left)  # traverse left
        post_order(node.right) # traverse right
        print(node.data)       # visit root

Key fact: An in-order traversal of a BST produces the values in ascending sorted order.

Uses of traversals:

  • Pre-order: copying a tree, serialising a tree structure
  • In-order: outputting sorted data from a BST
  • Post-order: deleting a tree (children deleted before parent), evaluating expression trees
To trace a traversal, use the "hand-hugging" technique: draw a path around the outside of the tree starting from the left of the root. For pre-order, record a node when you pass its left side. For in-order, record it when you pass underneath. For post-order, record it when you pass its right side.

Graph: A data structure consisting of a set of vertices (nodes) connected by edges (arcs). Unlike trees, graphs can contain cycles and nodes can have any number of connections.

Adjacency Matrix

A 2D array where matrix[i][j] represents the edge from vertex i to vertex j. For weighted graphs, the value is the weight; for unweighted, it is 1 (connected) or 0 (not connected).

Adjacency List

Each vertex stores a list of its adjacent vertices (and edge weights if applicable).

4.2.8 Hash Tables

Hash table: A data structure that maps keys to values using a hash function. The hash function converts a key into an array index, allowing near-constant-time access.

Hashing

A hash function takes a key (e.g., a string) and computes an integer index into the hash table's array. A good hash function:

  • Distributes keys uniformly across the table
  • Is deterministic — same key always produces the same hash
  • Is fast to compute
  • Minimises collisions
# Simple hash function example
def hash_function(key, table_size):
    total = 0
    for char in key:
        total += ord(char)       # sum ASCII values
    return total % table_size    # modulo to fit in table

# hash_function("Cat", 10) = (67+97+116) % 10 = 280 % 10 = 0

Collisions

A collision occurs when two different keys hash to the same index. Collision resolution strategies:

1. Open Addressing (Linear Probing):

  • If the target slot is occupied, check the next slot, then the next, and so on (wrapping around)
  • Simple to implement but can cause clustering — groups of occupied slots that slow down operations
  • Other probing methods: quadratic probing (check i² slots ahead), double hashing (use a second hash function to determine step size)

2. Chaining (Separate Chaining):

  • Each slot in the table holds a linked list of all key-value pairs that hash to that index
  • No clustering, but requires extra memory for linked list pointers
  • Performance degrades if lists become long
# Chaining example
# Table size = 5
# Insert: "Cat"->0, "Dog"->3, "Bat"->0 (collision with Cat)

# Index 0: [Cat: value] -> [Bat: value] -> None
# Index 1: None
# Index 2: None
# Index 3: [Dog: value] -> None
# Index 4: None

Load Factor

Load factor = number of items stored / table size. A higher load factor means more collisions. When the load factor exceeds a threshold (commonly 0.7), the table should be resized (usually doubled) and all items rehashed.

Hash Table Performance

OperationAverage CaseWorst Case
SearchO(1)O(n)
InsertO(1)O(n)
DeleteO(1)O(n)

Worst case occurs when all keys hash to the same index (all items in one chain or cluster).

4.2.9 Vectors

Vector: A mathematical object that has both magnitude (size) and direction. In computer science, vectors are represented as ordered lists of numbers and can be treated as a special form of a 1D array or as a function mapping indices to values.

Representations

Vectors can be viewed as:

  • A list of numbers: v = [3, 4] (a 2D vector), w = [1, 2, 5] (a 3D vector)
  • An arrow from the origin to a point in space
  • A function from index set {0, 1, ..., n-1} to real numbers: f(0) = 3, f(1) = 4
  • A dictionary mapping indices to values (useful for sparse vectors)

Vector Operations

Addition: Add corresponding components.

[3, 4] + [1, 2] = [3+1, 4+2] = [4, 6]

Scalar multiplication: Multiply every component by a scalar.

2 * [3, 4] = [6, 8]

Dot product (scalar product): Multiply corresponding components and sum the results. The result is a single number (scalar), not a vector.

[3, 4] . [1, 2] = (3*1) + (4*2) = 3 + 8 = 11

# General formula:
# u . v = u[0]*v[0] + u[1]*v[1] + ... + u[n-1]*v[n-1]

Convex combination: A weighted average of two vectors where the weights are non-negative and sum to 1. If u and v are vectors and 0 ≤ t ≤ 1:

result = (1-t) * u + t * v

# When t = 0, result = u
# When t = 1, result = v
# When t = 0.5, result = midpoint of u and v

Applications of Vectors

  • Computer graphics — positions, directions, forces
  • Machine learning — feature vectors representing data points
  • Physics simulations — velocity, acceleration
  • Natural language processing — word embeddings
AQA may ask you to perform vector arithmetic by hand. Remember: addition requires vectors of the same dimension, and the dot product always produces a scalar. Be able to explain vectors as both a mathematical concept and a data structure.

4.2.10 Dictionaries

Dictionary: An abstract data type that stores key-value pairs, where each unique key maps to a value. Also known as an associative array or map.

Key Properties

  • Keys must be unique (no duplicate keys)
  • Keys must be immutable (in most implementations)
  • Values can be of any type and can be duplicated
  • No inherent ordering (though some implementations maintain insertion order)

Operations

OperationDescriptionExample (Python)
CreateInitialise a dictionaryd = {"name": "Alice", "age": 17}
Add / UpdateInsert a new key-value pair or update existingd["grade"] = "A"
RetrieveLook up a value by its keyd["name"] returns "Alice"
DeleteRemove a key-value pairdel d["age"]
Check existenceTest if a key is in the dictionary"name" in d returns True
IterateLoop through keys, values, or bothfor key, val in d.items()

Implementation

Dictionaries are most commonly implemented using hash tables, which is why lookup, insertion, and deletion are all O(1) on average. The key is hashed to determine where the value is stored.

students = {
    "S001": {"name": "Alice", "grade": 92},
    "S002": {"name": "Bob", "grade": 85},
    "S003": {"name": "Charlie", "grade": 78}
}

# Access nested data
print(students["S001"]["name"])     # Alice

# Add a new student
students["S004"] = {"name": "Diana", "grade": 95}

# Check if a student exists
if "S002" in students:
    print(students["S002"]["grade"])  # 85

Dictionaries vs Other Structures

The AQA spec treats dictionaries as an ADT. You should understand that a dictionary is conceptually a collection of key-value pairs with certain operations, and that a hash table is one way to implement it. Be prepared to trace dictionary operations and explain why hash-based dictionaries offer O(1) average lookup.
