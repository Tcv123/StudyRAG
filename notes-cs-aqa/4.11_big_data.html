4.11.1 What is Big Data?

Big Data refers to datasets that are so large, fast-moving, or complex that traditional data-processing tools and relational databases are inadequate to store, manage, and analyse them effectively.

The Four Vs of Big Data

VMeaningExample
VolumeThe sheer amount of data generated. Can be terabytes, petabytes, or exabytes.Facebook generates hundreds of petabytes of user data per year. The Large Hadron Collider produces about 1 PB per second during experiments. YouTube receives 500+ hours of video uploads per minute.
VelocityThe speed at which new data is generated and must be processed. Some applications require real-time or near-real-time processing.Stock market transactions arrive in microseconds; live sensor feeds from IoT devices stream continuously; social media generates millions of posts per minute.
VarietyData comes in many different formats and from many sources, much of it unstructured.Text, images, video, audio, sensor readings, GPS coordinates, social media posts, medical scans, log files, JSON/XML documents, clickstream data.
VeracityThe uncertainty and trustworthiness of the data. Noisy, incomplete, biased, or inaccurate data reduces reliability of analysis.Social media sentiment may be sarcastic, bot-generated, or misleading. Sensor data may have calibration errors. Survey responses may be dishonest.

Some frameworks add a fifth V: Value — the usefulness and business insight that can be extracted from the data. Raw data has no value until it is processed into actionable information.

Types of Data

TypeDefinitionExamplesStorage
StructuredData organised in a fixed schema with clearly defined fields and data types. Easily stored in relational database tables with rows and columns.SQL databases, CSV files, spreadsheets, transaction records, sensor readings with fixed fieldsRelational databases (SQL)
UnstructuredData with no predefined schema or internal organisation. Cannot easily be stored in traditional rows and columns. Makes up approximately 80-90% of all data generated.Emails, free-form text documents, images, videos, audio recordings, social media posts, satellite imageryNoSQL databases, object storage, data lakes

Big Data questions often ask you to identify which of the four Vs is relevant to a given scenario. Read the scenario carefully: does it emphasise the amount of data (Volume), the speed (Velocity), the different formats (Variety), or the reliability/quality (Veracity)? Link your answer explicitly to the correct V with justification.

4.11.2 Machine Learning

Machine learning (ML) is a branch of artificial intelligence in which a computer system learns patterns from data and improves its performance on a task without being explicitly programmed with rules. Instead of writing rules by hand, the programmer provides data and the algorithm discovers the rules.

Types of Machine Learning

TypeHow It WorksData RequiredExample
Supervised learningThe model is trained on a labelled dataset — each input has a known correct output (the "ground truth"). The model learns to map inputs to outputs by minimising prediction error.Labelled data (expensive to create; requires human annotation)Classifying emails as spam/not-spam using thousands of previously labelled emails. Predicting house prices from features (size, location, bedrooms) using historical sales data.
Unsupervised learningThe model is given an unlabelled dataset and must discover hidden patterns, structures, or groupings on its own, with no correct answers provided.Unlabelled data (cheaper and more abundant)Customer segmentation: grouping shoppers by purchasing behaviour without pre-defined categories. Anomaly detection in network traffic.
Reinforcement learningAn agent interacts with an environment, taking actions and receiving rewards (positive feedback) or penalties (negative feedback). It learns a strategy (policy) to maximise cumulative reward through trial and error.No pre-labelled data; learns from experience (reward signal)Training an AI to play chess or Go — reward for winning, penalty for losing. Training a robot to walk. Optimising data centre cooling (Google DeepMind).

Classification and Clustering

TechniqueTypeWhat It DoesExample
ClassificationSupervisedAssigns each input to one of a set of predefined categories (labels). The model is trained on labelled examples and then predicts labels for new, unseen data.Classifying a tumour as benign or malignant based on measured features. Categorising support tickets by urgency. Identifying handwritten digits (0-9).
ClusteringUnsupervisedGroups data points so that items in the same cluster are more similar to each other than to those in other clusters. No labels are provided or predicted — the algorithm discovers natural groupings.Grouping news articles by topic without knowing the topics in advance. Segmenting customers by behaviour. Identifying communities in social networks.
Classification uses labelled data (supervised) — the categories are known in advance. Clustering uses unlabelled data (unsupervised) — the groups are discovered. This is a very common distinction in exam questions. If the question says "using historical data with known outcomes", it is supervised/classification. If it says "finding natural groups in data", it is unsupervised/clustering.

4.11.3 Neural Networks and Deep Learning

Structure of a Neural Network

An artificial neural network (ANN) is a computational model inspired by the structure of biological neurons. It consists of interconnected nodes (neurons) arranged in layers:

  • Input layer — receives the raw input data. Each node represents one feature (e.g., a pixel value, a temperature reading, a word embedding). The number of input nodes matches the number of features.
  • Hidden layer(s) — one or more intermediate layers that process inputs. Each node applies a mathematical function to a weighted sum of its inputs, then passes the result through an activation function.
  • Output layer — produces the final result. For classification, each output node typically represents one class, and the node with the highest value is the predicted class.

How a Single Neuron Works

Each connection between nodes has a weight (w) that determines how much influence one node has on the next. Each node also has a bias (b), an additional constant that shifts the activation threshold.

output = activation_function( w1*x1 + w2*x2 + ... + wn*xn + bias )

Example with 3 inputs:
  Inputs:  x1=0.5, x2=0.8, x3=0.2
  Weights: w1=0.4, w2=-0.3, w3=0.9
  Bias:    b=0.1

  Weighted sum = (0.5*0.4) + (0.8*-0.3) + (0.2*0.9) + 0.1
               = 0.2 + (-0.24) + 0.18 + 0.1
               = 0.24

  Apply activation function (e.g., ReLU):
  ReLU(0.24) = max(0, 0.24) = 0.24  (output of this neuron)

Common activation functions:

  • ReLU (Rectified Linear Unit): f(x) = max(0, x) — outputs x if positive, 0 if negative. Most commonly used in hidden layers because it is fast to compute and avoids the vanishing gradient problem.
  • Sigmoid: f(x) = 1 / (1 + e^(-x)) — squashes output to range (0, 1). Used for binary classification output.
  • Softmax: Converts a vector of values into a probability distribution that sums to 1. Used for multi-class classification output (e.g., probabilities for digits 0-9).

Deep Learning

Deep learning uses neural networks with many hidden layers (hence "deep"). These deep networks can learn increasingly abstract and complex features at each successive layer, enabling them to solve problems like image recognition, speech processing, and language translation that were previously intractable.
  • Early layers might detect simple features (e.g., edges, corners in an image; individual phonemes in audio).
  • Middle layers combine these into more complex features (e.g., shapes, textures; syllables, words).
  • Later layers recognise high-level concepts (e.g., a face, a specific object; a sentence, an emotion).

Deep learning requires large amounts of training data and significant computational power (GPUs). It has driven recent breakthroughs in image recognition, natural language processing, game playing, protein structure prediction, and generative AI.

Backpropagation and Gradient Descent

Backpropagation is the algorithm used to train a neural network by adjusting weights to minimise prediction error:

  1. Forward pass — input data passes through the network layer by layer to produce a prediction (output).
  2. Calculate error (loss) — the difference between the predicted output and the actual correct output is measured using a loss function (e.g., mean squared error for regression, cross-entropy for classification).
  3. Backward pass — the error is propagated backwards through the network. The algorithm calculates how much each weight contributed to the error using the chain rule of calculus (computing partial derivatives / gradients).
  4. Update weights — each weight is adjusted in the direction that reduces the error, by an amount proportional to its gradient.

Gradient descent determines how much to adjust each weight. The learning rate controls the step size:

  • Too large → the model may overshoot the optimal weights, oscillate, or diverge (fail to converge).
  • Too small → training is extremely slow and may get stuck in a local minimum (a sub-optimal solution).
  • In practice, adaptive learning rate algorithms (Adam, RMSprop) are used to automatically adjust the learning rate during training.
-- Pseudocode for training a neural network:
for each epoch (full pass through all training data):
    for each training example (input, correct_output):
        predicted = forward_pass(network, input)
        loss = calculate_loss(predicted, correct_output)
        gradients = backward_pass(network, loss)
        update_weights(network, gradients, learning_rate)
    evaluate on validation set
    if validation loss is increasing: STOP (early stopping to prevent overfitting)
You do not need to perform the calculus, but you must be able to explain the conceptual process: forward pass produces output → error is calculated using a loss function → error is propagated backwards to compute gradients → weights are adjusted to reduce error → repeat for many epochs until the model converges.

Training, Validation, and Test Data

DatasetPurposeWhen UsedTypical Split
Training setThe model learns patterns from this data by adjusting weights during backpropagation.During training (every epoch)~60-80%
Validation setUsed to tune hyperparameters (learning rate, number of layers, number of nodes) and detect overfitting. The model does not learn from this data.During training (checked after each epoch)~10-20%

Overfitting and Underfitting

ProblemDescriptionSymptomsSolutions
Overfitting The model has learned the training data too well, including noise, outliers, and random fluctuations. It has essentially memorised the training data rather than learning general patterns. High training accuracy, low test/validation accuracy. Large gap between training and validation performance. Use more training data; simplify the model (fewer layers/nodes); use regularisation (L1/L2 penalties on weights); use dropout (randomly disable neurons during training); early stopping (stop training when validation loss starts increasing); data augmentation.
Underfitting The model is too simple to capture the underlying patterns in the data. It fails to learn even the training data adequately. Low training accuracy AND low test accuracy. The model performs poorly everywhere. Increase model complexity (more layers/nodes); train for more epochs; add more relevant features; reduce regularisation; use a more sophisticated model architecture.

4.11.4 MapReduce and Distributed Computing

Why Distributed Computing?

Big Data is often too large to be stored or processed on a single machine. Distributed computing spreads storage and processing across a cluster of many interconnected commodity machines (nodes), enabling:

  • Parallel processing of massive datasets (divide the work among hundreds or thousands of nodes).
  • Fault tolerance — if one node fails, others continue; data is replicated across nodes for redundancy.
  • Horizontal scalability — add more nodes as data grows (cheaper than buying a single supercomputer).

MapReduce

MapReduce is a programming model (developed by Google) for processing large datasets in parallel across a distributed cluster. It abstracts away the complexity of parallelism, distribution, fault tolerance, and load balancing, allowing the programmer to focus on the logic. It has two main phases: Map and Reduce.

Map phase:

  • The input data is divided into chunks (splits) and distributed across the cluster nodes.
  • Each node applies a map function independently to its chunk, producing a set of intermediate key-value pairs.
  • The map function is applied to each input record independently — this is why it parallelises so well.

Shuffle and Sort (intermediate step):

  • The system automatically groups all intermediate key-value pairs by key, sorting them, and distributing them so that all values for the same key go to the same reducer node.

Reduce phase:

  • Each reducer node applies a reduce function to all the values associated with each key, combining or aggregating them to produce the final output.

MapReduce Example: Word Count

Count the occurrences of each word across a massive text corpus distributed across many machines:

Input (split across 3 nodes):
  Node 1: "the cat sat on the mat"
  Node 2: "the dog sat on the log"
  Node 3: "the cat and the dog"

Map phase (each node processes its chunk independently):
  Node 1 emits: ("the",1) ("cat",1) ("sat",1) ("on",1) ("the",1) ("mat",1)
  Node 2 emits: ("the",1) ("dog",1) ("sat",1) ("on",1) ("the",1) ("log",1)
  Node 3 emits: ("the",1) ("cat",1) ("and",1) ("the",1) ("dog",1)

Shuffle and sort (group by key, distribute to reducers):
  Reducer A gets: "and" -> [1]    "cat" -> [1, 1]    "dog" -> [1, 1]
  Reducer B gets: "log" -> [1]    "mat" -> [1]       "on"  -> [1, 1]
  Reducer C gets: "sat" -> [1, 1] "the" -> [1, 1, 1, 1, 1, 1]

Reduce phase (sum the values for each key):
  "and" -> 1     "cat" -> 2     "dog" -> 2
  "log" -> 1     "mat" -> 1     "on"  -> 2
  "sat" -> 2     "the" -> 6

Hadoop and HDFS

Apache Hadoop is an open-source framework for distributed storage and processing of Big Data:

  • HDFS (Hadoop Distributed File System) — stores data by splitting files into large blocks (typically 128 MB each) and replicating each block across multiple nodes (default 3 replicas) for fault tolerance. Architecture: a NameNode (master, stores metadata — which blocks belong to which file and where they are stored) and multiple DataNodes (workers, store actual data blocks).
  • MapReduce engine (YARN) — manages the execution of Map and Reduce tasks across the cluster, scheduling them on nodes where the data is already stored ("move computation to data" rather than moving data to computation — minimises network transfer).
When explaining MapReduce, always describe both the Map phase (applies a function to each input, producing key-value pairs) and the Reduce phase (aggregates values by key). Including a brief worked example demonstrates clear understanding. Remember: the power of MapReduce comes from the fact that the map function is applied independently to each data chunk, allowing massive parallelism.

4.11.5 Data Mining and Predictive Analytics

Data mining is the process of discovering patterns, correlations, trends, and anomalies within large datasets using statistical, mathematical, and computational techniques to extract useful knowledge and predict outcomes.

Common data mining techniques:

  • Association rule mining — finding items that frequently co-occur (e.g., "customers who buy bread also buy butter" — used for product recommendations and store layout optimisation).
  • Classification — predicting a categorical label for new data based on learned patterns (e.g., spam detection, medical diagnosis).
  • Clustering — grouping similar data points without predefined labels (e.g., customer segmentation, document categorisation).
  • Anomaly detection — identifying unusual data points that deviate from normal patterns (e.g., fraud detection in banking, intrusion detection in networks, fault detection in manufacturing).
  • Regression — predicting a continuous numerical value (e.g., predicting house prices, stock prices, energy consumption).

Predictive Analytics

Predictive analytics uses historical data, statistical algorithms, and machine learning to forecast future outcomes. It goes beyond describing what happened to predicting what will happen next.

Applications include: weather forecasting; medical diagnosis and prognosis; customer churn prediction; demand forecasting in supply chains; predictive maintenance (predicting when industrial equipment will fail before it does); credit scoring; targeted advertising.

4.11.6 Natural Language Processing (NLP)

Natural Language Processing (NLP) is a field of AI concerned with enabling computers to understand, interpret, and generate human language in a meaningful and useful way.

Key NLP tasks:

TaskDescriptionApplication
TokenisationSplitting text into individual words, subwords, or tokensFirst step in almost all NLP pipelines
Sentiment analysisDetermining whether text expresses positive, negative, or neutral opinionBrand monitoring, product review analysis, social media tracking
Named Entity Recognition (NER)Identifying and classifying entities (people, organisations, locations, dates, monetary values) in textInformation extraction, news analysis, legal document processing
Machine translationAutomatically translating text from one language to anotherGoogle Translate, DeepL, real-time subtitle translation
Speech recognitionConverting spoken language into text (speech-to-text)Voice assistants (Siri, Alexa), dictation software, accessibility tools
Text generationProducing human-like text given a prompt or contextChatbots, content creation, code generation, summarisation
Text classificationCategorising text documents into predefined categoriesSpam filtering, topic classification, intent detection in customer service

NLP is challenging because human language is inherently ambiguous (words have multiple meanings), context-dependent (meaning changes based on surrounding words), and full of idioms, sarcasm, implied meaning, and cultural references that are difficult for algorithms to interpret correctly.

Ethics questions are common and carry significant marks. Always: (1) consider multiple perspectives (individual, business, society, government), (2) reference specific laws or regulations (Data Protection Act 2018, UK GDPR) where relevant, (3) use concrete real-world examples, and (4) discuss both benefits and risks of Big Data use. A balanced argument with specific examples scores much higher than a one-sided general answer.
