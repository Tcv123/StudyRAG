4.11.1 What is Big Data?

Big Data refers to datasets that are so large, fast-moving or complex that traditional data-processing tools and techniques are inadequate to store, manage and analyse them.

The Four Vs of Big Data

VMeaningExample
VolumeThe sheer amount of data generated. Can be terabytes, petabytes or more.Facebook generates hundreds of petabytes of user data per year.
VelocityThe speed at which new data is generated and must be processed.Stock market transactions arrive in milliseconds; live sensor feeds stream continuously.
VarietyData comes in many different formats and from many sources.Text, images, video, sensor readings, GPS coordinates, social media posts.
VeracityThe uncertainty and trustworthiness of the data. Noisy, incomplete or biased data reduces reliability.Social media sentiment data may be sarcastic, bot-generated or misleading.

Types of Data

TypeDefinitionExamples
StructuredData organised in a fixed schema with clearly defined fields and types. Easily stored in relational database tables.SQL databases, CSV files, spreadsheets.
UnstructuredData with no predefined schema or organisation. Cannot easily be stored in rows and columns.Emails, images, videos, audio recordings, free-form text.
Semi-structuredData that has some organisational markers (tags, keys) but does not follow a rigid table structure.JSON, XML, HTML, NoSQL documents.
Big Data questions often ask you to identify which of the four Vs is relevant to a given scenario. Read the scenario carefully and link your answer explicitly to Volume, Velocity, Variety or Veracity.

4.11.2 Machine Learning

Machine learning (ML) is a branch of artificial intelligence in which a computer system learns patterns from data and improves its performance on a task without being explicitly programmed with rules for that task.

Types of Machine Learning

TypeHow It WorksExample
Supervised learningThe model is trained on a labelled dataset — each input has a known correct output. The model learns to map inputs to outputs.Classifying emails as spam or not spam using a dataset of previously labelled emails.
Unsupervised learningThe model is given an unlabelled dataset and must discover hidden patterns or groupings on its own, with no correct answers provided.Customer segmentation: grouping shoppers by purchasing behaviour without pre-set categories.
Reinforcement learningAn agent interacts with an environment, taking actions and receiving rewards or penalties. It learns a policy to maximise cumulative reward through trial and error.Training an AI to play chess — it receives a reward for winning and a penalty for losing.

Classification and Clustering

  • Classification (supervised) — assigning an input to one of a set of predefined categories. The model is trained on labelled examples. E.g., classifying a tumour as benign or malignant based on measured features.
  • Clustering (unsupervised) — grouping data points so that items in the same cluster are more similar to each other than to those in other clusters. No labels are provided. E.g., grouping news articles by topic without knowing the topics in advance.
Classification uses labelled data (supervised). Clustering uses unlabelled data (unsupervised). This is a very common distinction in exam questions.

4.11.3 Neural Networks and Deep Learning

Structure of a Neural Network

An artificial neural network (ANN) is a computational model inspired by the structure of biological neurons in the brain. It consists of interconnected nodes (neurons) arranged in layers:

  • Input layer — receives the raw input data. Each node represents one feature (e.g., a pixel value, a temperature reading).
  • Hidden layer(s) — one or more intermediate layers that process the inputs. Each node applies a mathematical function to a weighted sum of its inputs.
  • Output layer — produces the final result (e.g., a classification label, a predicted value).

Weights and Biases

Each connection between nodes has a weight, which determines how much influence one node has on the next. Each node also has a bias, an additional constant that shifts the activation threshold. During training, the network adjusts weights and biases to minimise prediction error.

The calculation at each node can be expressed as:

output = activation_function( w1*x1 + w2*x2 + ... + wn*xn + bias )

Common activation functions include the sigmoid function, ReLU (Rectified Linear Unit) and softmax (for multi-class classification).

Deep Learning

Deep learning uses neural networks with many hidden layers (hence "deep"). These deep networks can learn increasingly abstract and complex features at each successive layer, enabling them to solve problems like image recognition, speech processing and language translation.
  • Early layers might detect simple features (e.g., edges in an image).
  • Middle layers combine these into more complex features (e.g., shapes, textures).
  • Later layers recognise high-level concepts (e.g., a face, a specific object).

Backpropagation and Gradient Descent

Backpropagation is the algorithm used to train a neural network. It works by:

  1. Forward pass — input data passes through the network to produce an output (prediction).
  2. Calculate error (loss) — the difference between the predicted output and the actual correct output is measured using a loss function.
  3. Backward pass — the error is propagated backwards through the network. The algorithm calculates how much each weight contributed to the error (using the chain rule of calculus to find partial derivatives).
  4. Update weights — each weight is adjusted in the direction that reduces the error.

Gradient descent is the optimisation algorithm that determines how much to adjust each weight. It moves each weight in the direction of the steepest decrease in the loss function. The learning rate controls the size of each step:

  • Too large → the model may overshoot the optimal weights and fail to converge.
  • Too small → training is extremely slow and may get stuck in a local minimum.
You do not need to perform the calculus, but you must be able to explain the conceptual process: forward pass produces output, error is calculated, error is propagated back, weights are updated to reduce error.

Training, Test and Validation Data

DatasetPurposeTypical Split
Training setUsed to train the model — the model learns patterns from this data by adjusting weights.~60-80%
Validation setUsed during training to tune hyperparameters (e.g., learning rate, number of layers) and detect overfitting. The model does not learn from this data.~10-20%
Test setUsed only after training is complete to evaluate final model performance on unseen data. Gives an unbiased estimate of real-world accuracy.~10-20%

Overfitting and Underfitting

ProblemDescriptionSymptomsSolutions
Overfitting The model has learned the training data too well, including noise and random fluctuations. It performs excellently on training data but poorly on unseen data. High training accuracy, low test/validation accuracy. Use more training data; simplify the model (fewer layers/nodes); use regularisation; use dropout; stop training earlier (early stopping).
Underfitting The model is too simple to capture the underlying patterns in the data. It performs poorly on both training and test data. Low training accuracy, low test accuracy. Increase model complexity (more layers/nodes); train for more epochs; add more relevant features; reduce regularisation.

4.11.4 MapReduce and Distributed Computing

Why Distributed Computing?

Big Data is often too large to be stored or processed on a single machine. Distributed computing spreads storage and processing across a cluster of many interconnected machines (nodes), enabling:

  • Parallel processing of massive datasets.
  • Fault tolerance — if one node fails, others continue.
  • Horizontal scalability — add more nodes as data grows.

MapReduce

MapReduce is a programming model for processing large datasets in parallel across a distributed cluster. It has two main phases: Map and Reduce.

Map phase:

  • The input data is divided into chunks and distributed across the cluster nodes.
  • Each node applies a map function to its chunk of data, producing a set of intermediate key-value pairs.

Shuffle and Sort (intermediate step):

  • The intermediate key-value pairs are grouped together by key and distributed so that all values for the same key go to the same node.

Reduce phase:

  • Each node applies a reduce function to the grouped key-value pairs, combining or aggregating the values for each key to produce the final output.

MapReduce Example: Word Count

Count the occurrences of each word in a large text corpus:

Input:  "the cat sat on the mat"

Map phase (each word emits a key-value pair):
  ("the", 1), ("cat", 1), ("sat", 1),
  ("on", 1), ("the", 1), ("mat", 1)

Shuffle and sort (group by key):
  "cat"  -> [1]
  "mat"  -> [1]
  "on"   -> [1]
  "sat"  -> [1]
  "the"  -> [1, 1]

Reduce phase (sum the values for each key):
  "cat"  -> 1
  "mat"  -> 1
  "on"   -> 1
  "sat"  -> 1
  "the"  -> 2

Hadoop and HDFS

Apache Hadoop is an open-source framework for distributed storage and processing of Big Data. Its two core components are:

  • HDFS (Hadoop Distributed File System) — stores data across a cluster by splitting files into large blocks (typically 128 MB) and replicating each block across multiple nodes (default 3 copies) for fault tolerance. It follows a master-slave architecture with a NameNode (metadata) and multiple DataNodes (storage).
  • MapReduce engine — manages the execution of Map and Reduce tasks across the cluster nodes where the data is stored, minimising data transfer across the network ("move computation to data").
When explaining MapReduce in the exam, always describe both the Map phase (produces key-value pairs) and the Reduce phase (aggregates values by key). Including a brief example demonstrates clear understanding.

4.11.5 Data Mining and Predictive Analytics

Data mining is the process of discovering patterns, correlations and anomalies within large datasets using statistical, mathematical and computational techniques to predict outcomes and extract useful knowledge.

Common data mining techniques include:

  • Association rule mining — finding items that frequently occur together (e.g., customers who buy bread also buy butter).
  • Classification — predicting a categorical label for new data based on learned patterns.
  • Clustering — grouping similar data points without predefined labels.
  • Anomaly detection — identifying unusual data points (e.g., fraud detection).
  • Regression — predicting a continuous numerical value (e.g., house prices).

Predictive Analytics

Predictive analytics uses historical data, statistical algorithms and machine learning to forecast future outcomes. Applications include:

  • Weather forecasting.
  • Stock market predictions.
  • Medical diagnosis and prognosis.
  • Customer churn prediction (which customers are likely to leave).
  • Demand forecasting in supply chains.

Recommendation Systems

Recommendation systems suggest items to users based on their preferences or behaviour. Two main approaches:

  • Collaborative filtering — recommends items based on what similar users liked. "Users who liked X also liked Y." Does not require knowledge of item features, but needs a large number of user ratings.
  • Content-based filtering — recommends items similar to what the user has liked before, based on item attributes/features. "You liked sci-fi films, so here are more sci-fi films."

Many real-world systems use a hybrid approach combining both methods.

4.11.6 Natural Language Processing (NLP)

Natural Language Processing (NLP) is a field of AI concerned with enabling computers to understand, interpret and generate human language in a meaningful way.

Key NLP tasks and concepts:

  • Tokenisation — splitting text into individual words or tokens.
  • Sentiment analysis — determining whether text expresses a positive, negative or neutral opinion.
  • Named entity recognition (NER) — identifying entities such as people, organisations, locations and dates in text.
  • Machine translation — automatically translating text from one language to another.
  • Speech recognition — converting spoken language into text.
  • Text generation — producing human-like text (e.g., chatbots, summarisation).

NLP is challenging because human language is inherently ambiguous, context-dependent, and full of idioms, sarcasm and implied meaning.

4.11.7 Ethical and Social Implications of Big Data

The collection and analysis of Big Data raises significant ethical, legal and social concerns:

IssueDetail
PrivacyLarge-scale data collection (location tracking, browsing history, purchases) can reveal intimate details of individuals' lives, often without their informed consent. Data breaches can expose personal information.
SurveillanceGovernments and corporations can use Big Data for mass surveillance, monitoring citizens' communications, movements and activities.
Bias and discriminationIf training data reflects historical biases (e.g., racial, gender), machine learning models will perpetuate and amplify those biases in their predictions and decisions (e.g., biased hiring algorithms, unfair credit scoring).
Consent and transparencyUsers often do not understand what data is collected about them or how it is used. "Terms and conditions" are rarely read. Algorithms that affect people's lives (loan decisions, parole recommendations) are often opaque ("black box").
Data ownershipDisputes over who owns data — the individual who generated it, or the company that collected it. The right to be forgotten (GDPR) allows individuals to request deletion of their data.
SecurityStoring large volumes of personal data creates an attractive target for hackers. Organisations have a responsibility to protect data, but breaches remain common.
Job displacementAutomation and AI powered by Big Data may replace jobs in sectors such as manufacturing, customer service, transport and data analysis.
Filter bubblesRecommendation algorithms can create "echo chambers" where users are only shown content that reinforces their existing views, limiting exposure to diverse perspectives.
ProfilingCombining multiple data sources can create detailed profiles of individuals, enabling targeted advertising, price discrimination or social scoring.
Ethics questions are common in the exam. Always consider multiple perspectives (individual, business, society) and reference specific laws or regulations (e.g., Data Protection Act 2018, UK GDPR) where appropriate. Use concrete examples to support your arguments.
