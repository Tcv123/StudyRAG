4.11.1 What is Big Data?

Big Data refers to datasets so large, fast-moving, or complex that traditional data-processing tools and relational databases are inadequate to store, manage, and analyse them.

The Four Vs of Big Data

VMeaningExample
VolumeSheer amount of data. Terabytes to exabytes.Facebook: hundreds of PB/year. YouTube: 500+ hours uploaded per minute. LHC: ~1 PB/second during experiments.
VelocitySpeed of data generation and required processing speed. Some applications need real-time analysis.Stock trades in microseconds. Live IoT sensor streams. Millions of social media posts per minute.
VarietyData in many formats from many sources, much of it unstructured.Text, images, video, audio, GPS, sensor readings, log files, JSON, medical scans, clickstreams.
VeracityTrustworthiness and quality of data. Noisy, incomplete, biased data reduces reliability.Sarcastic social media posts. Bot-generated content. Sensor calibration errors. Dishonest survey responses.

Some frameworks add Value — the usefulness and business insight extractable from data. Raw data has no value until processed into actionable information.

4.11.2 Machine Learning

Machine learning (ML): A branch of AI where systems learn patterns from data and improve without being explicitly programmed with rules. The programmer provides data; the algorithm discovers the rules.

Types of Machine Learning

TypeHow It WorksDataExample
SupervisedTrained on labelled data (input + known correct output). Learns input-to-output mapping.Labelled (expensive to create)Spam classification using labelled emails. House price prediction from features.
UnsupervisedGiven unlabelled data. Discovers hidden patterns, groups, or structures.Unlabelled (abundant, cheap)Customer segmentation by purchase behaviour. Anomaly detection in network traffic.
ReinforcementAgent takes actions in environment, receives rewards/penalties. Learns policy to maximise reward.No labels; learns from experienceAI playing chess (reward for winning). Robot learning to walk. Data centre cooling optimisation.

Classification vs Clustering

TechniqueTypeWhat It DoesExample
ClassificationSupervisedAssigns inputs to predefined categories (labels known in advance)Tumour: benign or malignant. Handwritten digit recognition (0-9).
ClusteringUnsupervisedGroups data so items in same cluster are more similar to each other. No predefined labels.Grouping news articles by topic. Customer behaviour segments.
Classification = labelled data, categories known. Clustering = unlabelled data, groups discovered. If the question says "using historical data with known outcomes" = supervised/classification. "Finding natural groups" = unsupervised/clustering.

4.11.3 Neural Networks and Deep Learning

Structure of a Neural Network

An artificial neural network (ANN) is inspired by biological neurons. Interconnected nodes in layers:

  • Input layer — receives raw input. One node per feature.
  • Hidden layer(s) — process inputs. Each node computes a weighted sum + bias, then applies an activation function.
  • Output layer — produces the result. For classification, each node represents one class.

How a Single Neuron Works

output = activation_function(w1*x1 + w2*x2 + ... + wn*xn + bias)

Example:
  Inputs:  x1=0.5, x2=0.8, x3=0.2
  Weights: w1=0.4, w2=-0.3, w3=0.9
  Bias:    b=0.1

  Weighted sum = (0.5*0.4) + (0.8*-0.3) + (0.2*0.9) + 0.1
               = 0.2 - 0.24 + 0.18 + 0.1 = 0.24

  If using step activation: output = 1 (since 0.24 > 0)
  If using sigmoid: output = 1/(1+e^-0.24) = 0.56

Training (Backpropagation)

  1. Forward pass: input flows through network, producing a prediction
  2. Calculate error: compare prediction to actual label (loss function)
  3. Backward pass: propagate error backwards, calculating how much each weight contributed
  4. Update weights: adjust weights to reduce error (gradient descent)
  5. Repeat over many epochs (passes through the training data) until error is minimised

Deep Learning

Deep learning uses neural networks with many hidden layers (hence "deep"). Each layer learns increasingly abstract features. Deep learning powers image recognition, natural language processing, autonomous vehicles, and generative AI.

Overfitting: The model performs well on training data but poorly on unseen data because it has memorised noise rather than learned general patterns. Prevented by: more training data, regularisation, dropout, cross-validation.

4.11.4 Big Data Processing

Distributed Computing

Distributed computing splits a large task across multiple computers (nodes) that work in parallel. Necessary because big data cannot be processed on a single machine within a reasonable time.

MapReduce

A programming model for processing large datasets in parallel across a cluster. Two phases:

  1. Map phase: Each node processes a portion of the data and produces intermediate key-value pairs. Work is distributed across many nodes.
  2. Reduce phase: Intermediate results with the same key are combined (aggregated) to produce the final output.
-- Example: counting word frequencies in a large text corpus

MAP phase (run on each node with a portion of the text):
  Input:  "the cat sat on the mat"
  Output: (the, 1), (cat, 1), (sat, 1), (on, 1), (the, 1), (mat, 1)

SHUFFLE (group by key):
  (the, [1, 1]), (cat, [1]), (sat, [1]), (on, [1]), (mat, [1])

REDUCE phase (aggregate each key):
  (the, 2), (cat, 1), (sat, 1), (on, 1), (mat, 1)

Key advantage: Each Map task is independent, enabling massive parallelism. The Reduce phase aggregates results. This scales to thousands of nodes processing petabytes of data.

Hadoop Ecosystem

  • Hadoop — open-source framework for distributed storage and processing using MapReduce
  • HDFS (Hadoop Distributed File System) — stores files across a cluster with replication for fault tolerance
  • Apache Spark — faster alternative to MapReduce that processes data in memory rather than on disk

4.11.5 Data Mining

Data mining: The process of discovering patterns, correlations, anomalies, and trends in large datasets using statistical, mathematical, and computational techniques. It extracts useful knowledge from data that would be impossible to find manually.

Techniques:

  • Association rule mining — finding items that frequently occur together ("customers who buy X also buy Y"). Used in market basket analysis and recommendation systems.
  • Classification — predicting categorical outcomes from labelled training data
  • Clustering — discovering natural groupings in data
  • Regression — predicting numerical values (e.g., forecasting sales figures)
  • Anomaly detection — identifying unusual data points (fraud detection, intrusion detection)
  • Sentiment analysis — determining the emotional tone of text (positive, negative, neutral)

BASE (Basically Available, Soft state, Eventually consistent): The consistency model used by many NoSQL/big data systems. Sacrifices immediate consistency for availability and partition tolerance. Contrast with ACID which guarantees strict consistency.
CAP Theorem: A distributed system can guarantee at most two of three properties simultaneously: Consistency (all nodes see the same data), Availability (every request receives a response), Partition tolerance (system works despite network failures). Big data systems typically choose availability + partition tolerance (AP), sacrificing strict consistency.

4.11.8 Graph Schema vs Relational Schema

Relational Schema

Data is stored in tables with fixed columns. Relationships are expressed through foreign keys and joins. Works well for structured, consistent data.

-- Relational: Friend relationships require a join table
Person (PersonID PK, Name, Age)
Friendship (PersonID1 FK, PersonID2 FK, Since)

-- Query: find all of Alice's friends
SELECT p.Name
FROM Person p
INNER JOIN Friendship f ON p.PersonID = f.PersonID2
WHERE f.PersonID1 = (SELECT PersonID FROM Person WHERE Name = 'Alice');

-- For 6 degrees of separation queries, this requires complex recursive joins
-- — relational databases struggle with highly connected graph data

Graph Schema

Data is stored as nodes (entities) and edges (relationships). Relationships are first-class citizens — no join tables needed. Traversing relationships is fundamentally efficient.

Graph database: Stores data as nodes (entities) and edges (relationships), each with associated properties. Optimised for traversing relationships. Examples: Neo4j, Amazon Neptune, TigerGraph. 
-- Graph query (Cypher language, Neo4j):
-- Find Alice's friends
MATCH (alice:Person {name: 'Alice'})-[:FRIEND_OF]->(friend:Person)
RETURN friend.name;

-- Find friends of friends (2 hops) -- trivial in graph, complex in SQL
MATCH (alice:Person {name: 'Alice'})-[:FRIEND_OF*2]->(fof:Person)
RETURN fof.name;

-- Graph node: Person {name: 'Alice', age: 28}
-- Graph edge: [:FRIEND_OF {since: 2020}]
FeatureRelational (Table) SchemaGraph Schema
Data modelTables, rows, columnsNodes and edges with properties

Relationship traversalExpensive (multiple JOINs)Efficient (pointer-based)
Schema flexibilityRigid (all rows have same columns)Flexible (nodes can have different properties)

4.11.9 MapReduce — Extended Detail

Why MapReduce?

Traditional programs run on a single machine. Big data sets cannot fit on one machine or be processed in acceptable time by one CPU. MapReduce automatically distributes work across thousands of nodes in a cluster, handles node failures transparently, and aggregates results.

Detailed MapReduce Example: Log Analysis

Problem: Analyse web server logs (100 TB) to count requests per country.

INPUT (distributed across 1000 nodes, each with a portion of the logs):
  Node 1 chunk: "195.229.241.100 GET /index.html ... UK"
                "8.8.8.8 GET /about.html ... US"
                "195.229.241.200 GET /index.html ... UK"

MAP phase (each node processes its own chunk independently):
  Mapper output: key = country, value = 1
  ("UK", 1), ("US", 1), ("UK", 1)

SHUFFLE/SORT phase (framework groups all same-key pairs together):
  ("UK", [1, 1, 1, 1, ...])    -- all UK values from all nodes
  ("US", [1, 1, 1, ...])

REDUCE phase (one reducer per key, sums the values):
  ("UK", 58423921)
  ("US", 94821044)

OUTPUT: sorted count of requests by country
  -- Done in minutes across 1000 nodes vs weeks on one machine

Fault Tolerance in MapReduce

  • The master node (JobTracker in Hadoop) monitors all worker nodes
  • If a worker fails, its tasks are automatically reassigned to another node
  • Input data is stored in HDFS with multiple replicas (default: 3 copies) so data is not lost if a node fails
  • This enables commodity hardware (cheap, unreliable servers) to be used at scale — failures are expected and handled automatically

4.11.11 Hadoop Ecosystem — Extended

HDFS (Hadoop Distributed File System)

HDFS splits large files into blocks (default 128MB), distributes blocks across DataNodes, and replicates each block on 3 different nodes. A NameNode stores metadata (which blocks belong to which file, and where each block is). If a DataNode fails, another replica is used transparently. 
File: large_dataset.csv (500 MB)

HDFS splits into blocks:
  Block 1 (128MB) → stored on DataNode 1, 3, 7
  Block 2 (128MB) → stored on DataNode 2, 5, 9
  Block 3 (128MB) → stored on DataNode 4, 6, 8
  Block 4 (116MB) → stored on DataNode 1, 2, 10

NameNode knows: large_dataset.csv = [Block1, Block2, Block3, Block4]
DataNodes fail? Replication means data still accessible.
Map tasks scheduled close to the data (data locality → less network traffic).
