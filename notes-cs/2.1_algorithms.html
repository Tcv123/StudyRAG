2.1.1 Computational Thinking

Computational thinking is a structured problem-solving approach that involves breaking down complex problems into manageable parts, identifying the key information, and designing step-by-step solutions. It is not about thinking like a computer — it is about thinking in a way that allows a computer (or human) to solve the problem systematically.

Computational thinking involves three key techniques that are used together:

TechniqueDefinitionExample
DecompositionBreaking a complex problem into smaller, more manageable sub-problems that can be solved independently. Each sub-problem is simpler to understand, implement, test, and debug. Different team members can work on different sub-problems simultaneously.Building a game: decompose into sub-problems — graphics rendering, user input handling, scoring system, sound effects, level design, save/load functionality, multiplayer networking. Each can be developed and tested independently before being integrated.
AbstractionRemoving unnecessary detail and focusing only on the relevant information needed to solve the problem. Hides complexity behind a simpler representation. Identifies what IS important and filters out what is NOT.A London Underground map abstracts away the actual geographic positions of stations, the depth of tunnels, and the distances between stops. It shows only what a passenger needs: station names, lines, and connections. A weather forecast shows temperature and conditions, abstracting away complex atmospheric models.
Algorithmic thinkingDesigning a precise, unambiguous, step-by-step solution (algorithm) that can be followed mechanically to solve the problem. Steps must be in the correct order, cover all possible cases (including edge cases), and terminate (finish).Finding the largest number in a list: (1) Assume the first number is the largest. (2) Compare it to each subsequent number. (3) If a larger number is found, update the record. (4) After checking all numbers, the recorded value is the answer. This works for any list of any length.

Applying Computational Thinking — Worked Example

Problem: "Build a school registration system"

DECOMPOSITION — break into sub-problems:
  1. User login (authentication)
  2. Class list management (add/remove students)
  3. Taking the register (mark present/absent/late)
  4. Calculating attendance percentages
  5. Generating reports for parents/teachers
  6. Storing data in a database

ABSTRACTION — what details matter?
  Important: student names, dates, attendance status, class groups
  Not important: student height, favourite colour, shoe size
  We model only the information relevant to registration.

ALGORITHMIC THINKING — step-by-step solution for "calculate attendance":
  1. Count total number of school days in the period
  2. Count number of days the student was marked present
  3. Attendance % = (days present / total days) x 100
  4. If attendance < 90%, flag as concern
  5. Display result

2.1.3 Searching Algorithms

Linear Search

Linear search checks each item in the list one by one from the start until the target is found or the end of the list is reached. It works on both sorted and unsorted data. It is the simplest search algorithm. 
// OCR Pseudocode:
function linearSearch(list, target)
    for i = 0 to length(list) - 1
        if list[i] == target then
            return i           // found at index i
        endif
    next i
    return -1                  // not found (searched entire list)
endfunction

# Python:
def linear_search(data, target):
    for i in range(len(data)):
        if data[i] == target:
            return i
    return -1

Binary Search

Binary search repeatedly halves the search space by comparing the target to the middle element. If the target is smaller, search the left half; if larger, search the right half. The data must be sorted first. Each comparison eliminates half of the remaining elements. 
// OCR Pseudocode:
function binarySearch(list, target)
    low = 0
    high = length(list) - 1
    while low <= high
        mid = (low + high) DIV 2
        if list[mid] == target then
            return mid         // found
        elseif list[mid] < target then
            low = mid + 1      // target is in right half
        else
            high = mid - 1     // target is in left half
        endif
    endwhile
    return -1                  // not found
endfunction

# Python:
def binary_search(data, target):
    low, high = 0, len(data) - 1
    while low <= high:
        mid = (low + high) // 2
        if data[mid] == target:
            return mid
        elif data[mid] < target:
            low = mid + 1
        else:
            high = mid - 1
    return -1

2.1.4 Sorting Algorithms

Bubble Sort

Bubble sort repeatedly compares adjacent pairs of elements and swaps them if they are in the wrong order. After each complete pass through the list, the next largest unsorted element "bubbles up" to its correct position at the end. Passes repeat until no swaps are made in a complete pass (meaning the list is fully sorted). 
// OCR Pseudocode:
procedure bubbleSort(list)
    n = length(list)
    do
        swapped = false
        for i = 0 to n - 2
            if list[i] > list[i+1] then
                // Swap the two elements
                temp = list[i]
                list[i] = list[i+1]
                list[i+1] = temp
                swapped = true
            endif
        next i
        n = n - 1    // last element is now in correct position
    until swapped == false
endprocedure

# Python:
def bubble_sort(data):
    n = len(data)
    while True:
        swapped = False
        for i in range(n - 1):
            if data[i] > data[i + 1]:
                data[i], data[i + 1] = data[i + 1], data[i]
                swapped = True
        n -= 1
        if not swapped:
            break

Pass 2 (ignore last position — 9 is sorted):
  Compare 3,5 → no swap
  Compare 5,1 → SWAP → [3, 1, 5, 8, 2, 9]
  Compare 5,8 → no swap

Pass 3 (ignore last 2 positions):
  Compare 3,1 → SWAP → [1, 3, 5, 2, 8, 9]

Merge Sort

Merge sort uses a divide and conquer strategy. It works in two phases: (1) Split — recursively divide the list in half until each sub-list has one element (a single element is trivially sorted). (2) Merge — repeatedly merge pairs of sub-lists back together in sorted order. It is much more efficient than bubble sort for large datasets. 
// OCR Pseudocode (simplified):
function mergeSort(list)
    if length(list) <= 1 then
        return list    // base case: single element is already sorted
    endif
    mid = length(list) DIV 2
    left = mergeSort(list[0..mid-1])     // sort left half
    right = mergeSort(list[mid..end])     // sort right half
    return merge(left, right)            // merge the two sorted halves
endfunction

# Python:
def merge_sort(data):
    if len(data) <= 1:
        return data
    mid = len(data) // 2
    left = merge_sort(data[:mid])
    right = merge_sort(data[mid:])
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result

Merge Sort — Complete Trace

Sorting: [5, 3, 8, 1, 9, 2]

SPLIT PHASE (divide until single elements):
  [5, 3, 8, 1, 9, 2]
  ├── [5, 3, 8]              [1, 9, 2]
  ├── [5] [3, 8]             [1] [9, 2]
  └── [5] [3] [8]            [1] [9] [2]

MERGE PHASE (merge pairs in sorted order):
  Merge [3] + [8] → compare 3,8 → [3, 8]
  Merge [5] + [3,8] → compare 5,3→take 3; compare 5,8→take 5; take 8 → [3, 5, 8]

  Merge [9] + [2] → compare 9,2 → [2, 9]
  Merge [1] + [2,9] → compare 1,2→take 1; take 2; take 9 → [1, 2, 9]

  Final merge [3,5,8] + [1,2,9]:
    Compare 3,1 → take 1 → [1]
    Compare 3,2 → take 2 → [1, 2]
    Compare 3,9 → take 3 → [1, 2, 3]
    Compare 5,9 → take 5 → [1, 2, 3, 5]
    Compare 8,9 → take 8 → [1, 2, 3, 5, 8]
    Take remaining 9 →       [1, 2, 3, 5, 8, 9]

  Result: [1, 2, 3, 5, 8, 9]

Insertion Sort

Insertion sort builds the sorted list one element at a time. It takes each element from the unsorted portion and inserts it into its correct position within the already-sorted portion, shifting elements to the right as needed. It works like sorting playing cards in your hand. 
// OCR Pseudocode:
procedure insertionSort(list)
    for i = 1 to length(list) - 1
        key = list[i]            // element to insert
        j = i - 1
        while j >= 0 AND list[j] > key
            list[j+1] = list[j]  // shift element right
            j = j - 1
        endwhile
        list[j+1] = key          // insert into correct position
    next i
endprocedure

# Python:
def insertion_sort(data):
    for i in range(1, len(data)):
        key = data[i]
        j = i - 1
        while j >= 0 and data[j] > key:
            data[j + 1] = data[j]
            j -= 1
        data[j + 1] = key

Insertion Sort — Complete Trace

Sorting: [5, 3, 8, 1, 9, 2]
         ↑ sorted portion starts with first element

Step 1: Insert 3 into sorted [5]
  3 < 5, shift 5 right, insert 3 at start
  Result: [3, 5, | 8, 1, 9, 2]

Step 2: Insert 8 into sorted [3, 5]
  8 > 5, so insert at end (no shifting needed)
  Result: [3, 5, 8, | 1, 9, 2]

Step 3: Insert 1 into sorted [3, 5, 8]
  1 < 8, shift 8; 1 < 5, shift 5; 1 < 3, shift 3; insert 1 at start
  Result: [1, 3, 5, 8, | 9, 2]

Step 4: Insert 9 into sorted [1, 3, 5, 8]
  9 > 8, insert at end
  Result: [1, 3, 5, 8, 9, | 2]

Step 5: Insert 2 into sorted [1, 3, 5, 8, 9]
  2 < 9, shift; 2 < 8, shift; 2 < 5, shift; 2 < 3, shift; 2 > 1, insert after 1
  Result: [1, 2, 3, 5, 8, 9]
