Casting (Type Conversion)

Casting converts a value from one data type to another. This is necessary when, for example, you read user input (which is always a string) and need to use it as a number.

// OCR Pseudocode / Python:
int("5")       // String "5" → Integer 5
str(42)        // Integer 42 → String "42"
float("3.14")  // String "3.14" → Real 3.14
int(7.9)       // Real 7.9 → Integer 7 (truncates, does not round)
bool(1)        // Integer 1 → Boolean True
bool(0)        // Integer 0 → Boolean False

// Common use: reading numeric input
age = int(input("Enter your age: "))   // input() returns a string; int() converts it

2.2.2 Variables, Constants, and Operators

Variables and Constants

  • Variable: A named storage location in memory whose value can change during program execution. E.g., score = 0 then later score = score + 10.
  • Constant: A named value that cannot change once set. Declared with const in OCR pseudocode. E.g., const TAX_RATE = 0.2. Using constants makes code more readable and easier to maintain (change the value in one place, not everywhere it is used).
  • Local variable: Exists only inside the subroutine where it is declared. Cannot be accessed from outside. Destroyed when the subroutine finishes.
  • Global variable: Accessible from anywhere in the program. Persists for the entire program's lifetime. Should be avoided where possible (harder to debug, risk of unintended modification).

2.2.4 Arrays (1D and 2D)

An array is a fixed-size, ordered collection of elements of the same data type, stored in contiguous memory. Elements are accessed by their index (position number). OCR arrays are zero-indexed — the first element is at index 0. 
// 1D Array
array scores[5]           // declare array of 5 elements
scores[0] = 95            // assign value to first element
scores[4] = 72            // assign value to last element (index 4, not 5!)

for i = 0 to 4            // loop through all elements
    print(scores[i])
next i

// 2D Array (table/grid — rows and columns)
array grid[3][4]           // 3 rows, 4 columns
grid[1][2] = 7             // row 1, column 2

// 2D array as a database table:
// Each row = a record; each column = a field
// students[0][0] = "Alice"  (row 0, name field)
// students[0][1] = 17       (row 0, age field)

// Nested loop to process all elements of a 2D array:
for row = 0 to 2
    for col = 0 to 3
        print(grid[row][col])
    next col
next row

2.2.6 File Handling, Records, SQL, and Random Numbers

File Handling

// Reading from a file
myFile = open("data.txt")          // open file
data = myFile.readLine()           // read one line
myFile.close()                     // always close the file

// Writing to a file
myFile = open("output.txt")
myFile.writeLine("Hello World")    // write a line
myFile.close()

// Python equivalent:
with open("data.txt", "r") as f:   // "r" = read mode
    for line in f:
        print(line.strip())

with open("output.txt", "w") as f: // "w" = write mode (overwrites)
    f.write("Hello World\n")

Records

A record is a collection of related fields of different data types grouped together (unlike an array, which stores elements of the same type).

record Student
    name : string
    age : integer
    grade : character
endrecord

// Create and use a record:
pupil1 = new Student()
pupil1.name = "Alice"
pupil1.age = 17
pupil1.grade = 'A'

SQL (Structured Query Language)

// SELECT: retrieve specific columns FROM a table WHERE conditions are met
SELECT name, age FROM students WHERE age > 16
SELECT * FROM products WHERE price < 10.00 AND category = 'Electronics'
SELECT name, score FROM results WHERE score >= 50 ORDER BY score DESC

// Key clauses:
// SELECT: which columns to return (* = all)
// FROM: which table to query
// WHERE: filter rows by condition
// ORDER BY: sort results (ASC = ascending, DESC = descending)

Random Numbers

x = random(1, 10)    // generates a random integer between 1 and 10 inclusive

// Python:
import random
x = random.randint(1, 10)    // random integer 1-10
y = random.random()          // random float 0.0 to 1.0

Parameters and Arguments

  • Parameter: The variable name in the subroutine definition (e.g., name in procedure greet(name)). It is a placeholder.
  • Argument: The actual value passed when the subroutine is called (e.g., "Jamie" in greet("Jamie")).

Benefits of Subroutines

  • Code reuse: Write once, call many times from different places. No copy-paste duplication.
  • Easier to debug: Each subroutine can be tested independently in isolation.
  • Improved readability: The main program reads like a high-level overview; details are in subroutines.
  • Team development: Different programmers can work on different subroutines simultaneously.
  • Easier maintenance: Changing the subroutine's implementation automatically updates everywhere it is called.
Procedure = no return value (it does something but gives nothing back). Function = returns a value (it calculates something and gives the result back to the caller). Arrays are zero-indexed (first element at index 0, last at index length-1). Use FOR when count is known; WHILE when it is not. SQL: SELECT picks columns, FROM names the table, WHERE filters rows.
