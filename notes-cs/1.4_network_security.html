1.4 Network Security

OCR GCSE Computer Science (J277) — Paper 1

Network security protects computer systems, networks, and data from unauthorised access, damage, and theft. Understanding threats and their countermeasures is essential for both the exam and real-world practice.

1.4.1 Forms of Attack

ThreatHow It WorksPurpose / ImpactReal-World Example
Malware Malicious software installed without the user's informed consent, typically via email attachments, infected downloads, compromised websites, or USB drives. Types include viruses (attach to files, spread when executed), worms (self-replicate across networks), trojans (disguised as legitimate software), spyware (monitors user activity), ransomware (encrypts files and demands payment), and adware (displays unwanted adverts). Corrupt/delete files; steal personal data (passwords, banking details); give the attacker remote control of the system; encrypt files for ransom; slow down the system; use the computer as part of a botnet. The WannaCry ransomware (2017) encrypted files on 200,000+ computers across 150 countries, including NHS hospitals, demanding Bitcoin payment for decryption.
Phishing Fraudulent emails, texts, or fake websites that impersonate legitimate organisations (banks, HMRC, Amazon, social media) to trick users into revealing sensitive information. Exploits human trust rather than technical vulnerabilities. Spear phishing targets specific individuals with personalised messages. Steal login credentials, credit card numbers, personal identity information, or trick users into installing malware. Can lead to identity theft, financial fraud, or account takeover. An email appearing to be from a bank says "Your account has been compromised. Click here to verify your identity" and links to a fake login page that captures the user's credentials.
Brute-force attack Automated software systematically tries every possible password combination until the correct one is found. Can also use dictionary attacks (trying common words/passwords) or credential stuffing (trying leaked username/password pairs from other breaches). Gain unauthorised access to accounts, systems, or encrypted data. A simple 4-digit PIN has only 10,000 combinations and can be cracked in seconds. An 8-character lowercase password has 26^8 (208 billion) combinations but can still be cracked in hours with modern hardware. A 16-character mixed password with symbols is practically unbreakable by brute force.
DoS / DDoS Denial of Service: floods a server with so many fake requests that it cannot respond to genuine users, effectively taking the service offline. Distributed DoS (DDoS): uses a botnet (thousands of compromised computers) to generate the flood from many sources simultaneously, making it much harder to block. Take websites/services offline; cause financial losses (e-commerce sites losing sales); damage reputation; used for extortion ("pay us or we will DDoS you again"). Major gaming platforms, banking websites, and government services have been targeted by DDoS attacks causing hours or days of downtime.
Data interception (Man-in-the-Middle) An attacker secretly intercepts data travelling across a network between two parties. The attacker can read, modify, or inject data without either party knowing. Common on unsecured public Wi-Fi networks. Steal login credentials, financial data, personal information, session tokens, or confidential business communications. An attacker on a coffee shop's open Wi-Fi intercepts unencrypted HTTP traffic and captures login cookies, gaining access to the victim's email account.
SQL injection Malicious SQL code is entered into a web form input field (e.g., login form, search box). If the web application does not properly sanitise (validate and escape) user input, the malicious SQL is executed by the database server. Bypass authentication (login without knowing the password); extract the entire database contents (usernames, passwords, personal data); modify or delete data; drop tables; gain administrative access. Entering ' OR '1'='1 into a login form's password field. If the application constructs the SQL query by concatenating user input without sanitisation, this makes the WHERE clause always true, bypassing the password check entirely. 
-- How SQL injection works:
-- The application constructs:
SELECT * FROM Users WHERE username='admin' AND password='[user input]';

-- If the user enters:  ' OR '1'='1
-- The query becomes:
SELECT * FROM Users WHERE username='admin' AND password='' OR '1'='1';
-- '1'='1' is always TRUE, so the WHERE clause is always TRUE
-- The attacker is logged in as admin without knowing the password!

-- Prevention: use PARAMETERISED QUERIES (prepared statements):
SELECT * FROM Users WHERE username=? AND password=?;
-- The ? placeholders ensure user input is treated as DATA, never as SQL code

1.4.2 Prevention Methods

PreventionWhat It Protects AgainstHow It Works
Penetration testingAll vulnerabilities (before attackers find them)Authorised ethical hackers (with written permission) attempt to break into systems using the same techniques as malicious hackers. Findings are documented and used to fix weaknesses before real attackers exploit them. Should be conducted regularly (at least annually).
Anti-malware softwareMalware infections (viruses, worms, trojans, ransomware, spyware)Scans files against a database of known malware signatures (patterns). Also uses heuristic analysis (detecting suspicious behaviour patterns even for unknown malware). Real-time protection monitors files as they are opened/downloaded. Must be kept updated — new malware is created daily.
FirewallUnauthorised network connectionsMonitors incoming and outgoing network traffic and allows or blocks connections based on predefined security rules (IP addresses, port numbers, protocols). Can be hardware (standalone device) or software (running on each computer). Blocks connections not matching the allowed criteria. Does NOT scan file contents — that is the anti-malware's job.
User access levelsDamage from compromised accounts; insider threatsPrinciple of least privilege: each user is given the minimum access rights needed to do their job. An office worker does not need admin access to the server. If their account is compromised, damage is limited to what that account can access. Roles: admin, standard user, guest.
Strong passwordsBrute-force attacks, dictionary attacks, guessingPasswords should be: long (12+ characters); complex (mix of uppercase, lowercase, digits, symbols); unique (different for every account); not based on personal information (names, birthdays, pet names). Changed regularly. Use a password manager to generate and store unique passwords.
EncryptionData interception and theftConverts plaintext into unreadable ciphertext using a cryptographic key. Only someone with the correct decryption key can read the data. Protects data in transit (HTTPS, VPN) and at rest (encrypted hard drives, encrypted databases). Even if intercepted, encrypted data is useless without the key.
Physical securityUnauthorised physical access to hardwareLocked server rooms with key card access; CCTV surveillance; biometric locks (fingerprint, retina); cable locks for laptops; visitor sign-in logs; security guards. Prevents someone from physically accessing a server, plugging in a USB drive, or stealing hardware.
Two-factor authentication (2FA)Account compromise even if password is stolenRequires two different types of evidence to log in: (1) something you know (password), (2) something you have (phone for SMS code or authenticator app), or (3) something you are (fingerprint, face recognition). Even if an attacker steals the password, they cannot log in without the second factor.
Input sanitisation / validationSQL injection, XSS, and other injection attacksAll user input is checked, validated, and sanitised (special characters are escaped or removed) before being used in database queries or displayed on web pages. Use parameterised queries / prepared statements for SQL. Never trust user input.
For each threat, know HOW it works AND its purpose/impact — these are separate marks. A firewall controls network traffic (allows/blocks connections); anti-malware scans files for malicious code — they are different and complementary. For prevention methods, always explain the mechanism (how does it actually prevent the attack?) not just name it. For SQL injection, be able to give a specific example of the malicious input and explain how parameterised queries prevent it.
