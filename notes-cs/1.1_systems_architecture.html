1.1.1 Von Neumann Architecture and CPU Components

The Von Neumann architecture is a computer design where data and instructions are stored in the same memory and are processed sequentially (one instruction at a time). This is the basis of virtually all modern general-purpose computers. The key insight is that programs are data — they can be loaded, modified, and replaced without physically rewiring the machine.

Key CPU Components

ComponentFunctionDetail
ALU (Arithmetic Logic Unit)Performs all arithmetic calculations and logical comparisonsArithmetic: add, subtract, multiply, divide. Logic: AND, OR, NOT, comparisons (equal, greater than, less than). Results are stored in the accumulator.
Control Unit (CU)Coordinates all CPU operationsFetches instructions from memory, decodes them into operations, and directs other components (ALU, registers, buses) to carry them out. It sends control signals via the control bus. The CU does not process data itself.
RegistersUltra-fast storage inside the CPUSmall number of storage locations that hold data, addresses, or instructions currently being used. Much faster than RAM (accessed in a single clock cycle). Each register typically holds one word of data (e.g., 32 or 64 bits).
CacheFast memory between CPU and RAMStores copies of frequently accessed data and instructions from RAM so the CPU does not have to wait for slower RAM. L1 cache is fastest and smallest (per core); L2 is larger but slightly slower; L3 is largest and shared between cores. Cache is made of SRAM (faster than the DRAM used in main memory).
ClockSynchronises all CPU operationsAn electronic oscillator that generates a regular pulse. Each pulse is one clock cycle, and one or more stages of an instruction happen per cycle. Clock speed is measured in GHz (billions of cycles per second). A 3.5 GHz processor performs 3.5 billion clock cycles every second.

Buses

The CPU communicates with memory and I/O devices via a system bus consisting of three sub-buses:

BusCarriesDirection
Address BusMemory addresses — tells memory which location to read from or write toUnidirectional (CPU → memory)
Data BusActual data being transferred between CPU, memory, and I/O devicesBidirectional
Control BusControl signals: read, write, clock, interrupt, bus request/grantBidirectional

1.1.2 Registers

RegisterFull NameHoldsPurpose During the FDE Cycle
PCProgram CounterAddressHolds the address of the NEXT instruction to be fetched. Incremented by 1 after each fetch (unless a branch instruction changes it to a different address).
MARMemory Address RegisterAddressHolds the memory address the CPU wants to read from or write to. Connected to the address bus. The MAR is like a request slip sent to memory: "give me the data at this location".
MDRMemory Data RegisterDataHolds the data just read from memory, or the data about to be written to memory. Connected to the data bus. Also called the Memory Buffer Register (MBR).
ACCAccumulatorDataStores the result of the most recent ALU operation (arithmetic or logic). Acts as the primary working register for calculations.
CIRCurrent Instruction RegisterData (instruction)Holds the instruction currently being decoded and executed. The CU splits its contents into the opcode (what to do) and the operand (the data or address to use).
Always distinguish between registers that hold addresses (PC, MAR) and those that hold data (MDR, ACC, CIR). The PC holds the address of the next instruction, not the current one. The MAR holds addresses; the MDR holds data. This distinction is required for full marks.

1.1.3 The Fetch-Decode-Execute (FDE) Cycle

The CPU continuously repeats three stages for every instruction. This cycle runs billions of times per second.

1. Fetch

  1. The address in the PC is copied to the MAR.
  2. The PC is incremented by 1 (PC = PC + 1), so it now points to the next instruction.
  3. The address in the MAR is sent along the address bus to memory.
  4. The CU sends a memory read signal on the control bus.
  5. The instruction at that memory address is transferred along the data bus into the MDR.
  6. The contents of the MDR are copied to the CIR.

Summary: PC → MAR → Memory → MDR → CIR. PC incremented.

2. Decode

  1. The Control Unit examines the instruction in the CIR.
  2. It splits the instruction into the opcode (the operation to perform, e.g., ADD, LOAD, STORE) and the operand (the data or address to use).
  3. The CU determines which control signals to send and to which components.

3. Execute

  1. The decoded instruction is carried out. This could be:
    • An arithmetic/logic operation (ALU performs a calculation, result goes to ACC)
    • A data transfer (load data from memory into a register, or store register data to memory)
    • A branch (load a new address into the PC, changing the sequence of execution)
  2. The result may be stored in the ACC or written back to memory via the MDR.

Worked Example

Memory:
  Address 0: LOAD 5      (load value at address 5 into ACC)
  Address 1: ADD 6       (add value at address 6 to ACC)
  Address 2: STORE 7     (store ACC into address 7)
  Address 5: 10           (data)
  Address 6: 25           (data)
  Address 7: 0            (will receive result)

Cycle 1 (LOAD 5):
  Fetch:   PC=0 → MAR=0, PC becomes 1. Memory[0] → MDR="LOAD 5" → CIR
  Decode:  Opcode=LOAD, Operand=address 5
  Execute: MAR=5, Memory[5] → MDR=10, MDR → ACC. ACC now holds 10.

Cycle 2 (ADD 6):
  Fetch:   PC=1 → MAR=1, PC becomes 2. Memory[1] → MDR="ADD 6" → CIR
  Decode:  Opcode=ADD, Operand=address 6
  Execute: MAR=6, Memory[6] → MDR=25. ALU: ACC + MDR = 10+25 = 35 → ACC.

Cycle 3 (STORE 7):
  Fetch:   PC=2 → MAR=2, PC becomes 3. Memory[2] → MDR="STORE 7" → CIR
  Decode:  Opcode=STORE, Operand=address 7
  Execute: MAR=7, ACC → MDR=35, MDR written to Memory[7]. Address 7 now holds 35.

1.1.4 CPU Performance Factors

FactorHow It Improves PerformanceLimitations
Clock SpeedMeasured in GHz. Higher clock speed = more FDE cycles per second. A 4 GHz processor does 4 billion cycles/second. Doubling clock speed roughly doubles throughput for single-threaded tasks.Higher clock speed generates more heat and uses more power. Physical limits exist (signal propagation, heat dissipation). Diminishing returns — modern CPUs hit thermal walls above ~5 GHz.
Number of CoresEach core is an independent processor with its own ALU, CU, and registers. A quad-core CPU can run 4 threads simultaneously. Good for multi-tasking (running multiple programs) and multi-threaded applications.Software must be written to use multiple threads to benefit. A single-threaded program uses only one core regardless. Not all tasks can be divided into parallel sub-tasks. Overhead from coordinating between cores.
Cache SizeCache stores frequently accessed data closer to the CPU. Larger cache = more data available without slow RAM access = fewer delays. L1 is fastest/smallest (~1 nanosecond); L3 is slowest/largest (~10-20 ns). RAM access takes ~50-100 ns.Cache is very expensive per byte (SRAM vs DRAM). Diminishing returns as cache grows larger (larger caches are slower). Takes up physical space on the CPU die.
When explaining CPU performance, always state how the factor helps AND its limitations. For example: "More cores allow multiple threads to execute simultaneously, improving performance for multi-threaded applications such as video editing. However, if the software is single-threaded, additional cores provide no benefit." This balanced answer scores higher than just stating the positive effect.

1.1.5 Embedded Systems

An embedded system is a computer built into a larger device to perform one specific dedicated function. The program is stored in ROM (or flash memory) and runs automatically when the device is powered on. Unlike a general-purpose computer, an embedded system is designed for a single task and typically has no user-accessible operating system.

Characteristics

  • Performs one specific dedicated function — it does not run arbitrary user programs
  • Program stored in ROM or flash — runs automatically on power-on without booting an OS
  • Limited or no user interface — may have buttons/LEDs but not a keyboard/monitor
  • Real-time response — must respond within a guaranteed time frame (critical for safety applications)
  • Low power, compact, low cost — designed to be small and energy-efficient
  • Highly reliable — must work continuously without crashing (e.g., in a pacemaker)
  • Difficult to update — firmware updates may be infrequent or impossible once deployed

Examples

DeviceDedicated Function
Washing machineControls water temperature, spin speed, and cycle timing based on selected programme
Car engine management (ECU)Monitors sensors (temperature, air flow, RPM) and adjusts fuel injection and ignition timing in real-time
Anti-lock braking system (ABS)Detects wheel lock-up and rapidly pulses brakes to maintain traction
Traffic light controllerCycles through red/amber/green based on timers and vehicle sensor inputs
PacemakerMonitors heart rhythm and delivers electrical pulses to maintain regular heartbeat
Smart thermostatMonitors temperature sensor and controls heating system to maintain set temperature
Microwave ovenControls magnetron power level and timer based on user input
GPS navigation unitReceives satellite signals and calculates position/route

Embedded vs General-Purpose Computers

FeatureEmbedded SystemGeneral-Purpose Computer

For embedded systems questions, always give a specific example and state its dedicated function. Do not just say "a washing machine" — say "a washing machine controller that manages water temperature, spin speed, and cycle timing." Also know that embedded systems use ROM (non-volatile, runs automatically) rather than loading software from a hard drive.
