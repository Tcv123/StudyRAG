1.5 Systems Software

OCR GCSE Computer Science (J277) — Paper 1

Systems software manages the computer's hardware resources and provides a platform for application software to run. It acts as an intermediary between the user, application software, and hardware. The two main types are the operating system and utility software.

1.5.1 Operating System Functions

The operating system (OS) is the most important piece of systems software. Without it, the user would have to interact directly with the hardware using machine code. The OS manages six key functions:

OS FunctionWhat It DoesDetail and Examples
User interface Provides a way for the user to interact with the computer GUI (Graphical User Interface): windows, icons, menus, pointer (WIMP). Intuitive, easy to learn (e.g., Windows, macOS). CLI (Command Line Interface): typed text commands. More powerful and flexible; preferred by system administrators and programmers (e.g., Linux terminal, PowerShell). Touch interface: finger gestures on a touchscreen (e.g., iOS, Android).
Memory management Allocates and deallocates RAM to programs and manages virtual memory When a program opens, the OS allocates a block of RAM for it. When it closes, that RAM is freed for other programs. If RAM is full, the OS uses virtual memory (a section of the hard drive/SSD as overflow). The OS also provides memory protection — preventing one program from accessing another program's memory space (which would cause crashes or security vulnerabilities). Paging divides memory into fixed-size blocks and swaps them between RAM and disk as needed.
Process (multitasking) management Schedules CPU time between processes to enable multiple programs to appear to run simultaneously The OS uses scheduling algorithms (e.g., Round Robin) to rapidly switch between processes, giving each a small time slice of CPU time. This happens so fast (thousands of times per second) that the user perceives all programs running simultaneously. The OS also manages process creation, termination, and inter-process communication.
Peripheral management Communicates with hardware devices (keyboard, mouse, printer, monitor, etc.) Uses device drivers — specialised software that translates generic OS commands into device-specific instructions. Each hardware device needs a driver. The OS manages device drivers and routes data between devices and programs. Plug-and-play allows the OS to automatically detect and install drivers for new devices.
User management Creates and manages user accounts, access rights, and security Each user has an account with a username and password. The OS assigns access levels (administrator, standard user, guest) that determine what each user can do (install software, change settings, access files). Enforces the principle of least privilege. Manages login/logout and authentication (including multi-factor).
File management Organises files in a hierarchical directory (folder) structure The OS provides: file naming (extensions indicate type: .docx, .py, .jpg); directory structure (folders within folders); file operations (create, copy, move, rename, delete, search); file permissions (read, write, execute, by user/group); file allocation (tracking which disk sectors belong to which files using file systems like NTFS, ext4, APFS).

1.5.2 Utility Software

Utility software performs specific maintenance and housekeeping tasks to keep the system running efficiently. Utilities are part of systems software (not application software) because they maintain the system rather than perform user tasks.

UtilityPurposeHow It WorksKey Detail
Encryption software Protects data confidentiality Encrypts files, folders, or entire drives with a cryptographic key. Data becomes unreadable without the correct decryption key. Full-disk encryption (e.g., BitLocker on Windows, FileVault on macOS) protects all data if the device is lost or stolen. The thief gets the hardware but cannot read any of the data.
Defragmentation Reorganises fragmented files on an HDD Over time, files are stored in non-contiguous (scattered) blocks across the disk. Defragmentation rearranges them into contiguous blocks, reducing the distance the read/write head must travel. Only applies to HDDs (which have mechanical heads that must physically move). SSDs should NOT be defragmented — they have no moving parts (random access is equally fast from any location), and defragmentation causes unnecessary writes that reduce SSD lifespan. SSDs use TRIM instead.
Data compression Reduces file sizes for storage and transmission Lossless (ZIP, PNG): compresses by finding patterns; original data fully recoverable. Lossy (JPEG, MP3): removes data humans cannot easily perceive; smaller files but some quality loss; not reversible. Compression is essential for: reducing download times, fitting more data on storage devices, reducing bandwidth usage for streaming, and emailing large files.
Backup software Creates copies of data for disaster recovery Full backup: copies everything (slow, large). Incremental: copies only files changed since the last backup (fast, small, but restoration requires all increments). Differential: copies files changed since the last full backup. Follows the 3-2-1 rule: 3 copies of data, on 2 different media types, with 1 copy offsite (or in the cloud). Protects against hardware failure, ransomware, theft, fire, and accidental deletion.
Anti-malware Detects and removes malicious software Scans files using signature detection (comparing against a database of known malware patterns) and heuristic/behavioural analysis (detecting suspicious activity patterns that indicate unknown malware). Must be kept updated — hundreds of thousands of new malware variants are created daily. Provides real-time protection (scanning files as they are opened/downloaded) and scheduled full-system scans.
The OS manages hardware and provides a platform for applications; utility software performs specific maintenance tasks. These are different roles. Defragmentation only benefits HDDs (not SSDs) — this is a very common exam question. For memory management, be able to explain the process of moving data between RAM and secondary storage (virtual memory / paging) and why this causes slowdown (disk access is much slower than RAM access).
