Factors Affecting Network Performance

  • Bandwidth — the maximum data transfer capacity of the connection (measured in Mbps or Gbps). Higher bandwidth = more data per second. Like a wider pipe carrying more water.
  • Number of connected devices — more devices sharing bandwidth means less for each. A network with 100 users will be slower per user than one with 10.
  • Latency — the delay between sending data and it arriving. Affected by distance, number of router hops, and congestion. Measured in milliseconds (ms). Critical for gaming and video calls.
  • Wired vs wireless — wired connections (Ethernet) are generally faster, more reliable, and more secure than wireless (Wi-Fi).
  • Interference / obstacles — wireless signals are weakened by walls, floors, and electromagnetic interference from other devices (microwaves, Bluetooth, neighbouring Wi-Fi networks).
  • Network hardware quality — older switches, routers, or cables can bottleneck performance even if the internet connection is fast.

1.3.2 Network Topologies

A network topology is the arrangement of devices (nodes) and connections (links) in a network. It determines how data flows and affects performance, reliability, and cost.

Star Topology

All devices connect to a central switch (or hub) via individual cables. The most common LAN topology.

AdvantagesDisadvantages
If one cable fails, only that device is affected — the rest of the network continues workingThe central switch is a single point of failure — if it fails, the entire network goes down
Easy to add new devices without disrupting the network (just plug into a spare switch port)Requires more cabling than a bus topology (individual cable per device)
Easy to troubleshoot — faults can be isolated to individual cables or devicesThe switch can be expensive (enterprise-grade managed switches)
No collisions when using a switch (each port is a separate collision domain), so performance is consistent
High performance — each device has a dedicated connection to the switch

Mesh Topology

Full mesh: every device connects directly to every other device. Partial mesh: some (but not all) devices have redundant connections.

AdvantagesDisadvantages
Very resilient — if one connection fails, data can be rerouted via alternative paths. No single point of failure.Very expensive — requires a large amount of cabling (full mesh of n devices needs n(n-1)/2 connections)
Multiple data paths — data can take the fastest or least congested routeComplex to set up, configure, and maintain
Fault tolerant — ideal for critical infrastructureImpractical for large networks due to the number of connections required

The internet uses a partial mesh topology — core routers have multiple redundant connections to ensure reliability, but not every device connects to every other.

1.3.5 The Internet, DNS and the Cloud

The internet is a global network of interconnected networks communicating via the TCP/IP protocol suite. It is the physical infrastructure (cables, routers, servers).
The World Wide Web (WWW) is a collection of web pages accessed via browsers using HTTP/HTTPS. The web is a service that runs on the internet — they are not the same thing. Other internet services include email, FTP, VoIP, and streaming.

DNS (Domain Name System)

The DNS converts human-readable domain names (e.g., www.bbc.co.uk) into IP addresses (e.g., 151.101.0.81) that computers use to route data. It is like the internet's phone book.

  1. User types www.bbc.co.uk in the browser.
  2. The browser checks its local DNS cache. If not found, it queries a DNS server (usually provided by the ISP).
  3. The DNS server looks up the domain name and returns the corresponding IP address.
  4. The browser uses the IP address to connect to the BBC's web server.
  5. The web server responds with the requested web page.

Web Hosting

A web server stores website files (HTML, CSS, JavaScript, images) and serves them to clients. When a browser sends an HTTP request (e.g., GET /index.html), the web server processes the request and returns the appropriate files as an HTTP response.

The Cloud

AdvantagesDisadvantages
Accessible from anywhere with an internet connection, from any deviceRequires a reliable internet connection — no internet = no access
Automatic backups and redundancy (data replicated across data centres)Ongoing subscription cost (monthly/annual fees that accumulate over time)
Scalable — easily increase storage or computing power on demandData held by a third party — privacy and security concerns (who can access your data?)
No local hardware to maintain, update, or replacePotential data sovereignty issues — data may be stored in another country with different laws

1.3.6 Protocols and the TCP/IP Model

A protocol is a set of rules that govern how data is formatted, transmitted, and received across a network. Protocols ensure that devices from different manufacturers can communicate reliably. Without agreed protocols, devices would not understand each other's data.

The TCP/IP Model (4 Layers)

Data passes down through the layers on the sending side (each layer adds its own header — encapsulation) and up through the layers on the receiving side (each layer strips its header — de-encapsulation).

LayerNameFunctionProtocols
4 (Top)ApplicationProvides services directly to the user's application (web browsing, email, file transfer)HTTP, HTTPS, FTP, SMTP, IMAP, POP, DNS
3TransportBreaks data into segments; provides reliable (TCP) or fast (UDP) delivery between applications using port numbersTCP (reliable, ordered), UDP (fast, no guarantee)
2InternetAdds source and destination IP addresses; routes packets across networksIP (IPv4, IPv6)
1 (Bottom)LinkHandles physical transmission of frames over the network medium using MAC addressesEthernet, Wi-Fi

Wired vs Wireless Connections

FeatureWired (Ethernet/Fibre)Wireless (Wi-Fi)
SpeedGenerally faster and more consistentVariable; affected by distance and interference
ReliabilityVery reliable (dedicated cable per device)Less reliable (signal can drop, interference from walls/devices)
SecurityMore secure (physical access to cable needed to intercept)Less secure (signals broadcast through air; requires WPA2/WPA3 encryption)
PortabilityNot portable (device must be near a cable/socket)Portable (move freely within range)
CostHigher installation cost (cabling infrastructure)Lower installation cost (just need a WAP)

Bluetooth: Short-range wireless technology (typically 10m) for pairing personal devices (headphones, keyboards, smartwatches). Uses less power than Wi-Fi. Not designed for network access.

HTTP vs HTTPS: always mention that HTTPS uses SSL/TLS encryption to protect data in transit, while HTTP sends data in plaintext. DNS converts domain names to IP addresses — examiners want both parts stated explicitly. For client-server vs P2P: client-server is centralised and easier to manage/secure; P2P is cheaper but harder to secure.
