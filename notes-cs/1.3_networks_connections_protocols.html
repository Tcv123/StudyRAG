1.3.1 Network Types and Performance

A computer network is two or more computing devices connected together for the purpose of sharing data, resources, and communication. Networks allow users to share files, printers, internet access, and software licences, and enable communication via email, messaging, and video calls.
FeatureLAN (Local Area Network)WAN (Wide Area Network)
CoverageSmall geographic area — a single building, home, school campus, or officeLarge geographic area — cities, countries, or the entire world
Infrastructure ownershipAll hardware (cables, switches, routers) owned and managed by the organisationUses third-party communication links leased from telecoms providers (e.g., BT, Virgin Media). Undersea fibre optic cables, satellite links.
Connection typeEthernet cables (Cat5e/Cat6) or Wi-FiLeased lines, fibre optics, satellite, microwave links, submarine cables
SpeedTypically fast (100 Mbps to 10 Gbps)Variable; often slower than LAN due to distance and shared infrastructure
SecurityEasier to secure (physical access control, single admin, limited entry points)Harder to secure (data travels over public infrastructure, more attack vectors)
ExampleSchool network, home Wi-Fi, office LANThe internet (the world's largest WAN), a bank connecting branches across the country
The key distinction between LAN and WAN is ownership of infrastructure, not just size. A LAN uses infrastructure owned by the organisation; a WAN leases third-party infrastructure. The internet is the world's largest WAN — it connects millions of LANs together using leased and shared infrastructure.

Factors Affecting Network Performance

Network performance depends on several factors that determine how fast data can be transferred and how responsive the network feels:

  • Bandwidth — the maximum data transfer capacity of the connection (measured in Mbps or Gbps). Higher bandwidth = more data per second. Bandwidth is the theoretical maximum; actual throughput is usually lower due to overhead and congestion. Think of bandwidth as the width of a pipe: a wider pipe can carry more water per second.
  • Number of connected devices — more devices sharing bandwidth means less available for each. A network with 100 users will be slower per user than one with 10, because bandwidth is shared. This is especially noticeable on wireless networks where all devices share the same radio channel.
  • Latency — the delay between sending data and it arriving at its destination. Affected by distance, number of router hops, congestion, and processing delays at each node. Measured in milliseconds (ms). Low latency is critical for gaming, video calls, and remote surgery. Even with high bandwidth, high latency makes a connection feel sluggish.
  • Wired vs wireless — wired connections (Ethernet) are generally faster, more reliable, lower latency, and more secure than wireless (Wi-Fi). Wired connections also do not suffer from interference.
  • Interference / obstacles — wireless signals are weakened by walls, floors, metal objects, and electromagnetic interference from other devices (microwaves, Bluetooth, neighbouring Wi-Fi networks). The further from the access point, the weaker the signal and the slower the connection.
  • Network hardware quality — older switches, routers, or cables can bottleneck performance. A Cat5 cable limits you to 100 Mbps even if your internet connection is 1 Gbps. Upgrading to Cat6 cables and modern switches can dramatically improve LAN performance.
  • Network congestion — during peak times (e.g., evenings when everyone is streaming video), shared infrastructure becomes congested, reducing speed for everyone.

1.3.2 Network Topologies

A network topology is the arrangement of devices (nodes) and connections (links) in a network. It determines how data flows and affects performance, reliability, cost, and ease of maintenance. There are two types: physical topology (how devices are physically connected) and logical topology (how data actually flows).

Star Topology

All devices connect to a central switch (or hub) via individual cables. The most common LAN topology in homes, schools, and businesses.

AdvantagesDisadvantages
If one cable fails, only that device is affected — the rest of the network continues working normallyThe central switch is a single point of failure — if it fails, the entire network goes down (mitigated by having a backup switch)
Easy to add new devices without disrupting the network (just plug into a spare switch port)Requires more cabling than a bus topology (individual cable per device), increasing installation cost
Easy to troubleshoot — faults can be isolated to individual cables or devices quicklyThe switch itself can be expensive, especially enterprise-grade managed switches
No collisions when using a switch (each port is a separate collision domain), so consistent performanceLimited by the number of ports on the switch (though switches can be daisy-chained)
High performance — each device has a dedicated connection to the switch
Secure — switch sends data only to the intended recipient, not broadcast to all

Mesh Topology

Full mesh: every device connects directly to every other device. Partial mesh: some (but not all) devices have redundant connections. The internet's core infrastructure uses a partial mesh.

AdvantagesDisadvantages
Very resilient — if one connection fails, data can be rerouted via alternative paths. No single point of failure.Very expensive — requires a large amount of cabling. Full mesh of n devices needs n(n-1)/2 connections (e.g., 10 devices need 45 connections).
Multiple data paths — data can take the fastest or least congested route, improving performanceComplex to set up, configure, and maintain. Each device needs multiple network interfaces.
Fault tolerant — ideal for critical infrastructure where downtime is unacceptableImpractical for large networks due to the exponential growth in connections required

The internet uses a partial mesh topology — core routers have multiple redundant connections to ensure reliability, but not every device connects to every other. This balances cost with resilience.

1.3.5 The Internet, DNS and the Cloud

The internet is a global network of interconnected networks communicating via the TCP/IP protocol suite. It is the physical infrastructure (cables, routers, servers, exchanges).
The World Wide Web (WWW) is a collection of web pages and resources accessed via browsers using HTTP/HTTPS. The web is a service that runs on the internet — they are not the same thing. Other internet services include email (SMTP/IMAP), file transfer (FTP), voice calls (VoIP), video streaming, and online gaming.

DNS (Domain Name System)

The DNS converts human-readable domain names (e.g., www.bbc.co.uk) into IP addresses (e.g., 151.101.0.81) that computers use to route data. Without DNS, you would need to memorise numeric IP addresses for every website. DNS is often described as the internet's phone book.

How a DNS lookup works (step by step):

  1. User types www.bbc.co.uk in the browser.
  2. The browser checks its local DNS cache (recently visited sites are remembered). If found, the cached IP is used immediately.
  3. If not cached locally, the request goes to the ISP's DNS server (recursive resolver).
  4. If the ISP's DNS server does not know the answer, it queries the DNS hierarchy: root server → TLD server (.co.uk) → authoritative DNS server for bbc.co.uk.
  5. The authoritative DNS server returns the IP address (e.g., 151.101.0.81).
  6. The IP address is cached at each level for future requests (with a time-to-live expiry).
  7. The browser uses the IP address to connect to the BBC's web server and request the page.

Web Hosting and HTTP Requests

A web server stores website files (HTML, CSS, JavaScript, images, databases) and serves them to clients on request. The process works like this:

  1. The browser creates an HTTP GET request for a specific resource (e.g., GET /index.html HTTP/1.1).
  2. This request is sent as TCP/IP packets to the web server's IP address (resolved by DNS).
  3. The web server processes the request, locates the file, and sends back an HTTP response containing the file data and a status code (e.g., 200 OK for success, 404 Not Found if the page does not exist).
  4. The browser receives the response and renders (displays) the web page.

1.3.6 Protocols and the TCP/IP Model

A protocol is a set of rules that govern how data is formatted, transmitted, and received across a network. Protocols ensure that devices from different manufacturers, running different operating systems, can communicate reliably. Without agreed protocols, a Windows PC could not communicate with a Mac or a Linux server. Protocols are like languages — both sides must speak the same language to understand each other.

The TCP/IP Model (4 Layers)

The TCP/IP model divides network communication into four layers. Each layer has a specific responsibility and uses specific protocols. Data passes down through the layers on the sending side (each layer adds its own header — encapsulation) and up through the layers on the receiving side (each layer strips its header — de-encapsulation).

LayerNameFunctionProtocolsData Unit
4 (Top)ApplicationProvides services directly to the user's application (web browsing, email, file transfer). Determines the format and meaning of the data.HTTP, HTTPS, FTP, SMTP, IMAP, POP, DNSData / Message
3TransportBreaks data into segments; provides reliable delivery (TCP) or fast, connectionless delivery (UDP). Uses port numbers to identify which application should receive the data.TCP (reliable, ordered, error-checked), UDP (fast, no guarantee, no order)Segment
2InternetAdds source and destination IP addresses; handles routing of packets across networks. Each router examines the destination IP and forwards the packet towards its destination.IP (IPv4, IPv6), ICMPPacket
1 (Bottom)Link / Network AccessHandles physical transmission of frames over the network medium. Uses MAC addresses for local delivery within a LAN.Ethernet, Wi-Fi (802.11)Frame

Encapsulation example: When you send an email, the email data starts at the Application layer (SMTP). The Transport layer wraps it in a TCP segment with port numbers. The Internet layer wraps that in an IP packet with IP addresses. The Link layer wraps that in an Ethernet frame with MAC addresses. At the receiving end, each layer strips its header in reverse order until the original email data is delivered to the email application.

The TCP/IP Protocol Stack

The TCP/IP protocol stack is a set of communication protocols that govern how data is transmitted across the internet and other networks. It is arranged into four layers. Each layer adds its own header information as data travels down the stack (encapsulation) and strips it as data travels up (de-encapsulation). This layered approach means each layer only needs to worry about its own job — the layers above and below handle the rest.

Packet Structure

A packet is the fundamental unit of data transmitted across a network. Each packet has three main parts:

  • Header — control information placed at the front of the packet. Contains:
    • Source IP address — where the packet came from
    • Destination IP address — where the packet is going
    • Source port number — identifies the sending application (e.g., port 443 for HTTPS)
    • Destination port number — identifies the receiving application
    • Sequence number — allows the receiver to reassemble packets in the correct order
    • TTL (Time to Live) — a counter decremented by 1 at each router hop; when it reaches 0, the packet is discarded, preventing it from looping forever
  • Payload — the actual data being transmitted (a fragment of the original file, web page, email, etc.)
  • Trailer — placed at the end; includes a checksum (a calculated value used to detect errors in transmission; the receiver recalculates and compares — if they differ, the packet is corrupted and must be retransmitted)

|     HEADER       |          PAYLOAD            |    TRAILER     |
| (Src IP, Dst IP, |  (actual data fragment)     | (checksum)     |
|  Ports, Seq, TTL)|                             |                |
+------------------+-----------------------------+----------------+

TCP Three-Way Handshake

Before TCP transmits any data, it establishes a reliable connection using a three-way handshake. This ensures both sides are ready to communicate and agree on sequence numbers:

  1. SYN — the client sends a synchronise (SYN) packet to the server: "I want to connect; my sequence number starts at X."
  2. SYN-ACK — the server replies with a SYN-ACK: "Acknowledged your SYN (X+1). I want to connect too; my sequence number starts at Y."
  3. ACK — the client sends an acknowledgement (ACK): "Acknowledged your SYN (Y+1). Connection established."
Client                          Server
  |-------- SYN (seq=X) -------->|
  |<----- SYN-ACK (ack=X+1) ----|
  |-------- ACK (ack=Y+1) ------>|
          Connection open

After the handshake, data flows in both directions. TCP also uses a four-way termination (FIN, FIN-ACK, FIN, ACK) to close the connection cleanly.

DHCP — Dynamic Host Configuration Protocol

DHCP automatically assigns network configuration to devices when they join a network. Without DHCP, every device would need to be manually configured with an IP address — impractical in large networks or for guests.

When a device joins a network, it broadcasts a DHCP request. The DHCP server (usually the router) responds with:

  • IP address — a unique address for the device on this network (e.g., 192.168.1.15)
  • Subnet mask — defines which part of the IP address identifies the network vs the host (e.g., 255.255.255.0)
  • Default gateway — the router's IP address (used when sending data outside the LAN)
  • DNS server address — the address of the DNS server to use for name resolution

The IP address is leased for a set time. When the lease expires, the device must request a renewal — this is why your device's IP address can occasionally change.

DNS — Domain Name System (Detailed)

DNS is a hierarchical, distributed database that translates human-readable domain names into IP addresses. The hierarchy works as follows:

  1. Local cache — the device checks if it has recently resolved the domain and cached the result. If found, the cached IP is used immediately (fastest).
  2. ISP's recursive resolver — if not cached locally, the query goes to the ISP's DNS server, which searches its own cache.
  3. Root servers — if the ISP's server does not know, it queries one of the 13 root server clusters, which know the addresses of TLD servers.
  4. TLD (Top-Level Domain) server — the root server directs the query to the TLD server for the domain suffix (e.g., .com, .co.uk, .org).
  5. Authoritative name server — the TLD server points to the authoritative server for the specific domain (e.g., google.com), which holds the definitive IP address record and returns it.

Each result is cached at every level with a TTL (Time to Live) value — this is how long it should be kept before re-querying. Caching dramatically speeds up future lookups for popular sites.

NAT — Network Address Translation

NAT allows a router to map many private IP addresses (inside the LAN) to a single public IP address (facing the internet). This conserves the IPv4 address space and provides a basic layer of security (internal devices are not directly accessible from the internet).

When a device inside the LAN sends a request to the internet: the router replaces the private source IP with its public IP and records the mapping. When the response arrives, the router looks up the mapping and forwards the packet to the correct internal device. From the internet's perspective, all traffic appears to come from one public IP address.

DHCP automatically assigns IP address, subnet mask, default gateway, and DNS server — know all four items. DNS translates domain names to IP addresses — examiners want both parts stated. NAT allows multiple devices to share one public IP address — it extends the usable life of IPv4 and provides a degree of security by hiding internal IP addresses from the internet.

Wireless Security

  • WPA2 (Wi-Fi Protected Access 2) — uses AES-CCMP encryption (128-bit key). The current standard for most home and business Wi-Fi. Replaced the weaker WEP and WPA standards. Can be vulnerable to offline dictionary attacks against weak passwords.
  • WPA3 — the latest standard, introduced in 2018. Uses SAE (Simultaneous Authentication of Equals) handshake instead of PSK, making offline dictionary attacks much harder. Forward secrecy means past sessions cannot be decrypted even if the password is later compromised. Increasingly required for new Wi-Fi certified devices.
  • WEP (Wired Equivalent Privacy) — obsolete; broken encryption; must not be used. Can be cracked in minutes.

When comparing wired and wireless, always address at least four of: speed, reliability, security, latency, convenience, and cost. Examiners expect comparisons, not just descriptions — always say "wired is faster than wireless because..." rather than just "wired is fast." Network Protocols — Reference Table

A protocol is a set of agreed rules that govern how data is formatted, transmitted, received, and acknowledged between devices. Protocols allow devices from different manufacturers running different software to communicate reliably. Port numbers identify which application or service a packet is destined for — like a door number in a building. The IP address gets the packet to the building (device); the port number directs it to the right room (application).
