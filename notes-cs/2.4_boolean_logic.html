Boolean logic underpins all decision-making in computers. Every condition in every program evaluates to either True (1) or False (0). Logic gates implement Boolean operations in hardware circuits.

Understanding Each Gate

  • AND — Think of it as two switches in series (one after the other). The light only turns on if BOTH switches are on. In code: if age >= 18 AND hasTicket == True then enter()
  • OR — Think of two switches in parallel. The light turns on if EITHER switch is on. In code: if isStudent == True OR isChild == True then applyDiscount()
  • NOT — An inverter. If the input is on, the output is off, and vice versa. In code: if NOT loggedIn then showLoginPage()
  • NAND — AND followed by NOT. It is the exact opposite of AND. Output is 0 only when both inputs are 1. NAND is a universal gate (any other gate can be built from NAND gates alone).
  • NOR — OR followed by NOT. The exact opposite of OR. Output is 1 only when both inputs are 0. Also a universal gate.
  • XOR (Exclusive OR) — True when inputs are different. False when they are the same. Used in: parity checking, binary addition (half adder sum bit), simple encryption (XOR cipher), toggling bits.

How to Construct a Truth Table

  1. List all input variables as column headers on the left.
  2. Calculate the number of rows needed: 2n rows for n inputs (2 inputs = 4 rows; 3 inputs = 8 rows).
  3. Fill in all possible input combinations systematically (count in binary).
  4. Add columns for any intermediate sub-expressions (work these out before the final output).
  5. Calculate the final output column using the intermediate results.
