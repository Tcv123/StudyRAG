D1.2: Networks and Algorithms

1. Networks (Weighted Graphs)

A network is a graph where each edge has an associated weight (often representing distance, cost, time, or capacity).

2. Minimum Spanning Tree (MST)

A minimum spanning tree of a connected network is a spanning tree with minimum total weight.

Kruskal's Algorithm

  1. Sort all edges in increasing order of weight.
  2. Add each edge in turn, skipping any edge that would form a cycle.
  3. Stop when \(n - 1\) edges have been added.

Prim's Algorithm

  1. Start at any vertex; mark it as "in the tree".
  2. Repeatedly add the smallest-weight edge connecting a vertex in the tree to a vertex outside.
  3. Stop when all vertices are in the tree.

Prim's works well on the matrix form: cross out the start vertex's row, select the smallest entry in the start's column; circle it; cross out that column's row; etc.

OCR Exam Tip: When asked to apply Prim's or Kruskal's, show the order of edge selection and circle/underline each selected edge. Partial credit is given for the process, not just the final answer.

3. Shortest Path — Dijkstra's Algorithm

Finds the shortest path from a single source \(S\) to all other vertices in a weighted network with non-negative weights.

  1. Label \(S\) with permanent label 0.
  2. For each unlabelled neighbour of the most recently labelled vertex, set a temporary label = (label of current) + (edge weight), if this is smaller than the existing temporary label.
  3. Choose the vertex with the smallest temporary label — make it permanent.
  4. Repeat from step 2 until all vertices are permanent.
  5. Trace back from target to source using the permanent labels.

4. Route Inspection (Chinese Postman Problem)

Goal: find the shortest closed walk that traverses every edge of a network (typically at least once), returning to the start.

Algorithm

  1. Identify all odd-degree vertices.
  2. Pair them up in all possible ways.
  3. For each pairing, compute the sum of shortest paths between paired vertices.
  4. Choose the pairing with minimum sum — these paths must be repeated.
  5. Length of optimal route = total edge weight + minimum repeated sum.
If a network has 4 odd vertices \(A, B, C, D\), the three possible pairings are:
(\(AB, CD\)), (\(AC, BD\)), (\(AD, BC\)). Compute shortest path in each pair and choose the pairing with smallest total.

5. Travelling Salesman Problem (TSP)

TSP: find the shortest Hamiltonian cycle in a weighted network (visit every vertex exactly once, return to start).

TSP is NP-hard — no known polynomial-time exact algorithm. We use bounds.

Lower Bound (Delete–Least–Remaining method)

For each vertex \(v\):

  1. Remove \(v\) and find an MST of the remaining graph (weight \(M\)).
  2. Add back the two smallest edges incident with \(v\) (weights \(a, b\)).
  3. Lower bound candidate: \(M + a + b\).

Take the maximum over all \(v\) — this gives the best lower bound.

Upper Bound (Nearest Neighbour)

  1. Start at any vertex.
  2. Travel to the nearest unvisited vertex.
  3. Repeat until all vertices visited, then return to start.

Repeat with different starting vertices — take the smallest cycle length as the tightest upper bound.

Interpretation

The optimal TSP length lies between these two bounds. If they agree, you've found it exactly. Otherwise, you have a range.

6. Travel on a Directed Graph (Contextual)

Dijkstra works on directed networks: when updating, only use outgoing edges from the current vertex. All other steps unchanged.

7. Exam Style Strategy

  • For MST: Prim's is easier by hand when working from a matrix; Kruskal's is easier with a sorted edge list.
  • For Dijkstra's: keep labels tidy in boxes; cross out superseded temporary labels.
  • For route inspection, list all pairings explicitly — marks for method even if arithmetic slips.
  • For TSP, be clear about lower vs. upper bound — these are not the answer, just estimates.
